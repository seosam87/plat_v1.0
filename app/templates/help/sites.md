# Управление сайтами

Центральный реестр WordPress-сайтов, подключённых к платформе. Все остальные модули (ключевые слова, позиции, краулинг, контент-пайплайн) работают в контексте конкретного сайта.

## Что можно делать

### Добавление сайта
- **Имя** — произвольное название для удобства (например, «Poly-MSK»)
- **URL** — адрес сайта без trailing slash (например, `https://poly-msk.ru`)
- **WP Username** — логин WordPress с правами Application Password
- **Application Password** — генерируется в WP Admin → Users → Application Passwords. Хранится зашифрованным (Fernet), никогда не показывается в UI и логах

### Верификация подключения
Кнопка **Verify** отправляет запрос к `{url}/wp-json/wp/v2/users/me` с Application Password. Статусы:
- **connected** — WordPress отвечает 200, авторизация успешна
- **failed** — ответ не 200, неверный пароль или сайт недоступен
- **unknown** — проверка ещё не проводилась

### Автоопределение SEO-плагина
При успешной верификации платформа автоматически запрашивает `/wp-json/wp/v2/posts?per_page=1` и определяет SEO-плагин по полям ответа:
- **yoast** — найдено поле `yoast_head` или `yoast_head_json`
- **rankmath** — найдено поле `rank_math_title` или `rank_math_description`
- **unknown** — плагин не обнаружен (нет постов, или используется другой плагин)

Результат сохраняется в поле `seo_plugin` и обновляется при каждой верификации.

### Включение / Отключение
- **Disable** — все фоновые задачи (краулинг, проверка позиций, контент-пайплайн) пропускают этот сайт
- **Enable** — возобновляет обработку

### Удаление сайта
Удаляет сайт и каскадно все связанные данные: ключевые слова, позиции, краулы, задачи, расписания. **Необратимо.**

### Группы сайтов
Сайт может принадлежать группе (например, «Клиенты Москва»). Группы используются для контроля доступа — пользователь с ролью manager/client видит только сайты своей группы.

## Расписания
На странице детали сайта настраиваются два расписания:
- **Crawl Schedule** — краулинг (manual / daily / weekly)
- **Position Check Schedule** — проверка позиций (manual / daily / weekly / every 12h)

Расписания хранятся в PostgreSQL и синхронизируются с redbeat (Celery Beat). Переживают перезапуск контейнеров и `FLUSHALL` Redis.

## Страница детали сайта
Кликните по имени сайта в списке → откроется обзор:
- Счётчики: ключевые слова, краулы, открытые задачи
- Quick actions: запустить краулинг, проверить позиции, верифицировать подключение
- Настройка расписаний
- Последние 5 краулов

## Поля в базе данных

| Поле | Описание |
|------|---------|
| name | Отображаемое имя |
| url | Адрес сайта (unique) |
| wp_username | Логин WP |
| encrypted_app_password | Fernet-зашифрованный пароль |
| connection_status | unknown / connected / failed |
| seo_plugin | Определённый SEO-плагин (yoast / rankmath / unknown) |
| site_group_id | Привязка к группе сайтов |
| is_active | Включён / отключён |

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|---------|
| POST | `/sites` | Создать сайт |
| GET | `/sites` | Список сайтов (с учётом доступа) |
| GET | `/sites/{id}` | Детали сайта |
| PUT | `/sites/{id}` | Обновить сайт |
| DELETE | `/sites/{id}` | Удалить сайт |
| PATCH | `/sites/{id}/status` | Включить/отключить |
| POST | `/sites/{id}/verify` | Проверить WP-подключение |
| POST | `/sites/{id}/crawl` | Запустить краулинг |
| GET/PUT | `/sites/{id}/schedule` | Расписание краулинга |

## Ограничения текущей версии

- Нет привязки региона и целевого поисковика к сайту — **по замыслу**: регион и поисковик задаются на уровне ключевого слова (один сайт может отслеживаться в разных регионах и поисковиках одновременно)
- Нет health score (агрегированная оценка здоровья сайта)
- Нет мониторинга SSL-сертификата и uptime
- Нет поддержки мультиязычных сайтов (hreflang)

## Связи с другими модулями

- **Keywords** — ключевые слова привязаны к `site_id`
- **Positions** — позиции записываются для ключей конкретного сайта
- **Crawl** — краулинг запускается для конкретного сайта
- **Pipeline** — контент-пайплайн работает с WP-постами сайта
- **Projects** — проект создаётся для конкретного сайта
- **Tasks** — SEO-задачи привязаны к сайту

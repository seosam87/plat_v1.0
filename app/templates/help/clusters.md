# Кластеры и семантика

Модуль группировки ключевых слов в смысловые кластеры: ручная и автоматическая кластеризация, определение каннибализации, intent-разметка.

## Что можно делать

### Создание кластеров
- Вручную: форма с названием и целевым URL
- Автоматически: кнопка **Auto-Cluster** группирует ключи по пересечению URL в TOP-10
  - Параметр **min_shared** (по умолчанию 3) — минимальное количество общих URL для объединения в кластер
  - Результат — предложения, которые можно принять или отклонить

### Intent-разметка (коммерческость)
Каждому кластеру можно назначить тип запроса:
- **Коммерческий** — «купить», «цена», «заказать»
- **Информационный** — «как», «что такое», «руководство»
- **Навигационный** — бренд, название компании

Выбор intent — через выпадающий список прямо в карточке кластера.

### Обнаружение intent-ошибок
Кнопка **Intent ⚠** — находит коммерческие кластеры, целевой URL которых ведёт на информационную страницу (блог, новости, статья). Это частая SEO-ошибка: коммерческий запрос должен вести на страницу услуги или товара, а не на блог.

Определение «информационной страницы» — по URL-паттернам: `/blog/`, `/news/`, `/article/`, `/post/`, `/category/`, `/tag/`.

### Каннибализация
Страница **Cannibalization** — находит ключи, по которым ранжируются 2+ разных страницы сайта в TOP-100. Это размывает SEO-авторитет.

### Назначение ключей
Ключевые слова назначаются в кластер через API (`POST /clusters/assign`) или при auto-clustering.

### Missing pages
Endpoint `/clusters/sites/{id}/detect-missing-pages` — находит ключи без целевой страницы (ни у ключа, ни у кластера) и создаёт SEO-задачи.

### CSV-экспорт
Кнопка **Export CSV** — выгружает все ключи сайта с кластерами, URL, частотностью.

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|---------|
| POST | `/clusters/sites/{id}` | Создать кластер |
| GET | `/clusters/sites/{id}` | Список кластеров |
| PUT | `/clusters/{id}` | Обновить (name, target_url, intent) |
| DELETE | `/clusters/{id}` | Удалить |
| POST | `/clusters/assign` | Назначить ключи в кластер |
| POST | `/clusters/sites/{id}/auto-cluster` | Авто-кластеризация |
| GET | `/clusters/sites/{id}/cannibalization` | Каннибализация |
| GET | `/clusters/sites/{id}/intent-mismatches` | Intent-ошибки |
| GET | `/clusters/sites/{id}/export.csv` | CSV-экспорт |
| POST | `/clusters/sites/{id}/detect-missing-pages` | Missing pages |

## Ограничения текущей версии

- Intent определяется вручную (полуавтомат через парсинг Яндекса планируется)
- Auto-clustering работает по URL-пересечению, не по полному SERP
- Нет визуализации кластеров в виде дерева/графа

## Связи с другими модулями

- **Keywords** — ключи привязаны к кластерам через `cluster_id`
- **Positions** — данные позиций используются для auto-clustering и каннибализации
- **Projects/Tasks** — missing pages создают SEO-задачи

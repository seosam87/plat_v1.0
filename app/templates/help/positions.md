# Отслеживание позиций

Модуль записи и анализа поисковых позиций по ключевым словам. Данные поступают из нескольких источников: парсинг SERP (Playwright), GSC, Yandex Webmaster.

## Что можно делать

### Просмотр позиций
Таблица показывает последнюю позицию каждого ключевого слова:
- **Позиция** — цветовая индикация: зелёный (TOP-3), синий (TOP-10), серый (TOP-30), красный (ниже)
- **Delta** — изменение относительно предыдущей проверки (▲ рост, ▼ падение)
- **URL** — страница, которая ранжируется
- **Engine** — Google или Yandex

### Distribution bar (распределение)
Горизонтальная полоса над таблицей показывает распределение ключей:
- TOP-3 / TOP-10 / TOP-30 / TOP-100 / Не ранжируется
- Позволяет оценить общую картину без подсчётов вручную

### Фильтры
- **Top N** — показать только TOP-3, TOP-10, TOP-100
- **Engine** — Google, Yandex, или все
- **URL** — текстовый фильтр: какие ключи ранжируются на конкретной странице (поиск по подстроке)

### График истории (Chart.js)
Кнопка 📈 в строке ключа открывает модальное окно с графиком позиций за 90 дней. Ось Y инвертирована (позиция 1 вверху).

### Сравнение позиций по датам
Кнопка **«Сравнить даты»** — выбрать дату A и дату B, увидеть таблицу:
- Позиция на дату A и на дату B
- Delta между датами
- Полезно для оценки эффекта от SEO-работ

### Lost / Gained
Кнопка **«Lost / Gained»** — показывает ключи за последние 7 дней:
- **Gained** — вошли в TOP-10 (раньше не были или были ниже)
- **Lost** — выпали из TOP-10 (раньше были, теперь нет)

### Проверка позиций
Кнопка **«Check Positions»** запускает асинхронную задачу (Celery) для проверки позиций всех ключей сайта.

### Расписание
Настраивается на странице детали сайта:
- manual / daily / weekly / every 12h
- Хранится в PostgreSQL, синхронизируется с Redbeat

## Хранение данных

Таблица `keyword_positions` партиционирована по месяцам (`checked_at`). Партиции создаются автоматически. Это позволяет эффективно хранить миллионы записей позиций.

### Delta
Delta вычисляется при записи: `предыдущая_позиция - текущая_позиция`.
- Положительная delta = рост (стало лучше)
- Отрицательная delta = падение

## Алерты

При падении позиции более чем на N позиций (настраивается через `POSITION_DROP_THRESHOLD`) отправляется уведомление в Telegram.

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|---------|
| GET | `/positions/sites/{id}` | Последние позиции |
| GET | `/positions/sites/{id}/distribution` | Распределение TOP-3/10/30/100 |
| GET | `/positions/sites/{id}/lost-gained` | Ключи, вошедшие/выпавшие из ТОПа |
| GET | `/positions/sites/{id}/compare?date_a=&date_b=` | Сравнение двух дат |
| GET | `/positions/sites/{id}/by-url?url=` | Фильтр по URL ранжирования |
| GET | `/positions/keywords/{id}/history` | История для графика |
| POST | `/positions/sites/{id}/check` | Запуск проверки |
| GET/PUT | `/positions/sites/{id}/schedule` | Расписание |

## Ограничения текущей версии

- Нет visibility index (планируется в Reports)
- Сравнение дат показывает keyword_id, а не фразу (будет доработано)
- Нет экспорта позиций в Excel

## Связи с другими модулями

- **Keywords** — позиции записываются для ключевых слов
- **Sites** — позиции привязаны к `site_id`, расписание на уровне сайта
- **Clusters** — позиции используются для SERP-пересечений
- **Reports** — агрегация для дашборда

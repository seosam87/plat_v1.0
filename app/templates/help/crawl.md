# Краулинг и технический аудит

Модуль краулинга сайтов: обход страниц через Playwright, извлечение SEO-данных, сравнение снапшотов, автоматическое создание задач по ошибкам.

## Что можно делать

### Запуск краулинга
- Кнопка **«Trigger Crawl»** на странице детали сайта или через API
- Краулер обходит страницы в ширину (BFS), начиная с sitemap.xml или главной страницы
- Настройки: max 500 страниц, глубина до 5, задержка 500мс между запросами

### Извлечение SEO-данных
Для каждой страницы извлекается:
- **Title** — заголовок страницы
- **H1** — первый заголовок H1
- **Meta description** — мета-описание
- **Canonical URL** — значение `<link rel="canonical">`
- **noindex** — наличие `noindex` в meta robots
- **Schema.org** — наличие JSON-LD разметки
- **TOC** — наличие оглавления (table of contents)
- **HTTP-статус** — код ответа (200, 404, и т.д.)
- **Тип страницы** — category / article / landing / product (автоклассификация)

### Change Feed (лента изменений)
После каждого краула создаётся снапшот каждой страницы. При повторном крауле снапшоты сравниваются:
- **SEO Changed** — изменились title, H1, или meta description
- **Content Changed** — изменился контент
- **Status Changed** — изменился HTTP-статус
- **New Pages** — новые страницы, не встречавшиеся ранее

### Анализ краула

#### Дублирование Title и H1
Endpoint `/crawls/{id}/analysis/duplicates` — находит страницы с одинаковыми заголовками. Дубли title и H1 — частая SEO-ошибка.

#### Orphan-страницы
Endpoint `/crawls/{id}/analysis/orphans` — страницы без входящих внутренних ссылок. Поисковые роботы не могут обнаружить их через навигацию сайта.

#### Canonical-проблемы
Endpoint `/crawls/{id}/analysis/canonicals` — страницы, где canonical URL отличается от URL самой страницы. Может указывать на дублирование контента или ошибку конфигурации.

#### SEO-полнота
Endpoint `/crawls/{id}/analysis/completeness` — сводка: сколько страниц имеют title, H1, meta description, schema.org, TOC, canonical. Показывает, где есть пробелы.

### Автозадачи
По результатам краула автоматически создаются задачи:
- **404 Not Found** — страница возвращает 404
- **Lost Indexation** — страница приобрела `noindex` (раньше его не было)

### Расписание
- manual / daily / weekly / every 12h
- Настраивается на странице детали сайта
- Хранится в PostgreSQL, синхронизируется с RedBeat (Celery Beat)

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|---------|
| POST | `/sites/{id}/crawl` | Запустить краулинг |
| GET | `/crawls/{id}/pages` | Список страниц краула |
| GET | `/crawls/{id}/analysis/duplicates` | Дубли title и H1 |
| GET | `/crawls/{id}/analysis/orphans` | Orphan-страницы |
| GET | `/crawls/{id}/analysis/canonicals` | Проблемы canonical |
| GET | `/crawls/{id}/analysis/completeness` | Полнота SEO-полей |

## Ограничения текущей версии

- Нет обнаружения redirect chains (цепочек редиректов)
- Нет аудита картинок (битые src, отсутствующий alt)
- Нет сравнения с идеальной структурой (pillar-service-article) — планируется
- Нет сравнения с Sitemap (планируется)

## Связи с другими модулями

- **Sites** — краулинг привязан к сайту
- **Projects/Tasks** — автозадачи создаются из результатов краула
- **Upload (Screaming Frog)** — SF импорт будет дополнять данные краулера

# Контент-пайплайн

Модуль автоматического обогащения контента WordPress: генерация оглавления (TOC), schema.org разметка, внутренние ссылки. Двухэтапный workflow: обработка → проверка → публикация.

## Как работает

### Обработка страницы
1. Получает HTML контента из WordPress REST API
2. **TOC** — извлекает H2/H3 заголовки, генерирует оглавление, вставляет после первого абзаца
3. **Schema.org** — генерирует JSON-LD Article разметку, добавляет если отсутствует
4. **Внутренние ссылки** — находит в тексте фразы из ключевых слов сайта и оборачивает в ссылки (до 5 штук)
5. Вычисляет diff (что добавлено/удалено)
6. Сохраняет результат со статусом **awaiting_approval**

### Просмотр diff
В колонке «Changes» показано количество добавленных/удалённых строк. Клик открывает модальное окно с цветным diff:
- 🟢 Зелёные строки — добавлено
- 🔴 Красные строки — удалено

### Approve / Reject
- **Approve** — отправляет обогащённый контент в WordPress
- **Reject** — отклоняет изменения, ничего не меняет на сайте
- **Bulk Approve / Reject** — массовые операции через чекбоксы

### Rollback
После публикации (статус **pushed**) можно откатить — оригинальный контент сохраняется и будет восстановлен в WordPress.

### История по URL
Кнопка 🕘 — показывает все pipeline-операции для конкретной страницы: когда обрабатывалась, что менялось, был ли rollback.

### Batch Pipeline
Кнопка **Run Batch** — запускает пайплайн для всех страниц сайта, у которых нет TOC или schema.org.

## Статусы job

| Статус | Описание |
|--------|---------|
| pending | Создан, ожидает обработки |
| processing | Обрабатывается Celery-задачей |
| awaiting_approval | Обработан, ждёт одобрения |
| approved | Одобрен, отправляется в WP |
| pushed | Опубликован в WordPress |
| rolled_back | Откачен к оригиналу |
| failed | Ошибка или отклонён |

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|---------|
| POST | `/pipeline/sites/{id}/run` | Запустить для одной страницы |
| POST | `/pipeline/sites/{id}/batch` | Batch для всего сайта |
| GET | `/pipeline/jobs/{id}` | Детали job (с diff) |
| POST | `/pipeline/jobs/{id}/approve` | Одобрить |
| POST | `/pipeline/jobs/{id}/reject` | Отклонить |
| POST | `/pipeline/jobs/{id}/rollback` | Откатить |
| GET | `/pipeline/sites/{id}/jobs` | Список jobs |
| GET | `/pipeline/sites/{id}/history?url=` | История по URL |
| POST | `/pipeline/sites/{id}/bulk-approve` | Массовое одобрение |
| POST | `/pipeline/sites/{id}/bulk-reject` | Массовое отклонение |

## Связи с другими модулями

- **Sites** — пайплайн привязан к сайту и использует WP-подключение
- **Keywords** — ключевые слова с target_url используются для внутренних ссылок
- **Crawl** — batch pipeline берёт страницы из результатов краулинга

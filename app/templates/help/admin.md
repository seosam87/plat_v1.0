# Администрирование

Управление пользователями, группами сайтов, аудит действий, системные настройки.

## Пользователи

### Роли
- **Admin** — полный доступ ко всем сайтам и функциям
- **Manager** — доступ только к сайтам своей группы
- **Client** — ограниченный доступ к назначенным проектам

### Операции
- Создание пользователя (username, email, password, role)
- Редактирование (username, email, role)
- Смена пароля (только admin)
- Деактивация / активация (soft delete)

## Группы сайтов

Группы используются для контроля доступа:
- Сайт принадлежит одной группе (или без группы — только для admin)
- Пользователь назначается в группу и видит только её сайты
- Admin видит все сайты независимо от групп

### Операции
- Создать группу (название, описание)
- Назначить пользователя в группу
- Назначить сайт в группу
- Удалить пользователя из группы

## Аудит (Audit Log)

Страница `/ui/admin/audit` — журнал всех действий в системе:
- **Фильтры:** по типу действия, по пользователю
- **Логируемые действия:** site.create, site.update, site.delete, site.enable, site.disable, user.created, user.updated, user.deactivated, user.activated, user.password_changed

Каждая запись содержит: время, пользователь, действие, тип и ID сущности, дополнительные данные.

## Системные настройки

Страница `/ui/admin/settings` — обзор текущих настроек:

### Crawler
- Режим по умолчанию: httpx + BS4 (Playwright — по запросу)
- Задержка между страницами (ms)
- Максимум страниц за краул

### SERP Parser
- Дневной лимит запросов
- Задержка между запросами (ms)

### Внешние интеграции
- Google Search Console: статус подключения
- Yandex Webmaster: статус подключения
- DataForSEO: статус подключения

Настройки задаются через переменные окружения (.env).

# Ключевые слова

Модуль управления ключевыми словами: добавление, импорт из файлов, группировка, привязка целевых URL.

## Что можно делать

### Добавление вручную
Форма в верхней части страницы: введите фразу, опционально целевой URL и группу. Фраза автоматически приводится к нижнему регистру.

### Импорт из файлов
Страница **Upload Data** (`/ui/uploads`) поддерживает импорт из:
- **Topvisor** — экспорт позиций: фраза, частотность, целевая URL, история позиций по датам
- **Key Collector 4** — экспорт ключей: фраза, группа, позиция Яндекс, URL позиции

Форматы: `.xlsx`, `.csv`, `.tsv`. Колонки определяются автоматически по заголовкам (на русском и английском).

### Дедупликация при импорте
При повторном импорте файла можно выбрать стратегию обработки дубликатов (совпадение по фразе):
- **Пропустить** (skip) — существующие ключи не трогать, добавить только новые
- **Обновить** (update) — обновить непустые поля (частотность, URL, группу) из файла
- **Заменить** (replace) — удалить старый ключ и вставить новый из файла

### Inline-редактирование
Прямо в таблице ключевых слов можно:
- Изменить **целевой URL** — ввести новый адрес и нажать OK
- Изменить **группу** — выбрать из выпадающего списка (применяется мгновенно)

### Группы ключевых слов
- Создаются автоматически при импорте из Key Collector
- Поддерживают вложенность (родительская → дочерняя)
- Фильтр по группам — кнопки над таблицей
- Группу можно сменить inline в таблице

### Удаление
Кнопка **×** в строке ключа удаляет его с подтверждением.

## Поля в базе данных

| Поле | Описание |
|------|---------|
| phrase | Фраза (lowercase, до 1000 символов) |
| frequency | Частотность (из файла импорта) |
| engine | Поисковик: yandex / google |
| region | Регион (текст, например «Москва») |
| group_id | Привязка к группе ключевых слов |
| target_url | Целевая страница (до 2000 символов) |
| cluster_id | Привязка к кластеру (модуль Clusters) |

## Архитектурное решение: регион и поисковик

Регион и поисковик задаются **на уровне ключевого слова**, а не сайта. Это сделано намеренно: один сайт может отслеживаться одновременно в Яндексе (Москва) и Google (РФ), и разные ключи могут иметь разные регионы.

## Ограничения текущей версии

- Нет поиска по фразам в таблице
- Нет bulk-операций (массовое удаление, смена группы)
- Нет дедупликации при ручном добавлении (только при импорте)
- Коммерческость (intent) будет на уровне кластера, не ключевого слова

## Связи с другими модулями

- **Sites** — ключевые слова привязаны к `site_id`
- **Positions** — позиции записываются для ключевых слов
- **Clusters** — ключи объединяются в кластеры
- **Upload** — импорт из Topvisor и Key Collector

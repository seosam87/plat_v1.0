Create `app/templates/macros/empty_state.html` with a single macro `empty_state` that accepts these parameters per D-06:
- `reason` (str, required) — one-sentence explanation why there is no data
- `cta_label` (str, required) — primary button text
- `cta_url` (str, required) — primary button href
- `secondary_label` (str, default=none) — optional text link label
- `secondary_url` (str, default=none) — optional text link href
- `docs_url` (str, default=none) — reserved for future help docs, not wired up

The macro body per D-01, D-02, D-03, D-05, D-07:
1. Outer `<div>` with Tailwind classes: `bg-white rounded-lg shadow-sm border border-gray-200 p-6 my-4`
2. `<p>` with reason text: `class="text-gray-700 font-medium mb-3"` — renders `{{ reason }}`
3. Collapsible how-to section using `<details>/<summary>` (native HTML, no JS):
   - Guard: `{% if caller is defined %}` (CRITICAL — prevents UndefinedError when called without {% call %})
   - `<summary>` text: "Как использовать" with classes `cursor-pointer text-sm font-medium text-indigo-600 select-none`
   - Content div: `class="mt-2 text-sm text-gray-600 space-y-1 pl-2"` containing `{{ caller() }}`
4. CTA buttons div: `class="flex items-center gap-3 flex-wrap"`
   - Primary button: `<a>` with `class="inline-flex items-center px-4 py-2 rounded text-sm font-medium bg-blue-600 text-white hover:bg-blue-700"` — renders `{{ cta_label }}`, href `{{ cta_url }}`
   - Secondary link (conditional on `secondary_label and secondary_url`): `<a>` with `class="text-sm text-indigo-600 hover:underline"` — renders `{{ secondary_label }}`

NO icons (per D-02). NO inline styles (per D-03). NO import in base.html — each template imports explicitly.

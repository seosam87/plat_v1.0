## What Was Built

### Task 1: Kanban page fixes

**Bug 1 — Task creation form:** Added a "+ Create Task" button in the card header (flex row with h2) that toggles a hidden form. The form uses `hx-ext="json-enc"` to POST JSON to `/projects/{{ project_id }}/tasks/create` with fields: title (required), description, url. On success, the page reloads.

**Bug 2 — Breadcrumbs:** Added `{% block breadcrumbs %}` block with clickable Dashboard > Projects > {project_name} > Kanban trail. Projects links to `/ui/projects`.

**Bug 3 — Comment form 422:** The old form used `hx-headers='{"Content-Type":"application/json"}'` but HTMX still sent URL-encoded data. Fixed by:
- Loading `htmx-ext-json-enc@2.0.1` from unpkg CDN
- Adding `hx-ext="json-enc"` to the form element
- Removing `hx-headers` attribute (json-enc handles Content-Type automatically)
- Changed `hx-swap="beforeend"` to `hx-swap="none"` with `location.reload()` since backend returns JSON, not HTML

### Task 2: Plan page fixes

**Fix 1 — project_id context:** Added `"project_id": project_id` to the `ui_plan` handler's template context in `app/main.py` (line 948).

**Fix 2 — Breadcrumbs:** Added `{% block breadcrumbs %}` block with Dashboard > Projects > {project_name} > Content Plan trail.

**Fix 3 — Add Item form:** Added "+ Add Item" button in the header flex row. Hidden form with fields: proposed_title (required), planned_date (date input), notes. Uses json-enc to POST to `/projects/{{ project_id }}/plan`. Reloads on success.

**Fix 4 — Helpful empty state:** Replaced bare `<p>No content plan items yet.</p>` with centered guidance block mentioning the "+ Add Item" button and linking to the Kanban board.

**Fix 5 — Back to Kanban link:** Added `<- Back to Kanban` link at the bottom of the card.

**Bug 3 — Fix comment form 422:**
The comment form sends URL-encoded form data but the backend expects JSON. The form currently has `hx-headers='{"Content-Type":"application/json"}'` but HTMX still sends form-encoded data by default.

Fix: Add `hx-ext="json-enc"` to the form element AND add the HTMX json-enc extension script before the form. The json-enc extension serializes form fields as JSON.

Add before the comments section (before the closing `{% endblock %}`):
```html
<script src="https://unpkg.com/htmx-ext-json-enc@2.0.1/json-enc.js"></script>
```

Update the form tag to include `hx-ext="json-enc"`:
```html
<form class="mt-4 flex gap-2"
  hx-post="/projects/{{ project_id }}/comments"
  hx-target="#comments-list"
  hx-swap="beforeend"
  hx-ext="json-enc"
>
```

Remove the `hx-headers='{"Content-Type":"application/json"}'` line — `json-enc` handles this automatically.

Also add `hx-on::after-request="if(event.detail.successful) this.reset()"` to clear the input after posting.

The `hx-swap="beforeend"` means the response HTML will append. But the backend returns JSON, not HTML. Change to `hx-swap="none"` and add `hx-on::after-request="if(event.detail.successful) { this.reset(); location.reload(); }"` so the comment appears after reload.

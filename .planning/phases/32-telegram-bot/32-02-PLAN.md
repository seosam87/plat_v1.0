   ```

   ```
   All other deps (sqlalchemy, asyncpg, celery, redis, loguru, pydantic-settings) come from the main requirements.txt which is installed in the image.

2. Create `Dockerfile.bot` at project root:
   ```dockerfile
   FROM python:3.12-slim

   WORKDIR /opt/app

   # System deps for asyncpg
   RUN apt-get update && apt-get install -y --no-install-recommends \
       gcc libpq-dev \
       && rm -rf /var/lib/apt/lists/*

   # Install main project requirements (for shared app.models, app.config, etc.)
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt

   # Install bot-specific requirements
   COPY bot/requirements.txt bot/requirements.txt
   RUN pip install --no-cache-dir -r bot/requirements.txt

   # Copy application code (app/ for models+config, bot/ for bot logic)
   COPY app/ app/
   COPY bot/ bot/
   COPY alembic/ alembic/
   COPY alembic.ini .

   CMD ["python", "-m", "bot.main"]
   ```

   Key decisions:
   - Base image: python:3.12-slim (per CLAUDE.md constraint)
   - Copies app/ for model/config/task name access (NOT app.main startup)
   - Copies alembic/ for potential migration running
   - CMD runs `python -m bot.main` (bot entry point, created in Plan 03)
   - No Playwright, no WeasyPrint system deps — bot does not need them
   - gcc + libpq-dev for asyncpg compilation
  </action>

<task type="auto">
  <name>Task 2: docker-compose bot service + Nginx webhook proxy</name>
  <files>docker-compose.yml, nginx/conf.d/app.conf</files>

    - docker-compose.yml

  <action>
1. In `docker-compose.yml`, add `bot` service BEFORE the `volumes:` section (after `flower` service):
   ```yaml
     bot:
       build:
         context: .
         dockerfile: Dockerfile.bot
       command: python -m bot.main
       env_file: .env
       environment:
         TELEGRAM_BOT_PORT: "8443"
       depends_on:
         postgres:
           condition: service_healthy
         redis:
           condition: service_healthy
       restart: unless-stopped
       networks:
         - default
   ```
   IMPORTANT per D-04: bot does NOT depend on `api` service. It depends only on postgres + redis. This ensures graceful degradation — bot keeps running if API is down.

2. In `nginx/conf.d/app.conf`, add webhook proxy:

   a. Add upstream block (near existing upstream blocks if any, or at top):
   ```nginx
   upstream bot_upstream {
       server bot:8443;
   }
   ```

   b. Add location block inside the existing `server { }` block:
   ```nginx
   location /webhook/tg {
       proxy_pass http://bot_upstream;
       proxy_set_header Host $host;
       proxy_set_header X-Real-IP $remote_addr;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_read_timeout 60s;
   }
   ```

   This routes Telegram webhook callbacks through Nginx to the bot container on port 8443, per D-02.
  </action>

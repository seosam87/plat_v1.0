## Spacing Scale

Identical to Phase 20 — no deviations. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge internal padding (horizontal: `.2rem .6rem`) |
| sm | 8px | Compact element spacing, checklist item icon-to-text gap |
| md | 16px | Default element spacing, form field gaps between labels and inputs |
| lg | 24px | Card padding (`.card { padding: 1.5rem }`), tab panel inner padding |
| xl | 32px | Gap between header card and tab bar, gap between tab bar and tab panel |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Page-level breathing room (not used in dense intake views) |

Exceptions:
- Table cell padding: `0.75rem 1rem` (12px vertical, 16px horizontal) — matches existing `th, td` rule in base.html
- Textarea fields (goals, notes): `0.75rem` inner padding — inherits from form pattern
- Checklist item row: `0.75rem 1rem` padding — consistent with table rows

## Typography

Identical to Phase 20. Four roles, two weights only (400 + 600).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 |
| Label / Meta | 13px (0.8rem) | 600 (semibold) | 1.4 |
| Heading (page, card) | 18px (1.125rem / `text-lg`) | 600 (semibold) | 1.2 |
| Display (site name in header) | 20px (1.25rem / `text-xl`) | 600 (semibold) | 1.2 |

Do NOT introduce additional sizes. Form section headings within tabs use Heading (18px). Tab labels use Body (14px) at weight 500.

## Interaction Contracts

Defined from CONTEXT.md decisions D-01 through D-16. These are binding for the executor.

### Intake Entry Points

1. **Site detail page** (`/ui/sites/{id}`): Add "Intake" link/button next to existing badges. If intake exists and is complete, show `badge-connected` "Анкета заполнена". If draft, show `badge-unknown` "Анкета не заполнена" as a link to intake page.
2. **Site list page** (`/ui/sites`): Add intake status column/icon. Green checkmark if complete, gray dash if draft/absent.
3. **CRM sidebar**: Intake child item under CRM section (anticipated in Phase 20 sidebar).

### Intake Form Page (`/ui/sites/{id}/intake`)

- **Layout:** Single page with 5-tab bar, following the CRM detail page tab pattern exactly (D-01).
- **Tab bar:** Horizontal buttons with `border-bottom: 2px solid` active indicator. Default active tab: Доступы (Tab 1).
- **Tab switching:** JavaScript `classList` toggle (same as CRM detail.html `switchTab()` pattern). No full page reload, no HTMX.
- **Tab completion indicator:** Checkmark icon (green-600) appended to tab label text when section is saved (D-11). CSS: `color: #059669; margin-left: 0.3rem;` — simple inline SVG checkmark.
- **Header card:** Site name (Display 20px semibold) + site URL (Body 14px gray-500) + intake status badge + client badge (if client attached, reuse Phase 20 pattern).

### Tab 1: Доступы (Access) — D-04

- **Read-only display** of existing Site model data: WP connection status badge, login username.
- **Link to site settings:** `<a href="/ui/sites/{id}/edit">Настроить доступ</a>` styled as accent link.
- **No editable fields** in this tab — purely informational. "Сохранить раздел" button marks section as reviewed (saves a section flag, no form data).

### Tab 2: Цели и конкуренты (Goals & Competitors) — D-05

- **Fields:**
  - Главная цель: `<textarea rows="4">` — full-width
  - Целевые регионы: `<input type="text">` — full-width
  - Конкуренты: dynamic list of URL inputs (up to 10). Each row: `<input type="url">` + remove button (red-600 X icon). "Добавить конкурента" button appends a new input row via JavaScript (client-side only, no HTMX for add/remove).
  - Заметки: `<textarea rows="3">` — full-width
- **Save:** `hx-post="/ui/sites/{id}/intake/goals" hx-swap="none"` — saves section data, shows toast, marks tab checkmark.

### Tab 3: Аналитика (Analytics Setup) — D-06

- **Read-only indicators:**
  - GSC connection status: badge (connected/not configured/unchecked) pulled from `oauth_tokens` table
  - Metrika counter ID: badge showing counter ID if set, "Не настроено" if absent
  - Yandex region: display current `yandex_region` value or "Не указан"
- **Configure links:** Each indicator has a "Настроить" link pointing to the relevant settings page.
- **Save:** Marks section as reviewed (same as Tab 1 — no editable fields, just confirmation of review).

### Tab 4: Технический SEO (Technical SEO) — D-07

- **Auto-populated read-only:**
  - SEO plugin: display `Site.seo_plugin` value or "Не определен"
  - Sitemap status: badge based on whether sitemap was detected in latest crawl
- **User-editable:**
  - Заметки по robots.txt: `<textarea rows="3">` — free-text notes
- **Save:** `hx-post="/ui/sites/{id}/intake/technical" hx-swap="none"` — saves robots notes + marks section.

### Tab 5: Чеклист верификации (Verification Checklist) — D-08, D-09, D-10

- **Layout:** Card with 5 rows, each row is a checklist item.
- **Each row:** Icon (24px) + label (Body 14px) + status badge (3-state: connected/failed/unknown).
- **Checklist items:**
  1. WP подключен — checks `Site.wp_connection_status`
  2. GSC подключен — checks `oauth_tokens` for GSC
  3. Метрика подключена — checks `Site.metrika_counter_id`
  4. Sitemap найден — checks last crawl sitemap discovery
  5. Краул выполнен — checks if at least one crawl exists for this site
- **Refresh:** "Перепроверить" button: `hx-get="/ui/sites/{id}/intake/checklist" hx-target="#checklist-content" hx-swap="innerHTML"` — re-queries all statuses from DB.
- **Static snapshot on load** (D-08): checklist rendered server-side on page load, not auto-refreshing.
- **No save button** on this tab — checklist is auto-derived. Tab is always marked as "reviewed" once visited.

### Completion Flow — D-12, D-13, D-14

- **"Завершить intake" button:** Always visible in the page header (right side of header card), styled as `btn-primary`.
- **If not all sections saved:** JavaScript `confirm("Не все секции заполнены. Завершить все равно?")` — inline confirm pattern (matches project convention).
- **On confirm:** `hx-post="/ui/sites/{id}/intake/complete" hx-swap="none"` — sets intake status to `complete`, shows success toast, updates badge on page.
- **Reversibility:** Intake can be re-opened by visiting the form again (D-14: status on intake record, not Site model). Re-opening sets status back to `draft`.

### HTMX Partial Save Pattern — D-03

Each tab's "Сохранить раздел" button:
- `hx-post` to section-specific endpoint
- `hx-swap="none"` — no DOM replacement needed
- Server responds with `HX-Trigger: showToast` event header (existing toast pattern)
- On success, JavaScript updates the tab's checkmark indicator
- On error, toast shows error message

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (0.25rem) | Icon gaps, inline badge padding |
| sm | 8px (0.5rem) | Compact table cell padding, button sm |
| md | 16px (1rem) | Default form field gap, card inner padding row |
| lg | 24px (1.5rem) | Card padding (matches `.card` padding: 1.5rem), section heading margin-bottom |
| xl | 32px (2rem) | Main content horizontal padding (matches `.main-content` padding: 1.5rem 2rem) |
| 2xl | 48px (3rem) | Between distinct page sections |
| 3xl | 64px (4rem) | Not used in this phase |

Exceptions:
- Touch targets for icon-only buttons: minimum 44px height (accessibility). Applied via `style="min-height:44px"` on delete/action buttons.
- `max-w-4xl` container on tool index page (matches existing tools/index.html pattern).
- `max-w-5xl` container on tool detail/results pages (matches keyword_suggest/index.html pattern).

## Typography

All sizes use `system-ui, sans-serif`. Exactly 4 size tokens and 2 weight tokens are declared.

**Weights:**
- Regular: 400 (`font-normal`)
- Semibold: 600 (`font-semibold`)

No other weights are permitted in this phase.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Utility / label-small (table headers, badges) | 12px (0.75rem) | 600 | 1.3 | `text-xs font-semibold` |
| Body / table cell | 14px (0.875rem) | 400 | 1.5 | `text-sm` |
| Label / form label / section heading (H2) | 18px (1.125rem) | 600 | 1.3 | `text-lg font-semibold` |
| Page heading (H1) | 24px (1.5rem) | 600 | 1.2 | `text-2xl font-semibold` |

Notes:
- Table headers: 12px, weight 600, uppercase, color `text-gray-500` — matches existing `th` rule in base.html.
- Badge/status text: 12px, weight 600 — matches `.badge` rule. The ambiguous "12-13px" notation is removed; all badge and table-header text uses exactly 12px.
- H1 uses `font-semibold` (600). The 700/bold one-off exception is removed. Visual distinction between 600 and 700 at 24px is negligible.

## Component Inventory

These are the concrete UI components required for Phase 24. Each maps to a Jinja2 template file.

### Page: Tools Index — `/ui/tools/`

**Primary focal point:** Tool name H2 in each card — it is the largest text element on the card and sits directly above the CTA. All other card elements (description, badge) are visually subordinate to it.

Replace existing stub `tools/index.html` with:

- **Tool card grid** — 3 cards, one per tool. Each card contains:
  - Tool name (H2, `text-lg font-semibold text-gray-800`) — focal point
  - One-line description (body text, `text-sm text-gray-600`)
  - "Открыть инструмент" link button → `/ui/tools/{slug}/` (styled as `btn btn-primary` but as `<a>`)
  - Recent job count badge: "N заданий" in gray pill

### Page: Tool Landing — `/ui/tools/{slug}/`

Two-column layout at `max-w-5xl`:

**Left / top: Input form card** (`.card mb-6`)
- `<h2>` with tool name
- Textarea for keyword/URL input (multiline, rows=8 for keywords, rows=10 for URLs)
  - Placeholder shows format hint (one per line)
  - Character/line counter below: "0 / {limit} фраз" — updated via JS `oninput`
  - Limit: Commercialization 200 phrases, Meta-parser 500 URLs, Relevant-URL 100 phrases
- For Relevant-URL tool only: domain input field (single text input, `placeholder="example.com"`)
- Submit button: `btn btn-primary` — label per tool (see Copywriting)
- Rate limit note: "Лимит: 10 запросов в минуту" — `text-xs text-gray-500 ml-auto` (matches keyword_suggest pattern)

**Right / below: Previous jobs list** (`.card`)
- Table: columns = [Статус, Фраз, Создано, Действия]
- Status as colored badge (pending/running/complete/partial/failed)
- "Смотреть результаты" link → `/ui/tools/{slug}/{job_id}`
- Delete button (destructive — see Copywriting)
- Empty state: use `empty_state` macro with tool-specific copy (see Copywriting)

### Page: Tool Results — `/ui/tools/{slug}/{job_id}`

**Breadcrumb:** Dashboard › Инструменты › {Tool Name} › #{job_id}

**Status banner** (HTMX polling partial — `hx-trigger="every 10s"` per D-03):
- pending/running: blue banner with spinner + progress text "Обработка... N/Total фраз" (D-04)
- complete: green banner
- partial: amber banner with XMLProxy balance note
- failed: red banner with error message

**Results card** (`.card mt-4`) — visible only when status is complete or partial:
- Header row: "Результаты" H2 + count badge
- Filter input: `placeholder="Фильтр..."` (client-side JS, same pattern as suggest_results.html)
- Results table — columns differ per tool (see per-tool table schemas below)
- Export row: two buttons side-by-side — "Скачать CSV" + "Скачать XLSX"

**Per-tool table schemas:**

Commercialization (`commercialization`):
| Column | Width | Notes |
|--------|-------|-------|
| Фраза | 40% | keyword text |
| Коммерциализация | 15% | 0–100 integer, sortable |
| Интент | 20% | badge: commercial / informational / navigational |
| Геозависимость | 15% | badge: да / нет |
| Источник | 10% | Yandex badge (red "Я") |

| Column | Width | Notes |

| Title | 25% | truncated at 60 chars |
| Description | 25% | truncated at 120 chars |

Relevant URL (`relevant-url`):
| Column | Width | Notes |
|--------|-------|-------|
| Фраза | 30% | keyword text |
| URL | 30% | matching URL, link |
| Позиция | 10% | integer, sortable |
| Топ-3 конкурента | 30% | comma-separated domains |

## Interaction Contract

### Form Submission

1. User fills textarea + optional domain field → clicks CTA button
2. Form POSTs via HTMX to `/ui/tools/{slug}/` (hx-post)
3. Response: server redirects to `/ui/tools/{slug}/{new_job_id}` OR returns job_id in HX-Redirect header
4. Results page loads; status partial auto-polls every 10 seconds

### Line Counter

- JS `oninput` on textarea counts newlines + 1
- Display: `{count} / {limit} фраз` below textarea
- When count > limit: counter turns red (`color:#dc2626`), submit button disabled

### HTMX Polling

- Status partial rendered at page load
- If status is pending/running: partial self-replaces every 10s via `hx-trigger="load delay:10s"`
- If status is complete/partial/failed: partial renders final state with no further polling
- Spinner: SVG `animate-spin` class (matches existing suggest_status.html pattern)

### Export

- CSV and XLSX export: plain `<a href="/ui/tools/{slug}/{job_id}/export?format=csv">` link
- No HTMX needed — direct file download response

### Delete Job

- `hx-delete` with `hx-confirm` on the delete button in job list
- On success: server returns empty 200, HTMX removes the row (`hx-target="closest tr" hx-swap="outerHTML"`)
- If user is on the results page for the deleted job: redirect to `/ui/tools/{slug}/`

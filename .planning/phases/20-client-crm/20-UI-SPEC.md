## Spacing Scale

All values are multiples of 4. Follows the 8-point scale already used throughout the project (base.html `.card { padding: 1.5rem }`, `.main-content { padding: 1.5rem 2rem }`).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge internal padding (horizontal: `.2rem .6rem`) |
| sm | 8px | Compact element spacing, table cell vertical padding (`.75rem` = 12px exception noted below) |
| md | 16px | Default element spacing, card section gaps, form field gaps |
| lg | 24px | Card padding (`.card { padding: 1.5rem }`), section headers |
| xl | 32px | Gap between header section and tab panel, major layout gaps |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Page-level breathing room (not used in dense CRM views) |

Exceptions:
- Table cell padding: `0.75rem 1rem` (12px vertical, 16px horizontal) — matches existing `th, td` rule in base.html; do not deviate
- Interaction log textarea: `0.75rem` inner padding — inherits from form pattern
- Contact row inline-edit fields: `0.25rem 0.5rem` padding (4px × 8px) — compact sizing on multiples of 4

## Typography

Matches the existing type system in base.html. Four roles used across the CRM section:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 |
| Label / Meta | 13px (0.8rem) | 600 (semibold) | 1.4 |
| Heading (page, card) | 18px (1.125rem / `text-lg`) | 600 (semibold) | 1.2 |
| Display (client name) | 20px (1.25rem / `text-xl`) | 600 (semibold) | 1.2 |

Source: base.html `th { font-size: .875rem; font-weight: 600 }`, profile/index.html `text-lg font-semibold`, badge `font-size: .8rem; font-weight: 600`

Do NOT introduce a 5th size tier. The client name in the detail page header is visually distinct via size alone (20px vs 18px heading). Two weights only: 400 + 600.

## Color

Sourced from base.html inline CSS. All values are already deployed in production.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f5` | Page background (`body { background: #f5f5f5 }`) |
| Secondary (30%) | `#ffffff` | Cards (`.card`), modal dialog body, tab panel surface, table rows |
| Sidebar / Nav | `#1e1b4b` | Sidebar background — not a content surface, do not reuse as card color |
| Accent (10%) | `#4f46e5` (indigo-600) | See reserved list below |
| Destructive | `#dc2626` (red-600) | Delete client, delete contact, delete interaction entry only |

Accent (`#4f46e5`) reserved for:
1. "Добавить клиента" primary CTA button (`.btn-primary`)
2. Active sidebar child item (`#4338ca` — existing active state)
3. Tab underline indicator (active tab border-bottom)
4. "Прикрепить сайт" button on Sites tab
5. "Сохранить контакт" button inside create/edit client modal
6. Links in breadcrumb trail (`.breadcrumbs a { color: #4f46e5 }`)
7. Client badge link on site detail page (`"Клиент: …"` badge text color)

Secondary accent for informational states:
- Manager badge background: `#eef2ff` (indigo-50) with text `#4338ca` — matches existing `.badge-connected` pattern family
- Site count / task counter chip: `#f3f4f6` (gray-100) with text `#374151` (gray-700)

Destructive confirmation approach: inline `onclick="return confirm('…')"` — matches existing project pattern (profile/index.html delete key button). Do NOT introduce a custom modal for delete confirmation.

## Interaction Contracts

Defined from CONTEXT.md decisions. These are binding for the executor.

### Client List Page (`/crm/clients`)

- Table columns in order: Компания, Менеджер, Сайтов, Последнее взаимодействие
- Pagination: 25 rows per page (Claude's discretion — within the 20–50 range; 25 matches the projects list pattern)
- HTMX live-search: `hx-get="/crm/clients?q={value}" hx-trigger="keyup changed delay:300ms" hx-target="#clients-table-body" hx-swap="innerHTML"`
- Manager filter: `hx-get` on change, same target
- Date created filter: flatpickr range input (already loaded in base.html)
- "Добавить клиента" button opens modal: `hx-get="/crm/clients/new" hx-target="#modal-container" hx-swap="innerHTML"`

### Client Detail Page (`/crm/clients/{id}`)

- Header card: company name (Display 20px semibold) + legal name (Body 14px gray-500), INN/KPP row, phone + email row, manager badge, site count chip + open tasks chip
- Tab bar: four tabs with border-bottom active indicator (`border-b-2 border-indigo-600`)
- Tab switching: JavaScript `classList` toggle (no full page reload, no HTMX needed — tabs are rendered server-side and shown/hidden via JS `display` toggle)
- Default active tab: Sites

### Contacts Tab

- Table: Name, Phone, Email, Role, Telegram, — (actions)
- Inline edit trigger: click "Изменить" button on row → `hx-get="/crm/clients/{id}/contacts/{cid}/edit" hx-target="#contact-row-{cid}" hx-swap="outerHTML"`
- Inline edit save: `hx-put="/crm/clients/{id}/contacts/{cid}" hx-target="#contact-row-{cid}" hx-swap="outerHTML"`
- Inline edit cancel: `hx-get="/crm/clients/{id}/contacts/{cid}" hx-target="#contact-row-{cid}" hx-swap="outerHTML"` (re-fetches read-mode row)
- Add contact: small form below table (not modal) — `hx-post` appends new row to table

### Interactions Tab

- Inline compose form at top: textarea (3 rows) + "Добавить заметку" button
- Submit: `hx-post="/crm/clients/{id}/interactions" hx-target="#interactions-list" hx-swap="afterbegin"` (prepends newest)
- Each entry: date (gray-400 small), author name (indigo-600 link style), note text (body)
- Edit/delete own entry: small "Изменить" / "Удалить" text buttons visible on hover (opacity transition)
- Admin sees edit/delete on all entries

### Sites Tab

- Attached sites table: URL (link to site detail), crawl status badge, open tasks count
- Detach button: inline `hx-delete` with confirmation `hx-confirm="Открепить сайт…?"`
- Attach dropdown: `<select>` with `hx-get="/crm/clients/{id}/sites/search?q={value}"` for HTMX-powered search; shows only unattached sites

### Modal (Create / Edit Client)

- Fixed pattern from help/_modal.html: `position:fixed; top:0; left:0; right:0; bottom:0; background:rgba(0,0,0,.45); z-index:200`
- Modal card: `max-width:640px; width:92%; border-radius:10px; max-height:85vh; overflow-y:auto`
- Close: ESC key + click backdrop + × button (top-right)
- Form fields in order: Название компании*, Юридическое название, ИНН, КПП, Телефон, Email, Менеджер (dropdown), Заметки (textarea)
- Required fields marked with `*` in label
- Submit: `hx-post="/crm/clients"` or `hx-put="/crm/clients/{id}"`, closes modal on success, shows toast

### Client Badge on Site Pages

- Location: site detail page header, adjacent to site URL or status badges
- Markup: `<a href="/crm/clients/{id}" class="badge" style="background:#eef2ff;color:#4338ca;">Клиент: {name}</a>`
- If no client attached: show `<span class="badge badge-unknown">Без клиента</span>` + dropdown to attach (manager+ only)

## Soft Delete Decision (Claude's Discretion)

The CONTEXT.md deferred the client deletion strategy to Claude. Decision: **soft delete** (`is_deleted` boolean column, default False).

Rationale:
- Matches `Site.is_active` flag pattern in `app/models/site.py`
- Clients are referenced by FK in intake, proposals, and documents (Phases 21–23) — hard delete would cascade-break downstream records
- All list queries filter `Client.is_deleted == False` by default
- Admin can hard-purge via a future admin panel if needed

Implementation: `is_deleted: Mapped[bool] = mapped_column(Boolean, default=False, nullable=False)` on `Client` model.

## Area 4: Переход в UI

### Q4.1 — Guide mode

| Option | Description | Selected |
|--------|-------------|----------|
| Плашка сверху (Recommended) | URL действия + full-width плашка сверху с номером шага, кнопками «Отметить выполненным» / «Вернуться к playbook». sessionStorage. | ✓ |

| Plain link | Просто URL, без overlay. Возврат через breadcrumbs. | |

**User's choice:** Плашка сверху

### Q4.2 — Completion detection

| Option | Description | Selected |
|--------|-------------|----------|
| Вручную кнопкой (Recommended) | Пользователь сам нажимает «Сделано». Просто и предсказуемо. | |
| Автодетект | Система следит за триггерами для action_kind. Сложно, есть риск false positives. | |
| Гибрид | Ручная кнопка + подсказка-badge при обнаружении триггера. Двойной расход работы — избыточно для MVP. | ✓ |

**User's choice:** Гибрид
**Notes:** Пользователь пошёл против рекомендации. Claude предупредил о риске scope creep (6 action_kind × hint-trigger = ~60 LOC). В CONTEXT.md D-18 зафиксирован mapping триггеров и scope-watch: если планировщик оценит больше 1 плана — урезать до 2-3 самых частых action_kind (run_crawl, open_keywords) и остальные отложить в 999.8.1.

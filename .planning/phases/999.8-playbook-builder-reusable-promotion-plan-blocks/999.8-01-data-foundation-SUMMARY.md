# Phase 999.8 Plan 01: Data Foundation Summary

**One-liner:** 8-model SQLAlchemy schema + Alembic 0054 migration with 8-row BlockCategory seed + `| markdown` Jinja2 filter (python-markdown) + "Плейбуки" sidebar section + router skeleton with 3 placeholder routes — ready for Waves 2–3 to build CRUD, builder, and banner on top.

## What Was Built

### 1. Data Model — `app/models/playbook.py`

8 SQLAlchemy 2.0 models with typed `Mapped[]` columns, all using UUID PKs and timezone-aware timestamps:

| Model | Table | Purpose |
|---|---|---|
| `ExpertSource` | `expert_sources` | Optional author/expert attached to blocks |
| `BlockCategory` | `block_categories` | Block taxonomy (8 seeded categories) |
| `PlaybookBlock` | `playbook_blocks` | Reusable methodology block |
| `BlockMedia` | `block_media` | Videos/articles attached to a block (cascade delete) |
| `Playbook` | `playbooks` | Reusable template assembled from blocks |
| `PlaybookStep` | `playbook_steps` | Ordered block inside a template (uq on playbook_id+position) |
| `ProjectPlaybook` | `project_playbooks` | Applied copy (D-12: playbook_id nullable) |
| `ProjectPlaybookStep` | `project_playbook_steps` | Per-project, per-step status |

5 named enums: `actionkind`, `blockmediakind`, `projectplaybookstatus`, `projectplaybookstepstatus`, `playbookcategory`.

### 2. Alembic Migration 0054 — `alembic/versions/0054_add_playbook_tables.py`

- Creates all 8 tables in FK-safe order (parents first)
- Uses `sa.Enum(..., name="actionkind")` to avoid Alembic auto-named enum types
- JSONB columns with `server_default=sa.text("'[]'::jsonb")` for `prerequisites`
- D-12 FK rules:
  - `project_playbooks.playbook_id` → nullable, no cascade (template delete must not kill copies)
  - `project_playbooks.project_id` → CASCADE
  - `playbook_steps.playbook_id` → CASCADE
  - `block_media.block_id` → CASCADE
  - `project_playbook_steps.project_playbook_id` → CASCADE
- Unique constraint `uq_playbook_step_position` on `(playbook_id, position)`
- Inline `op.bulk_insert` seeds 8 BlockCategory rows in Russian (CD-05)
- Downgrade drops tables reverse order + drops all 5 enum types with `checkfirst=True`

**Round-trip verified:** `alembic upgrade → downgrade → upgrade` runs cleanly, seed data reappears.

### 3. Markdown Filter — `app/template_engine.py`

`_markdown_filter(text)` wraps `markdown.markdown(text, extensions=["nl2br", "tables", "fenced_code"])` in `markupsafe.Markup` and registers as `env.filters["markdown"]`.

Verified: `env.filters["markdown"]("**bold**") → Markup('<p><strong>bold</strong></p>')`.

### 4. Navigation Entry — `app/navigation.py` + `sidebar.html`

- New `"playbooks"` section inserted between `crm` and `settings` in `NAV_SECTIONS` with 3 children (Библиотека блоков, Шаблоны, Эксперты — last one admin-only)
- Added `book-open` heroicon case to the collapsible-section icon switch in `sidebar.html` (preserving all existing icons)
- Non-admin users see 2 children, admins see 3 (verified via `build_sidebar_sections`)

### 5. Router Skeleton — `app/routers/playbooks.py`

3 placeholder GET routes registered via `include_router(playbooks_router)` inline in `app/main.py`:

- `GET /ui/playbooks/blocks` → any authenticated user
- `GET /ui/playbooks` → any authenticated user
- `GET /ui/playbooks/experts` → admin only (`require_admin`)

Each returns an empty-state template extending `base.html`. Three placeholder templates created under `app/templates/playbooks/`.

`/ui/playbooks` also added to `_HELP_MODULE_MAP`.

### 6. Dependency — `requirements.txt`

Added `Markdown>=3.6,<4.0` in the Templating section. Installed version in container: **3.10.2**.

## Deviations from Plan

### Rule 3 — Blocking issue auto-fixed

**1. `[Rule 3 — Blocker] Swapped `get_async_session` → `get_db`**
- **Found during:** Task 3 (router skeleton)
- **Issue:** Plan snippet imported `from app.db.session import get_async_session`, but the codebase uses `from app.dependencies import get_db` (verified in `app/routers/templates.py`, `app/auth/dependencies.py`, and every other router)
- **Fix:** Used `get_db` from `app.dependencies` instead — matches established pattern for 40+ existing routers
- **Files modified:** `app/routers/playbooks.py`
- **Commit:** `015c034`

### Clarification — Sidebar icon case placement

The plan said "Find the existing `{% if section.icon == ... %}` / `{% elif section.icon == ... %}` chain". The codebase has **two** icon switches in `sidebar.html`: one for collapsible sections (lines 32–52) and one for top-level link sections (lines 92–96). Because "Плейбуки" has children (collapsible section in navigation.py), only the collapsible-section switch needed the `book-open` case. The top-level switch was not touched. This matches user-visible behavior correctly — verified via `build_sidebar_sections`.

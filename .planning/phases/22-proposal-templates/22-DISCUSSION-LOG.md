## Template Editor UX

### Тип редактора

| Option | Description | Selected |
|--------|-------------|----------|
| CodeMirror/Monaco (рекоменд.) | Полноценный код-редактор с подсветкой HTML/Jinja2, нумерацией строк, автозакрытием тегов. ~50KB JS | ✓ |
| Простой textarea | Обычное текстовое поле без подсветки. Минимальный JS | |
| WYSIWYG (TinyMCE/CKEditor) | Визуальный редактор HTML с тулбаром. Плохо совместим с Jinja2-тегами | |

**User's choice:** CodeMirror/Monaco
**Notes:** —

### Расположение превью

| Option | Description | Selected |
|--------|-------------|----------|
| Бок о бок (рекоменд.) | Редактор слева, рендер справа — обновляется по кнопке «Предпросмотр» | ✓ |
| Отдельная вкладка | Табы «Код» / «Предпросмотр» — переключение между режимами | |
| Под редактором | Редактор сверху, превью снизу — вертикальная раскладка | |

**User's choice:** Бок о бок
**Notes:** —

### Страница vs модалка

| Option | Description | Selected |
|--------|-------------|----------|
| Отдельная страница (рек.) | Полностраничный редактор — CodeMirror + превью бок о бок требуют места | ✓ |
| Модальное окно | Как создание клиента в Phase 20. Но модалка тесна для код-редактора + превью | |
| Модалка для мета + страница для кода | Создание в модалке (название, описание, категория), потом редирект на полностраничный редактор | |

**User's choice:** Отдельная страница
**Notes:** —

### Метаданные

| Option | Description | Selected |
|--------|-------------|----------|
| Тип + описание (рек.) | Тип документа (proposal/audit_report/brief) + описание. Тип нужен для Phase 23 | ✓ |
| Только название | Минимально — только name + body | |
| Тип + описание + теги | Тип, описание, плюс произвольные теги. Может быть избыточно | |

**User's choice:** Тип + описание
**Notes:** —

## Variable System

### Способ вставки переменных

| Option | Description | Selected |
|--------|-------------|----------|
| Панель справа (реком.) | Список всех переменных с описаниями под превью. Клик копирует/вставляет {{ variable }} | ✓ |
| Автокомплит в редакторе | Ввод {{ вызывает дропдаун с переменными. Нужна настройка CodeMirror autocomplete | |
| Только документация | Справочная страница с переменными. Админ пишет {{ ... }} вручную | |

**User's choice:** Панель справа
**Notes:** —

### Отображение неразрешённых

| Option | Description | Selected |
|--------|-------------|----------|
| Жёлтый фон (рекоменд.) | Обёрнуты в `<span class="unresolved">{{ var }}</span>` с жёлтым фоном | ✓ |
| Красный текст | Переменная показывается красным шрифтом с именем переменной | |
| Пусто с подсказкой | Пустое место + tooltip при наведении с именем переменной | |

**User's choice:** Жёлтый фон
**Notes:** —

## Preview Experience

### Выбор источника данных

| Option | Description | Selected |
|--------|-------------|----------|
| Дропдауны над превью (рек.) | Два selectа: Клиент → Сайт (фильтруется по клиенту). Кнопка «Предпросмотр» рендерит | ✓ |
| Модальный выбор | При нажатии «Предпросмотр» сначала модалка с выбором клиента/сайта, потом рендер | |

**User's choice:** Дропдауны над превью
**Notes:** —

### Метод рендеринга

| Option | Description | Selected |
|--------|-------------|----------|
| HTMX в iframe (рекоменд.) | HTMX POST на сервер, Jinja2 рендерит, HTML в iframe. Изоляция стилей | ✓ |
| HTMX в div | Сервер возвращает HTML прямо в div. Проще, но стили могут конфликтовать | |

**User's choice:** HTMX в iframe
**Notes:** —

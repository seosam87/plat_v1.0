## Layout Contracts

### Template List Page (`/ui/templates`)

**Focal point:** The template card grid is the primary focal element. On first visit with no templates, the empty-state CTA ("Создать шаблон") is the sole focal point, centered in the content area.

- Full-width `.main-content` area, single `.card` container
- Page header row: `h2` "Шаблоны КП" left-aligned + "Создать шаблон" button right-aligned (admin only — hidden for non-admins)
- Template grid: `display: grid; grid-template-columns: repeat(auto-fill, minmax(300px, 1fr)); gap: 24px;` — matches `grid-cols-[repeat(auto-fit,minmax(...,1fr))]` pattern already used in `gap/index.html`
- Each card: `.card` (white bg, 8px border-radius, 1px shadow) containing: template type badge (top-right), name (h3, 18px semibold), description (14px regular, 2-line clamp), metadata row (created date + author, 12px gray), action buttons row (Edit, Clone, Delete — admin only for Edit/Delete/Clone)
- Empty state: centered in card grid area, icon + heading + subtext + CTA button

**Source:** CONTEXT.md D-09, D-10 + existing `.card` class from base.html.

### Template Edit Page (`/ui/templates/{id}/edit`)

**Focal point:** The CodeMirror editor (left column) is the primary focal element — it occupies the largest flex proportion and is where the user performs the core action of authoring the template.

Three-column layout inside `.main-content`:

```
| CodeMirror editor (flex: 2) | iframe preview (flex: 2) | Variable panel (flex: 1, min 240px) |
```

- Outer wrapper: `display: flex; gap: 16px; height: calc(100vh - 140px);` — fills viewport minus breadcrumb + header
- **Left column (CodeMirror):** `.card` wrapper, no inner padding on editor area; `height: 100%; overflow: hidden;` — CodeMirror fills the card
  - Above editor: metadata form row (name input, type select, description input) in a 24px-padded header within the card
  - Below editor: action bar — "Сохранить шаблон" (primary) + "Сохранить как черновик" (secondary) + "Отмена" (ghost)
- **Center column (iframe preview):** `.card` wrapper; above iframe: two dependent selects — "Клиент" then "Сайт" (HTMX-driven dependent select, same pattern as CRM); "Обновить превью" button; below selects: `<iframe>` fills remaining height with `border: none; width: 100%;`
- **Right column (variable panel):** `.card` wrapper, `overflow-y: auto; height: 100%;`; grouped list of ~15 variables (see Variable Panel contract below)

**Source:** CONTEXT.md D-02, D-03, D-06, D-13, D-14, Specific Ideas (три колонки).

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, inline tight spacing |
| sm | 8px | Button internal padding (vertical), compact form elements |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding inside cards, grid gap |
| xl | 32px | Between major sections |
| 2xl | 48px | Major section breaks (not used in this phase's dense layout) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: CodeMirror editor container uses `0px` padding inside card (editor fills flush to card edge below header); iframe preview uses `0px` margin/border.

Touch targets (action buttons): minimum 36px height — consistent with `.btn` `padding: .4rem .9rem` already in base.html.

**Source:** base.html existing `.card { padding: 1.5rem }` = 24px = lg token; existing `.btn { padding: .4rem .9rem }` ≈ 7px vertical — kept as-is per project convention.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (0.875rem) | 400 (regular) | 1.5 | Card descriptions, variable panel text, form labels |
| Label | 12px (0.75rem) | 600 (semibold) | 1.4 | Table column headers, metadata date/author, badge text, variable category headers, variable name monospace text |
| Heading | 18px (1.125rem) | 600 (semibold) | 1.2 | Page title `h2`, template card name `h3` |
| Display | 20px (1.25rem) | 600 (semibold) | 1.2 | Page-level `h1` on edit page ("Редактировать шаблон") |

Max 2 weights across all type: **400 (regular)** and **600 (semibold)**.

**Source:** Derived from existing CRM index.html patterns — `font-size:1.125rem;font-weight:600` for headings; `font-size:0.875rem` for body; `font-size:.875rem;font-weight:600;text-transform:uppercase` for table headers.

### Variable Panel Item

Each variable item: `cursor: pointer; padding: 8px 16px; border-radius: 4px; hover: background #f3f4f6`

- Click action: inserts `{{ variable_name }}` at CodeMirror cursor position (JS bridge via CodeMirror dispatch)

### Unresolved Variable Span (rendered in iframe)

```html
<span class="unresolved-var" style="background:#fef3c7;color:#92400e;padding:0 4px;border-radius:2px;font-family:monospace;font-size:0.85em;">{{ unresolved_var }}</span>
```

Applied server-side by the Jinja2 SandboxedEnvironment when a variable has no resolved value — replaces the original `{{ var }}` token with this span.

### Delete Confirmation

Inline confirmation pattern (no modal): on clicking "Удалить", the button text changes to "Подтвердить удаление" with red background + a "Отмена" link appears beside it. Implemented via HTMX `hx-confirm` attribute: `hx-confirm="Удалить шаблон «{name}»? Это действие нельзя отменить."` — browser native `confirm()` dialog consistent with existing delete patterns in the project.

**Source:** CONTEXT.md D-11, CONTEXT.md code_context (soft delete pattern in Client model — but templates use hard delete per Claude's discretion since no document history is referenced before Phase 23).

## Interaction Contracts

### HTMX Interactions

| Trigger | Target | Method | Swap |
|---------|--------|--------|------|
| "Обновить превью" button | `#preview-frame` (custom JS sets iframe src) | POST `/ui/templates/{id}/preview` | iframe srcdoc update via JS — NOT direct hx-swap (iframe isolation requirement from CONTEXT.md D-14) |
| Клиент select change | `#site-select` | GET `/ui/templates/sites?client_id=X` | `outerHTML` — dependent select pattern from CRM |
| "Клонировать шаблон" button | `#toast-container` + redirect | POST `/ui/templates/{id}/clone` | HX-Redirect header → `/ui/templates/{clone_id}/edit` |
| "Удалить" button | `#template-card-{id}` | DELETE `/ui/templates/{id}` | `outerHTML` (removes card from grid) |
| "Сохранить шаблон" button (edit page) | `#save-status` | POST/PUT `/ui/templates/{id}` | `innerHTML` — shows save confirmation inline |

Preview POST body: `{ body: codemirror_value, client_id: selected, site_id: selected }`. Server returns rendered HTML. JS sets `iframe.srcdoc = response` for style isolation (CONTEXT.md D-14).

### CodeMirror Integration

- Initialize on `DOMContentLoaded` with `EditorView` from `@codemirror/view`
- Use `html()` language extension from `@codemirror/lang-html`
- Wrap in a `div#codemirror-mount` inside the left card column
- Variable panel click handler: `view.dispatch({ changes: { from: view.state.selection.main.head, insert: '{{ variable_name }}' } })`
- On form submit: copy `view.state.doc.toString()` into a hidden `<input name="body">` before POST

### Dependent Select (Client → Site)

Pattern: `hx-get="/ui/templates/sites?client_id={{value}}" hx-trigger="change" hx-target="#site-select" hx-swap="outerHTML"` — identical to CRM pattern used in `app/routers/crm.py`.

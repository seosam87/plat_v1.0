### Human Verification Required

#### 1. Visual rendering of diagnostic list in browser

**Test:** Trigger a position check on a site with no XMLProxy credentials configured. Wait for the task to complete.
**Expected:** Status area shows a red `✗` header with "Done. Positions: 0, alerts: 0" and a bulleted list below containing a red "✗ XMLProxy credentials not configured..." message. Page does NOT auto-reload.
**Why human:** Tailwind CSS utility classes (`text-red-600`, `text-amber-600`) must be present in the compiled stylesheet. If Tailwind is configured with purge/JIT and the new class names are not in the source scan, they will be stripped and the colors will not render even though the classes are in the HTML.

#### 2. Amber warning path end-to-end

**Test:** Trigger a position check on a site that has only Google keywords and DataForSEO credentials are absent.
**Expected:** Status header shows amber `⚠ Done. Positions: 0, alerts: 0` with a warning diagnostic item "N Google keyword(s) skipped — DataForSEO not configured...".
**Why human:** Cannot verify Tailwind purge behavior programmatically.

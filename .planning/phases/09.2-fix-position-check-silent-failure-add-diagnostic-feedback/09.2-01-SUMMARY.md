## What Was Built

### Task 1: Structured Diagnostics in Celery Task

Modified `app/tasks/position_tasks.py` to always include a `diagnostics` key in the return dict.

Key changes:
- `site_active_guard` early-return path augmented: `skip["diagnostics"] = [{"level": "warning", ...}]`
- `diagnostics = []` initialized after guard check
- Keyword split counts logged as `info` diagnostics
- `_check_via_xmlproxy` accepts optional `diagnostics` param and appends errors for: no creds, zero balance, low balance, site not found, error -34 (invalid creds), error -32 (no funds)
- `_check_via_dataforseo` accepts optional `diagnostics` param and appends errors for: site not found, batch request failure
- Google keywords skipped diagnostic: `warning` level with actionable message pointing to Settings
- Final return includes `diagnostics` key with 15 total `diagnostics.append` call sites

All failure modes now produce a diagnostic message:
- No credentials -> `error`: "XMLProxy credentials not configured. Go to Settings > Data Sources..."
- Zero balance -> `error`: "XMLProxy balance is 0 RUB. Top up your XMLProxy account."
- Low balance -> `warning`: "XMLProxy balance low: X RUB (threshold: Y)"
- No keywords -> `warning`: "No keywords found for this site. Import keywords first."
- Google skipped -> `warning`: "N Google keyword(s) skipped — DataForSEO not configured..."
- Site not found -> `error`: "Site not found in database"
- API -34 error -> `error`: "XMLProxy credentials are invalid. Check Settings > Data Sources."
- API -32 error -> `error`: "XMLProxy insufficient funds (error -32). Top up account."

### Task 2: Frontend Diagnostic Display

Modified `pollTaskStatus` in `app/templates/positions/index.html` SUCCESS branch.

Before: Single `text-emerald-600 ✓ Done! Positions: X, alerts: Y` line always shown green.

After:
- Header color determined by diagnostics: `hasError` → red, `hasWarning && written === 0` → amber, else → green
- Diagnostic list rendered below header with level-coded items
- Auto-reload only fires when `written > 0` (prevents diagnostic messages disappearing)

Color mapping:
- Error header: `text-red-600` with `✗` icon
- Warning header (0 results): `text-amber-600` with `⚠` icon  
- Success header: `text-emerald-600` with `✓` icon
- Error diagnostic items: `text-red-600`
- Warning diagnostic items: `text-amber-600`
- Info diagnostic items: `text-gray-500`

### Task 3: Unit Tests

Created `tests/test_position_tasks.py` with 5 tests covering all major diagnostics paths:

1. `test_site_guard_skip_includes_diagnostics` — guard early-return includes diagnostics key with warning
2. `test_no_keywords_returns_warning_diagnostic` — empty keyword list produces warning with actionable message
3. `test_google_skipped_returns_warning_diagnostic` — Google keywords without DataForSEO config produces warning
4. `test_success_path_includes_diagnostics_key` — successful check always returns diagnostics list
5. `test_xmlproxy_missing_creds_produces_error_diagnostic` — no XMLProxy creds produces error-level diagnostic

All 5 tests pass without database, Redis, or external API access.

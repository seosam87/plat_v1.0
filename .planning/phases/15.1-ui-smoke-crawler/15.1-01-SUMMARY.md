# Phase 15.1 Plan 01: Smoke Seed Fixture Summary

Deterministic session-scoped smoke_seed pytest fixture providing 17 UUID-stable
entities across CORE + EXTENDED waves, bound to an outer connection + outer
transaction with SAVEPOINT isolation so downstream per-request sessions can
join the seed without pollution.

## What was built

- `tests/fixtures/smoke_seed.py` — `SMOKE_IDS` dict (17 deterministic UUIDs),
  `SeedHandle` dataclass, session-scoped `smoke_seed` async fixture.
- CORE seed (`_seed_core`): SiteGroup, Site, admin User (bcrypt-hashed),
  KeywordGroup, 10 Keywords (engine mix), 5 KeywordLatestPosition, 2 GapKeyword,
  SuggestJob, CrawlJob, ClientReport, 4 ServiceCredential rows, AuditCheckDefinition,
  AuditResult. Deterministic IDs wired via SMOKE_IDS.
- EXTENDED seed (`_seed_extended`): Project, SeoTask, AnalysisSession,
  KeywordCluster, ContentBrief, Competitor.
- `tests/fixtures/test_smoke_seed_constants.py` — 5 constants tests.
- `tests/fixtures/test_smoke_seed_fixture.py` — 16 async fixture tests, all
  marked `@pytest.mark.asyncio(scope="session")` so they share the fixture's
  event loop.
- `tests/fixtures/conftest.py` — placeholder (no event_loop override needed).

## Deviations from Plan

### [Rule 3 - Blocking] Test database did not exist
- **Found during:** Task 2 first verification run
- **Issue:** `asyncpg.exceptions.InvalidCatalogNameError: database "seo_platform_test" does not exist`
- **Fix:** `CREATE DATABASE seo_platform_test OWNER seo_user;` via `docker exec seo-platform-postgres-1 psql`.
- **Files modified:** None (infrastructure fix).

### [Rule 3 - Blocking] Session-scoped async fixture event loop mismatch
- **Found during:** Task 2 verification
- **Issue:** pytest-asyncio 0.23 defaults to function-scoped event loops; the
  session-scoped `smoke_seed` fixture's AsyncConnection ended up on a different
  loop than the test functions, yielding
  `asyncpg InterfaceError: cannot perform operation: another operation is in progress`.
- **Fix:** Used `pytestmark = pytest.mark.asyncio(scope="session")` on the
  fixture-test module (supported in pytest-asyncio 0.23) so all consumer tests
  share the same session-scoped event loop. Removed the deprecated
  `event_loop` override attempt.
- **Files modified:** tests/fixtures/test_smoke_seed_fixture.py, tests/fixtures/conftest.py
- **Commit:** be42ba4

### [Rule 2 - Model name corrections] Model substitutions vs plan wording
Plan referenced `Task`, `AnalyticsSession`, `Brief` generically. The actual
codebase has:
- `SeoTask` (app/models/task.py) — seeded as SeoTask
- `AnalysisSession` (app/models/analytics.py) — seeded as AnalysisSession
- `ContentBrief` (app/models/analytics.py) — seeded as ContentBrief (FK to AnalysisSession)
- `KeywordCluster.site_id` (not project_id) — seeded with site_id

All substitutions are documented inline in `_seed_extended` docstring. No
Plan 02 `SMOKE_SKIP` entries are required — all 7 EXTENDED entities are
successfully seeded with the corrected model names.

## Plan 02 notes

No routes need to land in `SMOKE_SKIP` because every targeted entity was
seeded successfully. Plan 02 can import from `tests.fixtures.smoke_seed`:
- `SMOKE_IDS` for the `PARAM_MAP`
- `smoke_seed` fixture as the session-scoped seed provider
- `SeedHandle.connection` to bind per-request sessions with
  `join_transaction_mode="create_savepoint"`.

Downstream tests that consume `smoke_seed` must either be marked with
`@pytest.mark.asyncio(scope="session")` or be placed under a conftest that
sets a session-scoped event loop, per the workaround above.

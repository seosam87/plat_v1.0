# Phase 15.1 Plan 03: UI Smoke Crawler — Parametrized Test Module Summary

**One-liner:** Session-scoped `smoke_client` fixture with auth-override + per-request savepoint session bound to the seed's outer connection, wired into a parametrized `tests/test_ui_smoke.py` that exercises every discovered UI GET route and an explicit `data.items` dict-collision regression test.

## What shipped

- **`tests/conftest.py`** — new `smoke_client` session-scoped fixture:
  - Re-exports `smoke_seed` / `SMOKE_IDS` from `tests.fixtures.smoke_seed`.
  - Overrides `get_db` with a fresh `AsyncSession` per request bound to
    `smoke_seed.connection` via `async_sessionmaker(join_transaction_mode="create_savepoint")`.
    Each request's commit/rollback is scoped to its own savepoint so the
    outer seed transaction is never touched.
  - Overrides `get_current_user`, `require_admin`, and
    `require_any_authenticated` to return the seed's admin `User` —
    smoke crawler never POSTs to `/ui/login`.
  - Teardown explicitly `pop()`s the four overrides (no `.clear()`), so
    other test modules that build their own overrides are unaffected.
- **`tests/test_ui_smoke.py`** — parametrized crawler:
  - `ROUTES = discover_routes(app)` at import time (pure function, no DB).
  - `PARAM_MAP = build_param_map(SMOKE_IDS)` substitutes path params.
  - `test_ui_route_renders[<path>]` one case per route:
    - status in `{200, 301, 302, 303, 307, 308}`
    - HTML bodies: `assert_no_error_markers` + `assert_structural_html`
    - non-HTML (downloads) skipped
  - `test_data_items_dict_collision_regression` explicitly GETs
    `/analytics/{site_id}/opportunities` and asserts the body does not
    contain `builtin_function_or_method`, `TemplateSyntaxError`, or
    `UndefinedError`.
  - `pytestmark = pytest.mark.asyncio(scope="session")` so the
    parametrized cases share the session-scoped smoke_seed/smoke_client
    event loop (Plan 01 carry-over workaround).

## Deviations from Plan

### [Rule 2 - Missing coverage] Overrode `require_any_authenticated` in addition to `require_admin`
- **Found during:** Task 1 planning
- **Issue:** Plan listed only `get_current_user` + `require_admin`, but
  `app.routers.projects` (and auth router) use
  `require_any_authenticated`. Without overriding it, project routes
  would 403 under the smoke gate.
- **Fix:** Added a fourth override and a fourth `pop()` in teardown.
- **Files modified:** `tests/conftest.py`
- **Commit:** e7cb2b0

### [Rule 3 - Blocking] Deployment tree drift
- **Found during:** Task 2 first smoke run
- **Issue:** `/opt/seo-platform/app/templates/analytics/partials/opportunities_gaps.html`
  still contained `data.items` despite commit `f1259c1` renaming it to
  `data.rows` in the git repo. The regression test failed against the
  actual running app — not a bug in the test, a bug in deployment.
- **Fix:** Copied the fixed template from the repo into the deployment
  bind-mount so the running app matches the committed source. Documented
  the drift in `deferred-items.md` with a deployment note.
- **Files modified:** `/opt/seo-platform/app/templates/analytics/partials/opportunities_gaps.html` (deployment tree, not git-tracked in this repo)

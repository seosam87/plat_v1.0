# Phase 15.1 Plan 04: Slim smoke_test.py Summary

**One-liner:** Converted the 396-line `tests/smoke_test.py` into a 148-line thin CLI wrapper that delegates all route discovery and param resolution to `tests/_smoke_helpers.py`, eliminating duplication between the standalone CLI and the pytest suite.

## What shipped

- `tests/smoke_test.py`: 396 → 148 lines (63% reduction). Removed `discover_routes_from_nav`, `discover_routes_from_main`, `merge_routes`, `_normalize_template`, `resolve_url`, `SKIP_ROUTES`, `SKIP_PATTERNS`, `FAKE_SITE_ID`, `FAKE_PROJECT_ID`. These are now single-sourced in `tests/_smoke_helpers.py`.
- Preserved: `authenticate()` (real login for live-server runs), `get_first_site_id()` (best-effort DB lookup for CLI PARAM_MAP seeding), `print_report()` (colored terminal output), `run_smoke_test()` (top-level runner), argparse CLI with `--url` / `--email` / `--password`, `if __name__ == "__main__"` entry point.
- New behavior: CLI now runs the same `JINJA_ERROR_MARKERS` body scan as the pytest suite, so ad-hoc runs catch today's `builtin_function_or_method` bug class without needing pytest.

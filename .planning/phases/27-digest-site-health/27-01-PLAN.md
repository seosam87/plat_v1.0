Create `app/services/mobile_digest_service.py` as a standalone async service. Do NOT extend or import from `morning_digest_service.py` (it uses sync Session, would block the event loop).

The service must contain these functions using `AsyncSession`:

1. `async def get_top_position_changes(db: AsyncSession, limit: int = 5) -> list`:
   - Query `KeywordPosition` joined with `Site` (for `site_name`) and `Keyword` (for `phrase`).
   - CRITICAL: Always filter `KeywordPosition.checked_at >= cutoff` (7 days ago) — table is range-partitioned on `checked_at`, omitting this causes full partition scan.
   - Filter `KeywordPosition.delta.isnot(None)`.
   - Order by `desc(func.abs(KeywordPosition.delta))`, limit to `limit`.
   - Return list of dicts: `{"keyword": phrase, "site_name": site.name, "site_id": str(site.id), "delta": delta, "checked_at": checked_at}`.
   - Per D-01: This is block 1 (first in order).

2. `async def get_recent_crawl_errors(db: AsyncSession, limit: int = 5) -> list`:
   - Query `Page` joined with `Site` where `Page.http_status == 404` OR `Page.has_noindex.is_(True)`.
   - Order by `desc(Page.crawled_at)`, limit.
   - Return list of dicts: `{"url": page.url, "http_status": page.http_status, "has_noindex": page.has_noindex, "site_name": site.name, "site_id": str(site.id), "crawl_job_id": str(page.crawl_job_id)}`.

3. `async def get_recent_alerts(db: AsyncSession, limit: int = 5) -> list`:
   - Query `ChangeAlert` joined with `Site`.
   - Order by `desc(ChangeAlert.created_at)` (note: `ChangeAlert` has no explicit `created_at` — use `sent_at` as proxy, filter `sent_at.isnot(None)`, order by `desc(ChangeAlert.sent_at)`).
   - Actually check the model: ChangeAlert has `sent_at` but no `created_at`. Use `sent_at` for ordering. Include alerts where `sent_at` is NOT null (already dispatched).
   - Return list of dicts: `{"change_type": alert.change_type.value, "severity": alert.severity.value, "page_url": alert.page_url, "details": alert.details, "site_name": site.name, "site_id": str(site.id)}`.

4. `async def get_overdue_tasks(db: AsyncSession, limit: int = 5) -> list`:
   - Query `SeoTask` joined with `Site` where `SeoTask.due_date < today` and `SeoTask.status != TaskStatus.resolved`.
   - Order by `SeoTask.due_date` (oldest overdue first), limit.
   - Calculate `overdue_days = (today - task.due_date).days`.
   - Return list of dicts: `{"title": task.title, "due_date": task.due_date, "overdue_days": overdue_days, "site_name": site.name, "site_id": str(site.id)}`.

5. `async def build_mobile_digest(db: AsyncSession) -> dict`:
   - Calls all 4 functions above (per D-02: limit=5 for each).
   - Returns `{"position_changes": [...], "crawl_errors": [...], "alerts": [...], "overdue_tasks": [...], "today": formatted_date}`.
   - Format `today` as Russian short date: e.g., "10 апр" using a simple month map (not locale-dependent).

6. `async def get_mobile_site_health(db: AsyncSession, site_id: uuid.UUID) -> dict`:
   - This function will be used by Plan 02 (health card). Include it here so both plans share one service file.
   - 6 metrics per D-05:
     (1) `site_status`: Latest `CrawlJob` for this site — if `status == "done"`, site is reachable. Query: `select(CrawlJob).where(CrawlJob.site_id == site_id).order_by(desc(CrawlJob.finished_at)).limit(1)`.
     (2) `crawl_error_count`: Count of `Page` with `http_status == 404` OR `has_noindex == True` from the latest `CrawlJob`.
     (3) `last_crawl`: Latest `CrawlJob` — return `{"status": job.status.value, "finished_at": job.finished_at, "pages_crawled": job.pages_crawled}` or None.
     (4) `position_changes_count`: Count of `KeywordPosition` with `abs(delta) > 10` in last 7 days for this site. CRITICAL: filter `checked_at >= cutoff`.
     (5) `overdue_task_count`: Count of `SeoTask` where `site_id == site_id`, `due_date < today`, `status != resolved`.
     (6) `indexation_status`: Check `Site.metrika_token` — if not null return "connected", else "no_data".
   - For each metric, compute a `status` color: "green", "yellow", "red", or "grey" per RESEARCH.md thresholds:
     - Green: site reachable (done), 0 errors, 0 overdue, no sharp drops
     - Yellow: 1-5 errors, crawl >7 days old, 1-3 overdue
     - Red: site unreachable (failed), >5 errors, >3 overdue, position_changes_count > 0 (sharp drops)
     - Grey: no data
   - Return dict with all 6 metrics + their status colors.

Use `loguru` for logging. Add `from loguru import logger` and log each query with `logger.debug(...)`.

All imports at the top of the file. Type hints on all function signatures.

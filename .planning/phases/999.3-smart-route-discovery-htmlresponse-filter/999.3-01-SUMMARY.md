## What Changed

- Added `_is_html_route(route)` helper in `tests/_smoke_helpers.py` implementing the D-01 three-tier classification:
  1. Explicit `response_class` (skipping `DefaultPlaceholder` sentinel) — `HTMLResponse` subclass include, anything else exclude.

## Gap-closure addendum (2026-04-07, commit 4c200b3)

The verifier (`999.3-VERIFICATION.md`) found 3 gaps stemming from a single root cause and 1 reproducibility gap. All addressed in a single atomic commit.

### Gap 1 — Tier 2 dead under PEP 563 (root cause)

`_is_html_route` tier 2 read `endpoint.__annotations__["return"]` directly. Because `tests/_smoke_helpers.py` (and the entire codebase's router modules) use `from __future__ import annotations`, the value at runtime is a STRING like `'list[dict]'`. `get_origin('list[dict]')` is `None` and `isinstance('list[dict]', type)` is `False`, so tier 2 silently fell through to tier 3 (conservative include) for the codebase's dominant annotation style — defeating the entire phase goal.

**Fix:** resolve via `typing.get_type_hints(endpoint)`, wrapped in `try/except (NameError, AttributeError, TypeError)`. On resolution failure, `ann` stays `None` and the function falls through to tier 3 rather than crashing. The `except` clause is deliberately narrow per scope guard — broad `except:` would mask real bugs.

### Gap 2 — `/analytics/sessions/{session_id}/export` had no return annotation

This endpoint returns a CSV via `PlainTextResponse` and had neither `response_class=` nor a return annotation, so even after the Gap 1 fix it would fall through to tier 3 (include).

**Fix (preferred path per gaps_to_fix):** added `response_class=PlainTextResponse` to the `@router.get(...)` decorator in `app/routers/analytics.py:457`. This is a one-line, isolated change that fixes tier 1 classification and documents the response type for OpenAPI consumers. No SMOKE_SKIP fallback needed.

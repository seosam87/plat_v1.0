## Design System

| Property | Value |
|----------|-------|
| Tool | none — Jinja2 + Tailwind CDN (carries forward Phase 29 pattern) |
| Preset | not applicable |
| Component library | none — inline Tailwind classes, existing mobile partials |
| Icon library | Heroicons (inline SVG, stroke-width 1.5, 24x24) — source: Phase 29 UI-SPEC |

## Spacing Scale

Source: Phase 29 UI-SPEC (carried forward unchanged).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (`gap-1`), badge inner padding, count-badge padding |
| sm | 8px | Between error rows (`gap-2`), button inline padding, toggle button gap |
| md | 16px | Page content padding (`p-4`), form field margin-bottom (`mb-4`) |
| lg | 24px | Section heading margin-bottom (`mb-6`) |
| xl | 32px | Major vertical rhythm between sections (Индексация / Краулинг / Санкции) |
| 2xl | 48px | Not used in mobile context |
| 3xl | 64px | Bottom nav height reservation (`padding-bottom: 64px`) |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (`min-h-[44px]`) — source: Phase 29 UI-SPEC
- Bottom nav: 64px fixed height — source: base_mobile.html
- Toast bottom offset: 80px from bottom — source: base_mobile.html `#toast-container`
- Inline brief form expand area: `p-4 space-y-3` — matches task_form.html pattern
- Mode toggle buttons (Задача / ТЗ копирайтеру): `min-h-[44px]` each, full-width pair

## Typography

Source: Phase 29 UI-SPEC (carried forward unchanged). Verified against positions.html, tools/list.html.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / error list item text | 14px (`text-sm`) | 400 regular | 1.5 (default) |
| Label / secondary meta / error subtype | 12px (`text-xs`) | 400 regular | 1.5 |
| Page heading | 20px (`text-xl`) | 600 semibold (`font-semibold`) | 1.2 |
| Section heading / card title | 14px (`text-sm`) | 600 semibold (`font-semibold`) | 1.4 |

Rules:
- Maximum 2 font weights: 400 (regular) and 600 (semibold).
- No font-size outside this declared set.
- Error title in row: `text-sm` (regular 400). URL below title: `text-xs text-gray-500 truncate`.
- Count badge inside section heading: `text-xs font-semibold` on colored badge.
- Truncation: `truncate` on single-line URL strings in error rows.

## Color

Source: Phase 29 UI-SPEC (carried forward unchanged). New semantic additions for error types.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f5` | Page background (`body`) |
| Secondary (30%) | `#ffffff` | Cards, form surfaces, expanded brief form, toggle inactive state |
| Nav surface | `#1e1b4b` | Bottom navigation bar — source: base_mobile.html |
| Accent (10%) | `#4f46e5` (indigo-600) | Specific elements listed below |
| Accent light | `#818cf8` (indigo-400) | Active bottom-nav tab label + icon — source: base_mobile.html |
| Destructive | `#991b1b` (red-800) | Error toast text; error state messages |
| Destructive surface | `#fef2f2` (red-50) | Error toast background; sanction badge background |
| Success surface | `#ecfdf5` (green-50) | Success toast background; "✓ ТЗ создано" inline confirmation |
| Success text | `#065f46` (green-900) | Success toast text; "✓ ТЗ создано" text |
| Info surface | `#eff6ff` (blue-50) | Info toast background |
| Info text | `#1e40af` (blue-800) | Info toast text |
| Warning surface | `#fffbeb` (amber-50) | Sanction type badge |
| Warning text | `#92400e` (amber-800) | Sanction type badge text |

**Semantic colors for error type sections:**
- Индексация (indexing): count badge = `bg-blue-100 text-blue-700`
- Краулинг (crawl): count badge = `bg-orange-100 text-orange-700`
- Санкции (sanction): count badge = `bg-red-100 text-red-700`

**Priority radio colors (inline brief form and tasks form):**
- P1: label `text-red-600`, ring on selected `ring-red-500`
- P2: label `text-orange-600`, ring on selected `ring-orange-500`
- P3: label `text-gray-600`, ring on selected `ring-gray-400` (default selected)
- P4: label `text-blue-600`, ring on selected `ring-blue-500`

**Accent `#4f46e5` reserved for exactly these elements:**
1. Primary action button background: "Составить ТЗ" (active state), "Создать", "Создать и отправить" CTA buttons
2. Active mode toggle button background (Задача / ТЗ копирайтеру — active tab)
3. Progress bar fill during sync (`background:#4f46e5`)
4. Active tab underline / active bottom-nav indicator
5. Inline text links: "открыть" in "✓ ТЗ создано — открыть"
6. "Показать все (N)" text link per section

**Accent is NOT used for:**
- "Обновить" (manual refresh) button — use secondary outline `border-gray-300 text-gray-700 bg-white`
- "Отмена" / "Не создавать" buttons — use `bg-gray-100 text-gray-600`
- Count badges — use semantic color per error type (above)
- Inactive mode toggle — use `bg-gray-100 text-gray-500`

## Screen Inventory

All screens extend `base_mobile.html`. All include bottom nav with `active_tab` variable.

### 1. `/m/errors` — Errors Page

**Layout:** Single scrolling page. Top: site dropdown. Below: sync button + last-synced timestamp. Below: 3 error sections (Индексация / Краулинг / Санкции).

**Sections (top to bottom):**
1. **Page heading:** "Ошибки сайта" — `text-xl font-semibold mb-4`
2. **Site dropdown:** `<select>` full-width `min-h-[44px]`, shows all `Site.domain` values for user. Change → `hx-get="/m/errors?site_id={id}" hx-target="#errors-content" hx-swap="innerHTML"`. Cookie `m_errors_site_id` persists selection.
3. **Sync row:** "Обновить" secondary outline button (right-aligned) + "Последнее обновление: {datetime}" label (`text-xs text-gray-400`). "Обновить" POST → `hx-post="/m/errors/sync" hx-include="[name=site_id]" hx-target="#sync-progress" hx-swap="innerHTML"`. Shows inline progress polling slot `#sync-progress` (same `every 3s` pattern as Phase 29).
4. **`#errors-content`:** Contains the 3 error sections, HTMX swap target on site change.

**Each error section:**
- Section heading row: `flex items-center justify-between mb-3`
  - Left: section title (`text-sm font-semibold`) + count badge (`text-xs font-semibold px-2 py-0.5 rounded-full ml-2`)
  - Right: icon (Heroicons 20x20, stroke 1.5) — `document-magnifying-glass` for indexing, `arrow-path` for crawl, `shield-exclamation` for sanction
- Error card list: `bg-white rounded-lg divide-y divide-gray-100`
- Each error row (`.error-row`): `p-3 min-h-[44px]`
  - Left block (grows): error title `text-sm` (one line, truncate) + URL below `text-xs text-gray-400 truncate` (omitted for sanctions)
  - Right: "Составить ТЗ" button — `text-xs font-semibold text-indigo-600 underline min-h-[44px] px-2 flex items-center`
- "Показать все (N)" link below card list: `text-sm text-indigo-600 underline mt-2 block text-right` — HTMX expand into same section

**Host-not-linked state** (when Site.domain not found in Webmaster hosts):
- Replace card list with: `bg-white rounded-lg p-4 text-sm text-gray-500` — "Хост не привязан к Yandex Webmaster. Верифицируйте сайт и нажмите Обновить."
- "Составить ТЗ" buttons hidden.

**Empty state per section** (0 open errors):
- `bg-white rounded-lg p-4 text-sm text-gray-400 text-center` — "Ошибок не обнаружено"

**Empty state — no sites at all:**
- Replace `#errors-content` with: "Нет сайтов. Добавьте сайт в полной версии платформы."

**Sync progress slot (`#sync-progress`):**
- Polling partial identical to `tool_progress.html` pattern (`hx-trigger="every 3s"`)
- Running: spinner + "Синхронизируем ошибки..."
- Done: green check + "Обновлено" → `hx-trigger="load" hx-get="/m/errors/content?site_id={id}" hx-target="#errors-content"` (reload sections)
- Error: red warning + "Ошибка синхронизации. Попробуйте ещё раз."

### 2. Inline Brief Form (partial: `errors/partials/brief_form.html`)

Triggered by "Составить ТЗ" tap on an error row. Replaces `.error-row` via `hx-get="/m/errors/{error_id}/brief/form" hx-target="closest .error-row" hx-swap="outerHTML"`.

**Layout (replaces entire row):** `bg-white rounded-lg p-4 space-y-3 border-l-2 border-indigo-600`

**Fields (top to bottom):**
1. **Error title display** (read-only): `text-sm font-semibold text-gray-700 mb-2` — shows `error.title`
2. **Textarea** — "Описание работ" label `text-xs text-gray-500 mb-1`, textarea `w-full min-h-[100px] border border-gray-300 rounded-lg px-3 py-2 text-sm`, placeholder "Опишите, что нужно исправить..."
3. **Priority radio** — horizontal row `flex gap-3`, label "Приоритет" `text-xs text-gray-500 mb-1`
   - 4 inline radio buttons: P1 (red), P2 (orange), P3 gray default, P4 blue
   - Each: `<label>` wrapping hidden `<input type="radio">` + styled pill `px-3 py-1 rounded-full text-xs font-semibold border-2`
   - Selected: filled background, unselected: border-only
4. **Project select** — full-width `<select> min-h-[44px]`, placeholder "— Проект (необязательно) —", nullable
5. **Action buttons** — two-column `flex gap-2`:
   - [Сохранить]: accent full `flex-1 min-h-[44px] rounded-lg text-white font-semibold text-sm` `style="background:#4f46e5;"`
   - [Отмена]: secondary `flex-1 min-h-[44px] rounded-lg text-sm font-semibold text-gray-600 bg-gray-100`

**Success state** (replaces form via HTMX on POST `/m/errors/{error_id}/brief`):
- `bg-green-50 rounded-lg p-3 flex items-center gap-2`
- Green check icon (Heroicons `check-circle` 20x20 `text-green-600`)
- "ТЗ создано —" `text-sm text-green-800` + `<a href="/ui/tasks/{task_id}" class="text-indigo-600 underline text-sm">открыть</a>`

### 3. `/m/tasks/new` — Quick Create Page

**Layout:** Single page. Top: mode toggle. Below: form area (HTMX-swapped on toggle change).

**Sections (top to bottom):**
1. **Page heading:** "Новая задача" — `text-xl font-semibold mb-4`
2. **Mode toggle** — two-button row, full-width, `flex mb-4`:
   - [Задача] and [ТЗ копирайтеру] — each `flex-1 min-h-[44px] text-sm font-semibold rounded-lg`
   - Active: `style="background:#4f46e5;" text-white`
   - Inactive: `bg-gray-100 text-gray-500`
   - Toggle change → `hx-get="/m/tasks/new/form?mode={mode}" hx-target="#task-form" hx-swap="innerHTML"` + update query-param via `history.pushState`
3. **`#task-form`:** Form partial, loaded from `tasks/partials/task_form.html` or `tasks/partials/brief_form.html`

**mode=task form (default):**
- Textarea `name="text"` — `w-full min-h-[120px] border border-gray-300 rounded-lg px-3 py-2 text-sm`, placeholder "Текст задачи...", required, maxlength 2000
- Priority radio — same 4-option inline pill pattern as inline brief form above, default P3
- Project select — `<select name="project_id"> min-h-[44px]`, placeholder "— Выберите проект —", required (all projects for user)
- Submit: "Создать задачу" — full-width accent, `w-full min-h-[44px] rounded-lg text-white font-semibold text-sm`, `style="background:#4f46e5;"`

**mode=brief form:**
- Textarea `name="keywords"` — `w-full min-h-[120px]`, placeholder "Ключевые слова (по одному на строку)", required
- Tone select `name="tone"` — full-width `min-h-[44px]`, options: "Информационный", "Коммерческий", "Экспертный"
- Length select `name="length"` — full-width `min-h-[44px]`, options: "1000 слов", "2000 слов", "3000 слов", "5000 слов"
- Project select `name="project_id"` — full-width `min-h-[44px]`, placeholder "— Выберите проект —", required
- Recipient select `name="recipient_id"` — full-width `min-h-[44px]`, placeholder "— Клиент (необязательно) —", shows clients with `email IS NOT NULL OR telegram_username IS NOT NULL`, nullable (if null → save only, no send)
- Two submit buttons stacked (column):
  - "Создать и отправить" — full-width accent, shown only when recipient selected; disabled `opacity-50 cursor-not-allowed` when recipient empty
  - "Только сохранить" — full-width secondary outline `border border-gray-300 text-gray-700 bg-white`

**Validation error (mode=task, Project.site_id is null):**
- `showToast("У проекта нет привязанного сайта. Привяжите сайт в настройках проекта.", 'error')`

**Success (both modes):**
- `showToast("Задача создана", 'success')` (mode=task)
- `showToast("ТЗ создано и отправлено клиенту", 'success')` or `showToast("ТЗ сохранено", 'success')` depending on recipient
- Redirect to `/m/`

## Interaction Contracts

### HTMX Patterns

| Trigger | HTMX Attrs | Target | Swap |
|---------|-----------|--------|------|
| Site dropdown change | `hx-get="/m/errors?site_id={id}"` | `#errors-content` | `innerHTML` |
| "Обновить" button | `hx-post="/m/errors/sync" hx-include="[name=site_id]"` | `#sync-progress` | `innerHTML` |
| Sync polling | `hx-get="/m/errors/sync/status/{task_id}" hx-trigger="every 3s"` | `this` | `outerHTML` (stops in done/error) |
| "Составить ТЗ" tap | `hx-get="/m/errors/{error_id}/brief/form"` | `closest .error-row` | `outerHTML` |
| Inline brief form POST | `hx-post="/m/errors/{error_id}/brief"` | `this` (form) | `outerHTML` → success block |
| Mode toggle tap | `hx-get="/m/tasks/new/form?mode={mode}"` | `#task-form` | `innerHTML` |
| Mode toggle URL sync | JS `history.pushState({}, '', '/m/tasks/new?mode={mode}')` | — | — |
| "Показать все" tap | `hx-get="/m/errors/{type}/all?site_id={id}&page=1"` | `closest section .error-list` | `innerHTML` (replaces top-5 with full list) |

### Form Validation
- mode=task submit: disabled until `textarea` has content AND `project_id` is selected. JS `input`/`change` listeners toggle `disabled` + `opacity-50 cursor-not-allowed`.
- mode=brief "Создать и отправить": disabled until `keywords` has content AND `project_id` selected AND `recipient_id` selected. "Только сохранить" enabled as long as keywords + project filled.
- Inline brief "Сохранить": disabled until textarea has content.
- No inline field validation — server-side errors via toast.

### Navigation
`active_tab` values for new pages:

| Route | active_tab |
|-------|-----------|
| `/m/errors` | `'errors'` (new tab to be added to base_mobile.html bottom nav) |
| `/m/tasks/new` | `'more'` (existing "Ещё" tab) |

Note for planner: bottom nav in `base_mobile.html` needs a new "Ошибки" tab entry for `/m/errors`. Planner to decide final bottom nav structure for v4.0.

### Accessibility
- All interactive elements: `min-h-[44px]` touch target.
- Priority radio pills: `<label>` wrapping `<input type="radio" class="sr-only">` + styled pill div, so tap activates radio.
- Error row "Составить ТЗ": full tap area `min-h-[44px] flex items-center`.
- Mode toggle buttons: each `min-h-[44px]`.
- Sync progress slot: `aria-live="polite"` on `#sync-progress`.
- Inline brief success block: `aria-live="polite"` on expanded `.error-row`.

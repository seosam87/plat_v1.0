"""Document generator router: generate, list, download, send, regenerate, delete."""

Router prefix: `/ui/crm/clients/{client_id}/documents`
Tags: `["documents"]`
All endpoints require `require_manager_or_above` (per D-11).

**Endpoint: GET `/` — Documents tab content**
- Query params: `doc_type: str | None = None`, `date_from: str | None = None`, `date_to: str | None = None`
- Fetch client from DB (404 if not found)
- Fetch client sites for the generate form dropdown: `select(Site).where(Site.client_id == client_id)`
- Fetch templates: `await template_service.list_templates(db)`
- Fetch documents: `await document_service.list_documents(db, client_id, doc_type=..., date_from=..., date_to=...)`
- Parse `date_from`/`date_to` strings as `datetime.date` using `datetime.strptime(val, "%Y-%m-%d").date()` with try/except
- If request has `HX-Request` header, return just the `_documents_tab.html` partial (for HTMX refresh)
- Otherwise return full `crm/detail.html` with `active_tab="documents"` in context
- Template context: `client`, `sites`, `templates_list`, `documents`, `active_tab`, `doc_type_filter`, `date_from_filter`, `date_to_filter`

**Endpoint: POST `/generate` — Create document + dispatch Celery task**
- Form fields: `template_id: uuid.UUID = Form(...)`, `site_id: uuid.UUID = Form(...)`
- Fetch template, validate exists (400 if not)
- Call `enforce_version_cap(db, client_id, template_id)`
- Build filename: `build_filename(template.template_type.value, client.company_name)`
- Create document: `create_document(db, client_id=..., site_id=..., template_id=..., document_type=template.template_type, file_name=filename)`
- Dispatch: `task = generate_document_pdf.delay(str(doc.id), str(template_id), str(client_id), str(site_id))`
- Update `doc.celery_task_id = task.id`, `await db.commit()`
- Return `_gen_status.html` partial with `doc=doc, client_id=client_id` (generating state)

**Endpoint: GET `/{doc_id}/status` — HTMX polling endpoint**
- Fetch document by doc_id, validate client_id match
- Return `_gen_status.html` partial with current status
- If status is `ready`: add `HX-Trigger: refreshDocList` response header
- If status is terminal (`ready` or `failed`): partial has no `hx-trigger` attribute (polling stops)

**Endpoint: GET `/{doc_id}/download` — PDF download** (per DOC-03)
- Fetch document, validate `status == "ready"` and `pdf_data is not None` (400 if not ready)
- Return `Response(content=doc.pdf_data, media_type="application/pdf", headers={"Content-Disposition": f'attachment; filename="{doc.file_name}"'})`

**Endpoint: POST `/{doc_id}/send` — Dispatch send task** (per D-08, D-09, D-10, DOC-05)
- Form field: `channel: str = Form(...)` — must be "email" or "telegram"
- Fetch document, validate `status == "ready"`
- Fetch client for email address
- If `channel == "email"`:
  - Validate `client.email is not None`, return 400 with message "Email клиента не указан. Добавьте email в карточке клиента." if absent
  - `recipient = client.email`
- If `channel == "telegram"`:
  - `recipient = ""` (text notification, no specific recipient)
- Dispatch: `send_document.delay(str(doc.id), channel, recipient, str(client_id))`
- Return HTML snippet for toast: `<div class="toast toast-success">Документ отправляется...</div>` (or use inline style matching project pattern)

**Endpoint: POST `/{doc_id}/regenerate` — Create new version** (per D-04)
- Fetch original document to get template_id and site_id
- Validate template_id is not None (400 if template was deleted)
- Fetch template
- Call `enforce_version_cap(db, client_id, doc.template_id)`
- Create new document record + dispatch task (same logic as `/generate`)
- Return `_gen_status.html` partial (generating state)

**Endpoint: POST `/{doc_id}/delete` — Hard delete**
- Call `delete_document(db, doc_id)`
- If False (active job or not found), return 400
- `await db.commit()`
- Return redirect to documents tab or `HX-Trigger: refreshDocList` response

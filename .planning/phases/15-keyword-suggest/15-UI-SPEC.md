| Token | Value | Usage |

- Wordstat banner: `padding: 0.75rem 1rem` with dismiss `×` button 24px touch target

## Component Inventory

All components use existing project patterns. No new component library introduced.

### 1. Sidebar Navigation Entry

- Location: `app/templates/components/sidebar.html`
- New top-level section "Keyword Suggest" added after "Клиентские отчёты" section
- Icon: `magnifying-glass` (Heroicons, 1.25rem, stroke-width 1.5, stroke currentColor) — already present in sidebar icon set
- Section type: collapsible, with single child "Поиск подсказок"
- Child URL: `/ui/keyword-suggest/`
- Active color: `#4338ca` background on child when active

SVG path for `magnifying-glass` icon:
```

```
Use: `<path stroke-linecap="round" stroke-linejoin="round" d="m21 21-5.197-5.197m0 0A7.5 7.5 0 1 0 5.196 5.196a7.5 7.5 0 0 0 10.607 10.607Z" />`

### 2. Page Layout

- Extends `base.html`
- Page title (h1, text-2xl font-bold text-gray-900 mb-6): "Keyword Suggest"
- Breadcrumbs: Dashboard / Keyword Suggest
- Max content width: max-w-5xl (wider than client_reports due to results table)

### 3. Wordstat Banner (conditional)

Shown ONLY when no Yandex Direct OAuth token is configured. Dismissable (session-local JS dismiss, no backend call).

```
bg-amber-50 border border-amber-200 rounded-lg p-3 mb-4 flex items-center justify-between
```

- Left: amber warning icon (Heroicons `exclamation-triangle`, 1.25rem) + text "Настройте токен Яндекс.Директ в Настройках для просмотра частотности."
- Right: dismiss `×` button (`text-amber-600 hover:text-amber-800`, 24px touch target, `aria-label="Закрыть уведомление"`)
- Dismiss action: JavaScript `this.closest('.banner').remove()` — no HTMX, no backend

### 4. Search Form Card

- Wrapper: `.card` (white bg, 8px radius, 1.5rem padding, shadow)
- Card heading (h2, text-lg font-semibold text-gray-800 mb-4): "Поиск подсказок"

Form fields:

| Field | Type | Width | Notes |
|-------|------|-------|-------|
| Seed keyword | `<input type="text" name="seed">` | 100% | `placeholder="Введите ключевую фразу"`, `required`, max 100 chars |
| Google Suggest toggle | `<input type="checkbox" name="include_google">` | auto | Label: "Добавить Google Suggest" |
| Site context (optional) | `<select name="site_id">` | 100% | Populated from user's sites; optional — first option "Без привязки к сайту" |

Action row:
- "Найти подсказки" button: `.btn .btn-primary` + indigo accent, `font-weight:600`
- HTMX indicator: spinner SVG (`animate-spin`, 1.25rem) + "Загружаем..." text, visible only during `hx-request`
- Rate-limit note: static `<p class="text-xs text-gray-400 mt-2">Лимит: 10 запросов в минуту</p>`

HTMX submission:
```
hx-post="/ui/keyword-suggest/search"
hx-target="#suggest-status"
hx-swap="innerHTML"
hx-indicator="#search-indicator"
```

### 5. Polling Status Partial

Pattern mirrors `client_reports/partials/generation_status.html` exactly.

Three states rendered by server partial:

**Pending / running state:**
```
bg-blue-50 border border-blue-100 rounded-lg p-4 flex items-center gap-3
hx-get="/ui/keyword-suggest/status/{task_id}"
hx-trigger="load delay:3s"
hx-swap="outerHTML"
```
Content: spinner + "Загружаем подсказки..." + "Обычно занимает менее 30 секунд"

**Partial results warning (ban/proxy exhausted):**
```
bg-amber-50 border border-amber-200 rounded-lg p-3
```
Content: warning icon + "Получено {N} подсказок из ~{M} (прокси заблокированы, попробуйте позже)." The results table below still renders with partial data.

**Cache hit indicator:**
```
bg-green-50 border border-green-100 rounded-lg p-3 flex items-center gap-2
```
Content: checkmark icon (green) + "Из кэша (актуально {N} ч.) — {count} подсказок"

**Error state:**
```
bg-red-50 border border-red-200 rounded-lg p-3
```
Content: "Не удалось получить подсказки. Проверьте прокси-пул или повторите запрос через минуту."

### 6. Results Card

Rendered after polling completes. Appears as second `.card` below the search form card.

Card heading (h2, text-lg font-semibold text-gray-800 mb-4): "Результаты" + count badge (`<span class="badge badge-connected ml-2">{N}</span>`)

**Filter/search row (above table):**
- Text search input: `<input type="text" id="suggest-filter" placeholder="Фильтр по тексту...">`  — client-side JS `oninput` filter against table rows (no HTMX)
- Source filter: `<select id="source-filter">` — options: "Все источники", "Яндекс", "Google"
- Sort note: sorting handled by clicking column headers (JS `onclick` sort, no full reload)

**Table columns:**

| Column | Width | Notes |
|--------|-------|-------|
| Подсказка | ~50% | Keyword text, body 14px weight 400 |
| Источник | ~10% | Source badge: "Я" (red) or "G" (blue) — `<span class="badge">` |
| Частотность | ~15% | Numeric, right-aligned. Hidden column (CSS `display:none`) when Wordstat token not configured; shown after "Загрузить частотность" completes |
| Действия | ~25% | "Копировать" icon button per row |

Table uses existing `th, td` base.html CSS rules.

Sortable columns: Подсказка (alphabetical), Частотность (numeric desc/asc). Click `<th>` toggles asc/desc. Active sort column shows chevron icon (Heroicons, 0.75rem).

**Export row (below table):**
- "Копировать всё" button: copies all visible (filtered) keywords to clipboard, one per line. `aria-label="Скопировать все подсказки в буфер обмена"`. Gray secondary style (`border border-gray-300 bg-white text-gray-700`).
- "Скачать CSV" button: triggers GET `/ui/keyword-suggest/export?task_id={id}` — direct file download, no HTMX. Same gray secondary style. `aria-label="Скачать подсказки в формате CSV"`.

**"Загрузить частотность" button:**
- Appears below the export row ONLY when Wordstat token IS configured AND results are present
- Accent color (indigo): `.btn .btn-primary` style
- HTMX: `hx-post="/ui/keyword-suggest/{task_id}/wordstat"` → `hx-target="#wordstat-status"` → `hx-swap="innerHTML"`
- Shows inline loading status while Wordstat API fetches; on completion swaps in frequency values and un-hides the Частотность column

### 7. Wordstat Loading Status Partial

Small inline status area `#wordstat-status` below the "Загрузить частотность" button:

| State | Visual |
|-------|--------|
| Loading | Spinner + "Загружаем частотность из Яндекс.Директ..." |
| Complete | "Частотность загружена — {N} ключей обновлено." (green check icon) |
| Error | "Не удалось загрузить частотность. Проверьте токен Я.Директ в Настройках." (red) |

### 8. Toast Notifications

Reuse existing `#toast-container` + `.toast` + `.toast-success/.toast-error/.toast-info` classes from base.html.

## Interaction Contract (HTMX)

| Trigger | HTMX Verb | Target | Swap |
|---------|-----------|--------|------|
| "Найти подсказки" form submit | `hx-post="/ui/keyword-suggest/search"` | `#suggest-status` | `innerHTML` |
| Polling (pending/running) | `hx-get="/ui/keyword-suggest/status/{task_id}"` every 3s | self (`outerHTML`) | `outerHTML` |
| "Загрузить частотность" button | `hx-post="/ui/keyword-suggest/{task_id}/wordstat"` | `#wordstat-status` | `innerHTML` |
| "Скачать CSV" button | Direct `<a href>` GET — no HTMX | — | — |
| "Копировать всё" button | JS `navigator.clipboard.writeText()` — no HTMX | — | — |
| Text filter input | JS `oninput` — client-side DOM filter — no HTMX | table rows | DOM only |
| Source filter select | JS `onchange` — client-side DOM filter — no HTMX | table rows | DOM only |
| Column sort (th click) | JS `onclick` — client-side sort — no HTMX | table rows | DOM only |
| Banner dismiss (×) | JS `this.closest('.banner').remove()` — no HTMX | banner element | DOM only |

HTMX 2.0.x — no `hx-ws` or `hx-sse` attributes (moved to extensions in 2.0; not needed here).

## States per UI Element

### Search Form

| State | Visual |
|-------|--------|
| Default | Seed input empty, Google checkbox unchecked, CTA enabled |
| No seed entered | CTA disabled via HTML `required` attribute — no custom JS needed |
| Submitting | CTA shows HTMX indicator spinner + "Загружаем..."; inputs not disabled (HTMX 2.0 default) |
| Rate-limited (429 response) | Error partial swapped into `#suggest-status`; form re-enabled |

### Results Table

| State | Visual |
|-------|--------|
| Empty (0 results) | Empty state card: heading + body copy (see Copywriting Contract) |
| Populated | Table with filter row, export row, Wordstat button row |
| Filtered to 0 rows | Table renders with 0 visible rows; no separate empty state needed |
| Wordstat loading | Частотность column cells show "—" with spinner overlay on column header |
| Wordstat complete | Частотность column populated with integer values, column visible |
| Wordstat token absent | Частотность column absent from DOM entirely; "Загрузить частотность" button not rendered |

### Polling Status

| State | Visual |
|-------|--------|
| pending / running | Blue polling partial (3s auto-refresh) |
| complete (full results) | Cache hit green partial OR silent swap to results card |
| complete (partial, ban) | Amber partial warning above results card |
| failed | Red error partial |

## CSV Export Format

Source: CONTEXT.md "Claude's Discretion" — format unspecified, resolved here.

- Encoding: UTF-8 with BOM (Excel-safe for Russian text)
- Delimiter: comma (`,`)
- Headers: `Подсказка,Источник,Частотность`
- Частотность column: empty string if not loaded; numeric integer if loaded
- Source values: `Яндекс` or `Google`
- Filename: `suggest_{seed_slug}_{YYYY-MM-DD}.csv`
- Content-Disposition: `attachment`

<objective>
Build the complete UI layer for keyword suggest: FastAPI router with search/status/export endpoints, Jinja2 templates with HTMX polling, results table with client-side filtering and sorting, CSV export, and sidebar navigation entry. Includes Wordstat frequency dispatch endpoint (POST /{job_id}/wordstat) and its polling status endpoint so the HTMX button in suggest_results.html has a working HTTP entrypoint.

Purpose: Connects the backend (Plan 01) to the user-facing interface so users can enter a seed keyword, see results appear via polling, filter/sort them, export, and optionally dispatch Wordstat frequency loading.
Output: Router, templates, navigation entry, main.py registration, and router tests.

    Endpoints:

    1. **GET `/ui/keyword-suggest/`** -- main page
       - Query sites for optional site_id dropdown: `select(Site.id, Site.domain).order_by(Site.domain)`
       - Check if Yandex Direct token is configured: `get_credential_async(db, "yandex_direct")` -- if None, show Wordstat banner
       - Render `keyword_suggest/index.html` with `sites`, `has_wordstat_token`

    2. **POST `/ui/keyword-suggest/search`** -- dispatch suggest task
       - Rate limited: `@limiter.limit("10/minute")` (import limiter from `app.main`)
       - Actually get limiter from `request.app.state.limiter` to avoid circular import. Use pattern: define module-level `from slowapi import Limiter` and get instance via middleware.
       - Form params: `seed: str = Form(...)`, `include_google: bool = Form(False)`, `site_id: str | None = Form(None)`
       - Create SuggestJob in DB with status="pending", set cache_key
       - Check Redis cache first (async): if cache hit, set job.status="complete", job.cache_hit=True, job.result_count=len(data)
       - If cache miss: dispatch `fetch_suggest_keywords.delay(str(job.id))`; store celery_task_id
       - Return `suggest_status.html` partial with job data

    3. **GET `/ui/keyword-suggest/status/{job_id}`** -- HTMX polling endpoint
       - Load SuggestJob from DB
       - If status in ("pending", "running"): return polling partial (auto-refresh 3s)
       - If status == "complete" or "partial": load results from Redis via job.cache_key, return results partial
       - If status == "failed": return error partial

    4. **GET `/ui/keyword-suggest/export`** -- CSV download
       - Query param: `job_id`
       - Load results from Redis via job.cache_key
       - Generate CSV with UTF-8 BOM per UI-SPEC:
         - Headers: `Подсказка,Источник,Частотность`
         - Source: "Яндекс" or "Google"
         - Frequency: empty string if not loaded
         - Filename: `suggest_{seed_slug}_{YYYY-MM-DD}.csv`
       - Return StreamingResponse with Content-Disposition: attachment

    5. **POST `/ui/keyword-suggest/{job_id}/wordstat`** -- dispatch Wordstat frequency fetch
       - This is the endpoint the HTMX button in `suggest_results.html` targets (`hx-post="/ui/keyword-suggest/{{ job.id }}/wordstat"` `hx-target="#wordstat-status"`).
       - Load SuggestJob from DB by `job_id`. Return 404 if not found.
       - Verify job has status in ("complete", "partial") and has a `cache_key` -- otherwise return error HTML partial "Сначала дождитесь результатов подсказок."
       - Check if Yandex Direct token is configured: `get_credential_async(db, "yandex_direct")` -- if None, return error HTML partial "Токен Яндекс.Директ не настроен. Перейдите в Настройки."
       - **Lazy import inside endpoint body:** `from app.tasks.suggest_tasks import fetch_wordstat_frequency` (NOT at module level — Plan 03 creates this task, and both plans are Wave 2; lazy import avoids ImportError if Plan 02 executes first)
       - Dispatch `fetch_wordstat_frequency.delay(str(job.id))`
       - Return an HTMX partial for `#wordstat-status` showing a spinner + "Загружаем частотность..." text with auto-poll:
         `hx-get="/ui/keyword-suggest/{{ job.id }}/wordstat-status"` `hx-trigger="load delay:3s"` `hx-swap="outerHTML"`

    6. **GET `/ui/keyword-suggest/{job_id}/wordstat-status`** -- Wordstat polling endpoint
       - Load SuggestJob from DB. Read the Wordstat task result from Redis (check if frequency data has been written to the cached suggestions under `job.cache_key`).
       - Detection: load the cached suggestions JSON, check if the first item has a `"frequency"` key. If yes, Wordstat is complete.
       - **If frequency not yet loaded:** return polling partial with spinner + "Загружаем частотность..." + `hx-get="/ui/keyword-suggest/{{ job.id }}/wordstat-status"` `hx-trigger="load delay:3s"` `hx-swap="outerHTML"`
       - **If frequency loaded:** return success partial "Частотность загружена" (green text) + trigger a full results reload by including `hx-trigger="load"` `hx-get="/ui/keyword-suggest/status/{{ job.id }}"` `hx-target="#suggest-results"` `hx-swap="innerHTML"` to refresh the results table with frequency column now visible.
       - **Alternative simpler approach:** return a small HTML partial with "Частотность загружена" + an OOB swap `hx-swap-oob="innerHTML:#suggest-results"` containing the updated results table (with frequency column unhidden). This avoids a separate reload request.

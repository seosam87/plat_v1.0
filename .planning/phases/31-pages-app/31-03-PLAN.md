**quick_fix_toc task** (per D-13 — direct push, no pipeline):
```python
@celery_app.task(bind=True, max_retries=3, queue="wp")
def quick_fix_toc(self, page_id: str) -> dict:
```
Steps:
1. `with get_sync_db() as db:` load Page by id
2. Resolve wp_post_id: query `WpContentJob` WHERE `page_url == page.url AND site_id == page.site_id` ORDER BY `created_at DESC LIMIT 1`. Extract `wp_post_id`. If None, try WP REST API lookup by URL slug (use httpx sync client to `GET {site.wp_url}/wp-json/wp/v2/posts?slug={slug}`). If still None, raise error — cannot push without wp_post_id.
3. Fetch current WP content: httpx GET `{site.wp_url}/wp-json/wp/v2/posts/{wp_post_id}` with auth headers (reference `_sync_auth_headers()` pattern from wp_content_tasks.py — load Site, decrypt wp_app_password with Fernet).
4. Extract content from response JSON: `content = resp.json()["content"]["rendered"]`
5. Apply TOC: `headings = extract_headings(content)`, `toc_html = generate_toc_html(headings)`, `new_content = inject_toc(content, toc_html)`
6. Push to WP: httpx POST `{site.wp_url}/wp-json/wp/v2/posts/{wp_post_id}` with `{"content": new_content}` + auth headers
7. Update Page: `page.has_toc = True; db.commit()`
8. Return `{"status": "pushed", "page_id": page_id}`
9. On error: `self.retry(exc=exc)`; return `{"status": "failed", "error": str(exc)}`

**quick_fix_schema task** (per D-13 — direct push, no pipeline):
```python
@celery_app.task(bind=True, max_retries=3, queue="wp")
def quick_fix_schema(self, page_id: str) -> dict:
```
Steps:
1. Load Page via get_sync_db()
2. Resolve wp_post_id (same as TOC task)
3. Load SchemaTemplate: `template = db.execute(select(SchemaTemplate).where(SchemaTemplate.site_id == page.site_id).limit(1)).scalar_one_or_none()`. If no template, return error.
4. Build page_data dict: `{"title": page.title, "url": page.url, "description": page.meta_description, "page_type": page.page_type.value, "content_type": page.content_type.value}`
5. Render schema: `schema_json = render_schema_template(template.template_json, page_data)` (SYNC function — safe in Celery)
6. Generate tag: `schema_tag = f'<script type="application/ld+json">{schema_json}</script>'`
7. Fetch current WP content (same as TOC)
8. Inject: `new_content = inject_schema(content, schema_tag)`
9. Push to WP
10. Update Page: `page.has_schema = True; db.commit()`
11. Return result

**1. Add bulk Celery tasks to app/tasks/pages_tasks.py** (per D-16, D-18):

**bulk_fix_schema task:**
```python
@celery_app.task(bind=True, max_retries=0, queue="wp")
def bulk_fix_schema(self, site_id: str) -> dict:
```
Steps:
1. `import redis, json; r = redis.Redis.from_url(settings.REDIS_URL)`
2. `with get_sync_db() as db:` find latest done CrawlJob for site, query all Pages WHERE `has_schema == False AND crawl_job_id == latest_crawl_id`
3. `total = len(pages)`. Set Redis: `r.setex(f"bulk:{self.request.id}:progress", 3600, json.dumps({"done": 0, "total": total, "errors": [], "status": "running"}))`
4. For each page:
   - Try: resolve wp_post_id, load SchemaTemplate, render_schema_template, fetch WP content, inject_schema, push to WP, update `page.has_schema = True`
   - On error: append error to errors list, continue (one page failure must not stop others — per CLAUDE.md Celery constraint)
   - Increment done counter: `progress["done"] += 1; r.setex(f"bulk:{self.request.id}:progress", 3600, json.dumps(progress))`
5. Final status: `progress["status"] = "done" if not progress["errors"] else "error"; r.setex(...)`
6. Return `{"done": progress["done"], "total": total, "errors": len(progress["errors"])}`

**bulk_fix_toc task:** Same pattern but uses extract_headings + generate_toc_html + inject_toc instead of schema functions. Filters `has_toc == False`.

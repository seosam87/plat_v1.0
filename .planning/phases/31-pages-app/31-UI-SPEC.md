## Spacing Scale

Source: Phase 30 UI-SPEC (carried forward unchanged).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (`gap-1`), badge inner padding, status icon gap |
| sm | 8px | Between page cards (`gap-2`), button inline padding, status icon row gap |
| md | 16px | Page content padding (`p-4`), form field margin-bottom (`mb-4`) |
| lg | 24px | Section heading margin-bottom (`mb-6`) |
| xl | 32px | Major vertical rhythm between sections |
| 2xl | 48px | Not used in mobile context |
| 3xl | 64px | Bottom nav height reservation (`padding-bottom: 64px`) |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (`min-h-[44px]`) — source: Phase 29 UI-SPEC
- Bottom nav: 64px fixed height — source: base_mobile.html
- Toast bottom offset: 80px from bottom — source: base_mobile.html `#toast-container`
- Progress bar height: 8px (`h-2`) — source: tool_progress.html pattern
- SERP preview title max-width: 600px equivalent (truncate at ~60 chars) — client-side JS
- HTML diff block padding: `p-3` (12px) to allow readable line-diff content

## Typography

Source: Phase 30 UI-SPEC (carried forward unchanged).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / page list item text | 14px (`text-sm`) | 400 regular | 1.5 (default) |
| Label / secondary meta / URL / word_count | 12px (`text-xs`) | 400 regular | 1.5 |
| Page heading | 20px (`text-xl`) | 600 semibold (`font-semibold`) | 1.2 |
| Section heading / card title / tab label | 14px (`text-sm`) | 600 semibold (`font-semibold`) | 1.4 |

Rules:
- Maximum 2 font weights: 400 (regular) and 600 (semibold).
- No font-size outside this declared set.
- URL in page card: `text-xs text-gray-500 truncate` — single-line truncation.
- Title in expanded detail: `text-sm font-semibold` (one line, truncate).
- SERP preview title: `text-sm font-semibold text-blue-700` (truncated at 60 chars).
- SERP preview URL: `text-xs text-green-700`.
- SERP preview description: `text-sm text-gray-600` (truncated at 160 chars).
- HTML diff `<ins>` and `<del>` text: `text-sm` / `text-xs` matching surrounding context.

## Color

Source: Phase 30 UI-SPEC (carried forward unchanged). New semantic additions for pages audit status and diff rendering.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f5f5f5` | Page background (`body`) |
| Secondary (30%) | `#ffffff` | Cards, form surfaces, expanded detail, pipeline job card |
| Nav surface | `#1e1b4b` | Bottom navigation bar — source: base_mobile.html |
| Accent (10%) | `#4f46e5` (indigo-600) | Specific elements listed below |
| Accent light | `#818cf8` (indigo-400) | Active bottom-nav tab label + icon — source: base_mobile.html |
| Destructive | `#991b1b` (red-800) | Error toast text; reject button text; failed job status text |
| Destructive surface | `#fef2f2` (red-50) | Error toast background; reject confirm button bg; failed job bg tint |
| Success surface | `#ecfdf5` (green-50) | Success toast background; pushed job bg tint; approve confirm button bg |
| Success text | `#065f46` (green-900) | Success toast text; pushed job status text |
| Info surface | `#eff6ff` (blue-50) | Info toast background |
| Info text | `#1e40af` (blue-800) | Info toast text |
| Warning surface | `#fffbeb` (amber-50) | awaiting_approval badge background |
| Warning text | `#92400e` (amber-800) | awaiting_approval badge text |

**Semantic colors for audit status icons (status icons inside page cards):**
- Schema present: `text-green-600` (checkmark icon)
- Schema missing: `text-gray-300` (x-mark or dash icon)
- TOC present: `text-green-600` (checkmark icon)
- TOC missing: `text-gray-300` (x-mark or dash icon)
- Indexed (has_noindex=false): `text-green-600` (checkmark icon)
- Noindex (has_noindex=true): `text-red-500` (exclamation icon)

**HTML diff colors:**
- `<ins>` (added content): `bg-green-100 text-green-800` — inline highlighting
- `<del>` (removed content): `bg-red-100 text-red-800 line-through` — inline highlighting
- Diff container border: `border-l-2 border-gray-300`

**Pipeline job status badges:**
- awaiting_approval: `bg-amber-50 text-amber-800 text-xs font-semibold px-2 py-0.5 rounded-full`
- pushed: `bg-green-50 text-green-800 text-xs font-semibold px-2 py-0.5 rounded-full`
- failed: `bg-red-50 text-red-800 text-xs font-semibold px-2 py-0.5 rounded-full`

**2-tap confirmation button states:**
- First state (Принять): `bg-white border border-gray-300 text-gray-700` — secondary outline
- Confirm state (Подтвердить?): `style="background:#ecfdf5;" border border-green-600 text-green-800 font-semibold` — 2-second window
- First state (Отклонить): `bg-white border border-gray-300 text-gray-700` — secondary outline
- Confirm state (Подтвердить отклонение?): `bg-red-50 border border-red-600 text-red-800 font-semibold` — 2-second window
- First state (Откатить): `bg-white border border-gray-300 text-gray-600` — secondary outline
- Confirm state (Подтвердить откат?): `bg-amber-50 border border-amber-600 text-amber-800 font-semibold` — 2-second window

**Accent `#4f46e5` reserved for exactly these elements:**
1. Primary action button background: "Запустить краулинг" (empty state CTA), "Подтвердить" (bulk confirm screen), active tab underline border
2. Progress bar fill during bulk operations (`style="background:#4f46e5;width:{pct}%"`)
3. Active tab underline / active bottom-nav indicator
4. Quick fix button text when shown as text-link style: `text-indigo-600 underline`
5. "Загрузить ещё" text link: `text-indigo-600 underline`

**Accent is NOT used for:**
- 2-tap confirm buttons — use semantic green/red/amber per action type (above)
- "Отмена" / cancel buttons — use `bg-gray-100 text-gray-600`
- Status icons — use semantic green/red/gray (above)
- Diff highlights — use green-100/red-100 (above)

### 1. `/m/pages` — Pages List

**active_tab:** `'pages'` (new tab to be added to base_mobile.html bottom nav — planner decides icon; recommend Heroicons `document-text` 24x24 stroke 1.5)

**Layout:** Single scrolling page. Top: site dropdown. Below: 4 audit tabs with count-badges. Below: page card list with "Загрузить ещё". Below: bulk operation buttons.

**Sections (top to bottom):**
1. **Page heading:** "Страницы сайта" — `text-xl font-semibold mb-4`
2. **Site dropdown:** `<select name="site_id"> full-width min-h-[44px] mb-3`, shows all `Site.domain` values for user. Change → `hx-get="/m/pages?site_id={id}&tab=all" hx-target="#pages-content" hx-swap="innerHTML"`. Cookie `m_pages_site_id` persists selection (same pattern as Phase 30 `m_errors_site_id`).
3. **Audit tabs (tab bar):** `flex border-b border-gray-200 mb-4` — 4 tabs:
   - Все | Без Schema | Без TOC | Noindex
   - Each tab: `flex-1 py-2 text-sm min-h-[44px]`
   - Active: `font-semibold border-b-2 border-indigo-600 text-indigo-600`
   - Inactive: `text-gray-500`
   - Each tab displays count-badge inline: `text-xs bg-gray-100 text-gray-600 rounded-full px-1.5 ml-1`
   - Tab change → `hx-get="/m/pages?site_id={id}&tab={tab}" hx-target="#pages-content" hx-swap="innerHTML"`
4. **`#pages-content`:** Contains page card list. HTMX swap target on site and tab change.
5. **Bulk operation buttons** (below `#pages-content`, NOT inside HTMX swap target):
   - Two full-width buttons stacked, `mt-6 space-y-2`:
   - "Добавить Schema на все статьи" — `w-full min-h-[44px] rounded-lg border border-gray-300 text-gray-700 bg-white text-sm font-semibold` — tap → navigate to `/m/pages/bulk/schema/confirm?site_id={id}`
   - "Добавить TOC на все статьи" — same styling — tap → navigate to `/m/pages/bulk/toc/confirm?site_id={id}`

**Page card (`.page-row`):**
- Container: `bg-white rounded-lg divide-y divide-gray-100`
- Each row: `.page-row p-3 min-h-[44px] flex items-center justify-between`
- Left block (grows): URL `text-xs text-gray-500 truncate block`
- Right block: status icon trio `flex items-center gap-1` (3 icons, each 20x20, one for Schema, one for TOC, one for Index status)
  - Schema icon: Heroicons `code-bracket` — green if has_schema, gray-300 if missing
  - TOC icon: Heroicons `list-bullet` — green if has_toc, gray-300 if missing
  - Index icon: Heroicons `eye` — green if indexed, red-500 if noindex
- Full row is tappable: `hx-get="/m/pages/detail/{page_id}" hx-target="closest .page-row" hx-swap="outerHTML" hx-indicator=".page-row-spinner"`

**"Загрузить ещё" (pagination):**
- Below card list: `text-sm text-indigo-600 underline mt-3 block text-center cursor-pointer`
- `hx-get="/m/pages?site_id={id}&tab={tab}&offset={next_offset}" hx-target="closest .pages-list" hx-swap="beforeend"`
- Hidden when fewer than 20 results remain.

**Empty state (no Page records for site):**
- `bg-white rounded-lg p-6 text-center`
- Heading: `text-sm font-semibold text-gray-700 mb-2` — "Нет данных о страницах"
- Body: `text-xs text-gray-500 mb-4` — "Запустите краулинг, чтобы получить список страниц сайта."
- CTA button: `min-h-[44px] rounded-lg text-white font-semibold text-sm px-6 py-3 style="background:#4f46e5;"` — "Запустить краулинг"
- CTA action: `hx-post="/api/crawl/start" hx-vals='{"site_id":"{id}"}' hx-target="#pages-content" hx-swap="innerHTML"` → shows inline progress partial

**Empty state (no sites):**
- `bg-white rounded-lg p-4 text-sm text-gray-500 text-center` — "Нет сайтов. Добавьте сайт в полной версии платформы."

### 2. Inline Expand Detail (partial: `pages/partials/page_detail.html`)

Triggered by tapping a `.page-row`. Replaces the row via HTMX `outerHTML` swap.

**Layout (replaces entire .page-row):** `bg-white rounded-lg p-4 space-y-3 border-l-2 border-indigo-600`

**Sections (top to bottom):**
1. **URL display:** `text-xs text-gray-500 truncate block mb-2` — full URL
2. **Metadata grid:** `grid grid-cols-2 gap-2 mb-3`
   - Each cell: `bg-gray-50 rounded p-2`
   - Label: `text-xs text-gray-400 block`
   - Value: `text-sm font-semibold text-gray-700`
   - Fields: Title / H1 / Meta description (spans full width) / Word count / HTTP status
   - Title cell: `col-span-2` — `text-sm font-semibold text-gray-700 truncate`
   - Meta description: `col-span-2` — `text-sm text-gray-600` (2-line clamp `line-clamp-2`)
   - Word count: `text-sm font-semibold` + label "Слов"
   - HTTP status: `text-sm font-semibold` + label "HTTP" — color `text-green-600` if 200, `text-red-600` if 4xx/5xx
3. **Quick fix buttons** (conditional, shown only when applicable):
   - "Добавить TOC" — shown only if `!has_toc`. Style: `w-full min-h-[44px] rounded-lg text-white font-semibold text-sm style="background:#4f46e5;"` — `hx-post="/m/pages/fix/{page_id}/toc" hx-target="closest .page-row" hx-swap="outerHTML"`
   - "Добавить Schema" — shown only if `!has_schema`. Same accent style. `hx-post="/m/pages/fix/{page_id}/schema" hx-target="closest .page-row" hx-swap="outerHTML"`
   - "Изменить Title/Meta" — always shown. Style: `w-full min-h-[44px] rounded-lg border border-gray-300 text-gray-700 bg-white text-sm font-semibold` — navigate `href="/m/pages/{site_id}/{page_id}/edit"`
4. **"Создать задачу" button:** `w-full min-h-[44px] rounded-lg bg-gray-100 text-gray-600 text-sm font-semibold` — navigate `href="/m/tasks/new?mode=task&page_id={page_id}&site_id={site_id}"`
5. **Collapse button:** `w-full text-center text-xs text-gray-400 mt-2` — "Свернуть ▲" — `hx-get="/m/pages/detail/{page_id}/collapsed" hx-target="this.closest('.page-row-expanded')" hx-swap="outerHTML"` (restores original compact row)

**Quick fix success state** (response to TOC/Schema fix POST):
- Replace expanded detail with: `bg-green-50 rounded-lg p-3 flex items-center gap-2 min-h-[44px]`
- Green check icon (Heroicons `check-circle` 20x20 `text-green-600`)
- Text: `text-sm text-green-800` — "TOC добавлен и отправлен в WP" or "Schema добавлена и отправлена в WP"
- `showToast("TOC добавлен", 'success')` — via HX-Trigger response header

**Quick fix error state:**
- `showToast("Ошибка при добавлении TOC. Попробуйте ещё раз.", 'error')` — via HX-Trigger response header

### 2-Tap Confirmation JS Pattern

All 3 confirmation buttons (Принять / Отклонить / Откатить) follow identical JS pattern. No modals. No HTMX on first tap — pure JS.

```javascript
// Applied to each confirm-button with data attributes:
// data-confirm-text="Подтвердить?"
// data-confirm-class="bg-green-50 border-green-600 text-green-800 font-semibold"  (or red/amber variant)
// data-action-url="/m/pipeline/{job_id}/approve"
// data-target="#job-{job_id}"

function initTwoTapButton(btn) {
  let timer = null;
  btn.addEventListener('click', function() {
    if (btn.dataset.ready === 'true') {
      // Second tap: fire HTMX POST
      htmx.ajax('POST', btn.dataset.actionUrl, {
        target: btn.dataset.target,
        swap: 'outerHTML'
      });
      return;
    }
    // First tap: transform button
    const originalText = btn.textContent;
    const originalClass = btn.className;
    btn.textContent = btn.dataset.confirmText;
    btn.className = btn.className.replace(/border-gray-300 text-gray-700 bg-white/, btn.dataset.confirmClass);
    btn.dataset.ready = 'true';
    // Timeout: reset after 2 seconds
    timer = setTimeout(function() {
      btn.textContent = originalText;
      btn.className = originalClass;
      btn.dataset.ready = 'false';
    }, 2000);
  });
}
document.querySelectorAll('[data-confirm-text]').forEach(initTwoTapButton);
```

Script placement: inline `<script>` at bottom of pipeline page template, or appended to `base_mobile.html` block `{% block scripts %}`.

### SERP Preview JS Pattern

In `/m/pages/{site_id}/{page_id}/edit` — client-side only, no HTMX.

```javascript
const titleInput = document.getElementById('title-input');
const metaInput = document.getElementById('meta-input');
const serpTitle = document.getElementById('serp-title');
const serpDesc = document.getElementById('serp-desc');
const titleCounter = document.getElementById('title-counter');
const descCounter = document.getElementById('desc-counter');

function truncate(str, max) {
  return str.length > max ? str.substring(0, max) + '...' : str;
}

titleInput.addEventListener('input', function() {
  serpTitle.textContent = truncate(this.value, 60);
  titleCounter.textContent = this.value.length + '/60';
  titleCounter.className = this.value.length > 60
    ? 'text-xs text-red-500 text-right mt-1'
    : 'text-xs text-gray-400 text-right mt-1';
});

metaInput.addEventListener('input', function() {
  serpDesc.textContent = truncate(this.value, 160);
  descCounter.textContent = this.value.length + '/160';
  descCounter.className = this.value.length > 160
    ? 'text-xs text-red-500 text-right mt-1'
    : 'text-xs text-gray-400 text-right mt-1';
});
```

Script placement: inline `<script>` at bottom of edit page template inside `{% block scripts %}`.

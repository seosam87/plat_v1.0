<read_first>

</read_first>
<action>
Create `app/models/change_monitoring.py`:

**ChangeType** enum (str, PyEnum):
- `page_404` вАФ Page returned 404
- `noindex_added` вАФ noindex appeared on page
- `schema_removed` вАФ JSON-LD schema disappeared
- `title_changed` вАФ Title tag changed
- `h1_changed` вАФ H1 tag changed
- `canonical_changed` вАФ Canonical URL changed
- `meta_description_changed` вАФ Meta description changed
- `content_changed` вАФ Content preview changed
- `new_page` вАФ New page discovered

**AlertSeverity** enum (str, PyEnum):
- `error` вАФ Immediate Telegram alert
- `warning` вАФ Included in digest only
- `info` вАФ Feed only, no notification

**ChangeAlertRule** вАФ global rules for which changes trigger which severity:
- `id`: UUID PK
- `change_type`: SAEnum(ChangeType), unique, not null
- `severity`: SAEnum(AlertSeverity), not null, default `warning`
- `is_active`: Boolean, default True
- `description`: String(255), nullable
- `created_at`: DateTime(timezone=True)

**ChangeAlert** вАФ history of sent alerts:
- `id`: UUID PK
- `site_id`: UUID FK вЖТ sites.id CASCADE, not null
- `crawl_job_id`: UUID FK вЖТ crawl_jobs.id CASCADE, not null
- `change_type`: SAEnum(ChangeType), not null
- `severity`: SAEnum(AlertSeverity), not null
- `page_url`: String(2000), not null
- `details`: Text, nullable вАФ human-readable description (e.g. "Title: old вЖТ new")
- `sent_at`: DateTime(timezone=True), nullable вАФ when Telegram message was sent (null if not yet sent)
- `created_at`: DateTime(timezone=True)

**DigestSchedule** вАФ per-site weekly digest schedule:
- `id`: UUID PK
- `site_id`: UUID FK вЖТ sites.id CASCADE, unique, not null
- `is_active`: Boolean, default False
- `day_of_week`: Integer, default 1 (Monday=1, Sunday=7)
- `hour`: Integer, default 9 (9:00 UTC)
- `minute`: Integer, default 0
- `cron_expression`: String(50), nullable вАФ derived from day/hour/minute for redbeat
- `created_at`: DateTime(timezone=True)
- `updated_at`: DateTime(timezone=True), onupdate

Table names: `change_alert_rules`, `change_alerts`, `digest_schedules`.
</action>

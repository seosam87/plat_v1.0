# Plan v3-03-04: Router, UI for alert rules and digest schedule, site detail integration

<objective>
Create the monitoring router with endpoints for alert rules management, alert history, digest schedule, and manual digest trigger. Build the monitoring UI page with alert rules table, alert history feed, and digest schedule settings. Add "Мониторинг" button to site detail page.
</objective>

<tasks>

<task id="01">
<title>Create monitoring router</title>
<read_first>
- app/routers/audit.py (router pattern)
- app/services/change_monitoring_service.py
- app/services/digest_service.py
- app/models/change_monitoring.py
</read_first>
<action>
Create `app/routers/monitoring.py`:

Router: `prefix="/monitoring"`, tags `["monitoring"]`

**Page endpoint:**

1. `GET /{site_id}` — Main monitoring page (HTML). Fetches:
   - Site, alert rules, recent alerts for site (last 50), digest schedule
   - Template: `monitoring/index.html`

**Alert rules endpoints (global):**

2. `GET /rules` — List all alert rules (JSON)
3. `PUT /rules/{rule_id}` — Update rule (severity, is_active). Body: partial update.

**Alert history endpoints:**

4. `GET /{site_id}/alerts` — List recent alerts for site (JSON, last 100). Query param: `severity` filter.
5. `GET /{site_id}/alerts/count` — Count of unacknowledged alerts per severity.

**Digest endpoints:**

6. `GET /{site_id}/digest-schedule` — Get current digest schedule (JSON).
7. `PUT /{site_id}/digest-schedule` — Update digest schedule. Body: `{is_active, day_of_week, hour, minute}`.
8. `POST /{site_id}/digest/send` — Manually trigger digest for site (calls `send_digest` via Celery task).

All endpoints gated behind `require_admin`.
</action>

# Plan v3-03-03: Weekly digest Celery Beat task and schedule management

<objective>
Create the digest service and Celery Beat task: collect changes for a site over the past week, format a Telegram digest message, and send it. Per-site schedule management (day_of_week, hour, minute) via redbeat. Extends existing schedule_service patterns.
</objective>

<action>
Create `app/services/digest_service.py`:

### Digest builder (sync — for Celery)

**`build_digest(db: Session, site_id: UUID, days: int = 7) -> dict`**
Query `change_alerts` for site_id where `created_at >= now() - days`.
Group by severity (error, warning, info). Return:
```python
{
    "site_id": str,
    "site_name": str,
    "period": "2026-03-26 – 2026-04-02",
    "error": [{"change_type": str, "page_url": str, "details": str}, ...],
    "warning": [...],
    "info": [...],
    "total": int,
}
```

**`send_digest(db: Session, site_id: UUID) -> dict`**
Build digest → format via `telegram_service.format_weekly_digest()` → send via `send_message_sync()`.
Returns `{"sent": bool, "total_changes": int}`.

### Schedule management (async — for FastAPI)

**`async def get_digest_schedule(db: AsyncSession, site_id: UUID) -> DigestSchedule | None`**
Get the digest schedule for a site.

**`async def upsert_digest_schedule(db: AsyncSession, site_id: UUID, is_active: bool, day_of_week: int = 1, hour: int = 9, minute: int = 0) -> DigestSchedule`**
Create or update digest schedule. Compute `cron_expression` from params:
- `f"{minute} {hour} * * {day_of_week}"` (standard cron: min hour dom month dow)
- Note: cron day_of_week is 0=Sunday in cron, but user passes 1=Monday..7=Sunday. Convert: `day_of_week % 7` for cron.

If `is_active=True`, register with redbeat. If `is_active=False`, remove from redbeat.

**`def register_digest_beat(site_id: str, cron_expression: str) -> None`**
Register a redbeat entry for `send_weekly_digest` task with the given cron.
Use `RedBeatSchedulerEntry` pattern from existing `schedule_service.py`.

**`def remove_digest_beat(site_id: str) -> None`**
Remove the redbeat entry for this site's digest.

**`def restore_digest_schedules_from_db() -> None`**
Called on Beat startup. Load all active DigestSchedules from DB and register in redbeat.

<action>
Create `app/services/change_monitoring_service.py`:

### Pure detection function

**`detect_changes(page_url: str, old_snap: dict | None, new_snap: dict, http_status: int) -> list[dict]`**

Analyze diff between old and new snapshots. Returns list of `{change_type: str, details: str}`.

Logic:
- If `old_snap is None` → `[{"change_type": "new_page", "details": ""}]`
- If `http_status == 404` and old was not 404 → add `page_404`
- If `old["has_noindex"] == False` and `new["has_noindex"] == True` → add `noindex_added`
- If `old["has_schema"] == True` and `new["has_schema"] == False` → add `schema_removed`
- If `old["title"] != new["title"]` → add `title_changed` with details `f"{old['title']} → {new['title']}"`
- If `old["h1"] != new["h1"]` → add `h1_changed` with details
- If `old["canonical_url"] != new["canonical_url"]` → add `canonical_changed` with details
- If `old["meta_description"] != new["meta_description"]` → add `meta_description_changed` with details
- If `old["content_preview"] != new["content_preview"]` → add `content_changed`

### DB functions (sync — called from Celery)

**`get_alert_rules_sync(db: Session) -> dict[str, dict]`**
Returns `{change_type: {"severity": str, "is_active": bool}}` for all rules.

**`save_change_alerts(db: Session, site_id: UUID, crawl_job_id: UUID, changes: list[dict], rules: dict) -> list[ChangeAlert]`**
For each change:
- Look up severity from rules. Skip if rule is inactive.
- Create ChangeAlert record (sent_at=None initially).
- Return created alerts.

**`mark_alerts_sent(db: Session, alert_ids: list[UUID]) -> None`**
Set `sent_at = now()` for the given alerts.

### Telegram dispatch (sync)

**`dispatch_immediate_alerts(db: Session, site_name: str, alerts: list[ChangeAlert]) -> int`**
Filter alerts where severity == "error". For each, format message and call `telegram_service.send_message_sync()`. Mark as sent. Return count of messages sent.

### Orchestrator

**`process_crawl_changes(db: Session, site_id: UUID, site_name: str, crawl_job_id: UUID) -> dict`**
Called after crawl completes. Steps:
1. Get all pages from current crawl
2. For each page, find previous snapshot (from prior crawl job)
3. Run `detect_changes()` per page
4. Get alert rules
5. Save change alerts
6. Dispatch immediate (error) alerts via Telegram
7. Return `{"total_changes": N, "alerts_sent": N}`
</action>

**`format_change_alert(site_name: str, change_type: str, page_url: str, details: str = "") -> str`**
```
🔴 <b>SEO Alert</b>
Site: {site_name}
Change: {CHANGE_TYPE_LABELS[change_type]}
URL: {page_url}
{details if details else ""}
```

`CHANGE_TYPE_LABELS` dict mapping change_type → Russian label:
- `page_404` → "Страница вернула 404"
- `noindex_added` → "Добавлен noindex"
- `schema_removed` → "Удалена schema.org"
- `title_changed` → "Изменён title"
- `h1_changed` → "Изменён H1"
- `canonical_changed` → "Изменён canonical"
- `meta_description_changed` → "Изменено meta description"

**`format_weekly_digest(site_name: str, changes: list[dict], period: str) -> str`**
```
📊 <b>Еженедельный дайджест: {site_name}</b>
Период: {period}

🔴 Критичные ({error_count}):
{list of error changes}

⚠️ Предупреждения ({warning_count}):
{list of warning changes}

ℹ️ Информация ({info_count}):
{summary of info changes}

Всего изменений: {total}
```

If message exceeds 4000 chars (Telegram limit), truncate info section.

# Plan v3-02-05: Fix workflow — TOC preview, CTA injection, schema injection via pipeline

<objective>
Implement the fix workflow for auto-fixable audit checks: generate TOC preview for user approval, inject CTA block from site template, inject type-aware schema from templates. Each fix goes through the existing pipeline (generate → diff → approve → push → verify → record). After a successful fix, the audit result is updated to "fixed" status with a timestamp.
</objective>

<action>
Create `app/services/audit_fix_service.py`:

### Pure fix generators (no DB, testable)

**1. `generate_toc_fix(html: str) -> dict | None`**
- Extract headings from HTML using `content_pipeline.extract_headings()`
- If no headings → return None (can't generate TOC)
- Add heading IDs: `content_pipeline.add_heading_ids()`
- Generate TOC: `content_pipeline.generate_toc_html()`
- Inject TOC: `content_pipeline.inject_toc()`
- Compute diff: `content_pipeline.compute_content_diff()`
- Return `{"processed_html": str, "toc_html": str, "diff": dict, "headings_count": int}`

**2. `generate_cta_fix(html: str, cta_template: str) -> dict | None`**
- If no `cta_template` → return None
- Check if CTA already present (reuse `detect_cta_block` from audit_service)
- If already present → return None
- Append CTA to end of content: `html + "\n" + cta_template`
- Compute diff
- Return `{"processed_html": str, "diff": dict}`

**3. `generate_schema_fix(html: str, schema_tag: str) -> dict | None`**
- If `has_schema_ld(html)` → return None (already has schema)
- Inject schema: `content_pipeline.inject_schema(html, schema_tag)`
- Compute diff
- Return `{"processed_html": str, "diff": dict}`

**4. `generate_links_fix(html: str, keywords_with_urls: list[dict]) -> dict | None`**
- Find link opportunities: `content_pipeline.find_link_opportunities()`
- If no opportunities → return None
- Insert links: `content_pipeline.insert_links()`
- Compute diff
- Return `{"processed_html": str, "diff": dict, "links_added": int}`

### Verification function

**5. `verify_html_integrity(original: str, processed: str) -> dict`**
Basic sanity checks on processed HTML:
- Count of `<h2>` + `<h3>` tags hasn't changed (headings not destroyed)
- No unclosed `<script>` tags introduced
- Length of processed >= 80% of original (no catastrophic content loss)
- Return `{"valid": bool, "warnings": list[str]}`

### Pipeline integration (async)

**6. `async def create_fix_job(db: AsyncSession, site_id: UUID, page_url: str, wp_post_id: int | None, original_html: str, processed_html: str, fix_action: str) -> WpContentJob`**
- Create a `WpContentJob` with:
  - original_content = original_html
  - processed_content = processed_html
  - diff_json = computed diff
  - rollback_payload = `{"original_content": original_html, "wp_post_id": wp_post_id}`
  - status = `awaiting_approval`
- Return the created job

**7. `async def mark_audit_fixed(db: AsyncSession, site_id: UUID, page_url: str, check_code: str, job_id: UUID) -> None`**
- Update the AuditResult for (site_id, page_url, check_code):
  - status = "fixed"
  - details = f"Fixed via pipeline job {job_id} on {datetime.now()}"
  - wp_content_job_id = job_id
</action>

<title>Add fix endpoints to audit router</title>
<read_first>

</read_first>
<action>
Add fix endpoints to `app/routers/audit.py`:

**14. `POST /{site_id}/fix/preview`** — Generate a fix preview without applying.
- Body: `{"page_url": str, "fix_action": str}` (fix_action: `inject_toc`, `inject_schema`, `inject_cta`, `inject_links`)
- Fetches page HTML from WP
- Calls the appropriate `generate_*_fix()` function
- For `inject_schema`: gets template via `schema_service.render_schema_for_page()`
- For `inject_cta`: uses `site.cta_template_html`
- Returns JSON: `{"preview": processed_html_snippet, "diff": diff_dict, "valid": bool, "warnings": []}` (after running `verify_html_integrity`)
- Does NOT save anything yet

**15. `POST /{site_id}/fix/apply`** — Apply a fix (create pipeline job).
- Body: `{"page_url": str, "fix_action": str, "wp_post_id": int | None}`
- Fetches HTML, generates fix, verifies integrity
- If valid: calls `create_fix_job()` → job in `awaiting_approval` status
- Marks audit result as "fixed" with job reference
- Returns JSON: `{"job_id": str, "status": "awaiting_approval"}`
- The existing pipeline approve/push flow handles the rest

**16. `POST /{site_id}/fix/apply-and-approve`** — Apply + auto-approve + push (shortcut).
- Same as apply, but also sets job to approved and dispatches `push_to_wp.delay()`
- Returns `{"job_id": str, "status": "approved", "push_task_id": str}`

Note: the user flow is:
1. See failed check in audit table → click "Исправить"
2. System shows preview with diff (endpoint 14)
3. User clicks "Применить" (endpoint 15 or 16)
4. Job appears in pipeline → approve → push (or auto-approve via endpoint 16)
5. Audit result updated to "fixed"
</action>

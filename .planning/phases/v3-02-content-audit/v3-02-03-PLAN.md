# Plan v3-02-03: Schema template engine and CRUD

<objective>
Create the schema template service: render JSON-LD from user-customizable templates with page data placeholders, CRUD for templates (create from default, customize per site, delete custom), and functions to select the right template for a page based on content_type and site overrides.
</objective>

Create `app/services/schema_service.py`:

### Pure functions (no DB)

**1. `render_schema_template(template_json: str, page_data: dict) -> str`**
Takes a JSON-LD template string with `{{placeholder}}` markers and a dict of page data.
- Replace all `{{key}}` with `page_data.get(key, "")` 
- Use simple string replacement (not Jinja2 — templates are JSON strings, not HTML)
- Parse the result as JSON to validate, then return as pretty-printed JSON string
- If JSON parse fails (e.g. missing required placeholder), return the raw replaced string with a warning logged

**2. `generate_schema_tag(rendered_json: str) -> str`**
Wrap rendered JSON in `<script type="application/ld+json">...</script>` tag.

**3. `get_page_data_for_schema(title: str, url: str, description: str = "", site_name: str = "", author: str = "Author", date_published: str = "") -> dict`**
Build a standard dict of page data fields that templates can reference:
```python
{
    "title": title,
    "url": url,
    "description": description,
    "site_name": site_name,
    "author": author,
    "date_published": date_published,
}
```

### Schema type selection

**4. `def select_schema_type_for_page(content_type: str, page_type: str) -> str`**
Decide which schema type to use:
- `informational` + `article` → `Article`
- `commercial` + `product` → `Product`
- `commercial` + `landing` → `Service`
- `commercial` + `category` → `LocalBusiness`
- Default → `Article`

### Async DB functions

**5. `async def get_template(db: AsyncSession, site_id: UUID | None, schema_type: str) -> SchemaTemplate | None`**
Get the most specific template for a site+type:
- First try: site-specific template (site_id + schema_type)
- Fallback: default template (site_id IS NULL + schema_type + is_default=True)
- Returns None if neither exists

**6. `async def get_all_templates(db: AsyncSession, site_id: UUID | None = None) -> list[SchemaTemplate]`**
Get all templates: defaults + site-specific. Order by schema_type.

**7. `async def create_site_template(db: AsyncSession, site_id: UUID, schema_type: str, name: str, template_json: str) -> SchemaTemplate`**
Create a site-specific template. If one already exists for this site+type, update it (upsert).

**8. `async def delete_site_template(db: AsyncSession, template_id: UUID) -> bool`**
Delete a site-specific template. Refuse to delete system defaults (is_default=True). Returns True if deleted.

**9. `async def reset_to_default(db: AsyncSession, site_id: UUID, schema_type: str) -> bool`**
Delete the site-specific override for a type, effectively resetting to the system default.

**10. `async def render_schema_for_page(db: AsyncSession, site_id: UUID, page_data: dict, content_type: str, page_type: str) -> str | None`**
High-level function: select schema type → get template → render → wrap in script tag.
Returns the complete `<script>` tag or None if no template found.

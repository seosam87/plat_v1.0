<action>
Create `app/routers/audit.py`:

Router: `prefix="/audit"`, tags `["audit"]`

**Page endpoints:**

1. `GET /{site_id}` — Main audit page (HTML). Fetches:
   - Site by ID (404 if not found)
   - Latest crawl pages for the site (deduplicated by URL, latest crawl_job only)
   - Audit results for those pages
   - Active check definitions
   - Template: `audit/index.html`
   - Query params: `search` (filter by URL/title), `content_type` (filter: all/informational/commercial/unknown), `status` (filter: all/has_issues/all_pass), `page` (pagination, 50 per page)

2. `POST /{site_id}/run` — Run audit on all pages (or filtered subset).
   - Fetches rendered HTML for each page from WP REST API (using existing `_fetch_wp_content` pattern from `wp_content_tasks.py`)
   - Runs `classify_and_update_pages()` to set content_type
   - Runs `run_checks_for_page()` per page
   - Saves results via `save_audit_results()`
   - Returns JSON `{"pages_audited": N, "issues_found": N}`
   - This is a Celery task dispatch (not inline) — see task id 02

3. `POST /{site_id}/run-single` — Run audit on a single page URL.
   - Body: `{"page_url": str}`
   - Fetches HTML, runs checks, saves results
   - Returns JSON with check results for that page

4. `PUT /{site_id}/pages/{page_id}/content-type` — Manually set content_type for a page.
   - Body: `{"content_type": "informational"|"commercial"|"unknown"}`
   - Calls `update_content_type()`
   - Returns `{"status": "updated"}`

**Check definition endpoints:**

5. `GET /checks` — List all check definitions (JSON).
6. `POST /checks` — Create a new check definition. Body: `{code, name, applies_to, severity, auto_fixable, fix_action}`.
7. `PUT /checks/{check_id}` — Update check definition fields. Body: partial update.
8. `DELETE /checks/{check_id}` — Soft-delete (set is_active=False) a check definition.

**Schema template endpoints:**

9. `GET /{site_id}/templates` — List all schema templates (defaults + site-specific).
10. `POST /{site_id}/templates` — Create/update site-specific template. Body: `{schema_type, name, template_json}`.
11. `DELETE /{site_id}/templates/{template_id}` — Delete site-specific template (reset to default).

**CTA template endpoint:**

12. `PUT /{site_id}/cta` — Save CTA template HTML for site.
    - Body: `{"cta_html": str}`
    - Updates `site.cta_template_html`
    - Returns `{"status": "saved"}`

**Audit results endpoint:**

13. `GET /{site_id}/results` — Get audit results as JSON (for HTMX partial refresh).
    - Query params: `page_url` (optional filter)
    - Returns list of audit result dicts

Helper: `_get_site_or_404(db, site_id)` — same pattern as metrika router.

<action>
Create `app/tasks/audit_tasks.py`:

**Task: `run_site_audit`**
```python
@celery_app.task(
    name="app.tasks.audit_tasks.run_site_audit",
    bind=True,
    max_retries=2,
    queue="wp",
    soft_time_limit=300,
    time_limit=360,
)
def run_site_audit(self, site_id: str) -> dict:
```

Steps:
1. Get site from DB, validate active
2. Get latest crawl pages (deduplicated by URL)
3. Get active check definitions
4. Classify content_type for all pages
5. For each page (up to 200):
   - Fetch rendered HTML from WP REST API (using `_fetch_wp_content` pattern)
   - Run `run_checks_for_page(html, page_data, checks)`
   - Save results
6. Return `{"status": "done", "pages_audited": N, "issues_found": N}`

Uses `asyncio.new_event_loop()` pattern (same as `metrika_tasks.py`).

Register in `app/celery_app.py` include list: add `"app.tasks.audit_tasks"`.
</action>

**2. Filter bar** (card):
- Search input: `<input type="text" placeholder="Поиск по URL или title..." id="search-input">`
- Content type select: `<select id="type-filter"><option value="all">Все типы</option><option value="informational">Информационные</option><option value="commercial">Коммерческие</option><option value="unknown">Не определён</option></select>`

**3. Summary stats row** (4 mini-cards):
- Всего страниц: `{{ pages | length }}` (blue #2563eb)
- С проблемами: count of pages with at least one fail/warning (red #dc2626)
- Без проблем: count of pages with all pass (green #059669)
- Не проверено: count of pages with no audit results (gray #6b7280)

**4. Pages table:**
- Columns: Страница | Тип | TOC | Schema | Автор | Статьи | CTA | Ссылки | noindex | Действия

- Each check column: green checkmark (✓) if pass, red cross (✗) if fail, yellow warning (⚠) if warning, dash (—) if not applicable
- Content type cell: clickable to change (inline select via HTMX PUT)
- Actions: "Проверить" button (run single page audit)

**5. Check definitions card** (collapsible at bottom):
- Table of all check definitions: Code | Название | Тип | Серьёзность | Авто-фикс | Активен | Действия

- "Добавить проверку" form

**6. CTA template card:**
- `<textarea>` for CTA HTML (pre-filled from `site.cta_template_html`)
- "Сохранить CTA" button: calls `PUT /audit/{{ site.id }}/cta`

**7. Schema templates card:**
- List of templates (defaults + site-specific)

- "Создать шаблон" button → modal with type select, name input, JSON textarea

**8. JS functions:**

- `runSingleAudit(pageUrl)` — POST to `/audit/{site_id}/run-single`

- `updateContentType(pageId, value)` — PUT content_type

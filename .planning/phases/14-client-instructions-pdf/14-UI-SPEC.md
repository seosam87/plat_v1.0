# Phase 14 — UI Design Contract: Client Instructions PDF

> Visual and interaction contract for the "Клиентские отчёты" feature.
> Generated by gsd-ui-researcher. Source: codebase scan + 14-CONTEXT.md + REQUIREMENTS.md.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Jinja2 + Tailwind CSS CDN (existing project pattern) |
| Preset | not applicable — no shadcn; Tailwind via `https://cdn.tailwindcss.com` in base.html |
| Component library | none — inline CSS + Tailwind utility classes |
| Icon library | Heroicons inline SVG (stroke-width 1.5, existing sidebar pattern) |
| Font | system-ui, sans-serif (base.html body declaration) |

shadcn gate: not applicable — stack is Python/FastAPI/Jinja2/HTMX, not React/Next.js/Vite.

**Two rendering contexts exist in this phase:**

| Context | Technology | Tailwind? |
|---------|-----------|-----------|
| Web UI page (`/ui/client-reports/`) | Jinja2 + Tailwind CDN + HTMX | Yes |
| PDF document (WeasyPrint output) | Jinja2 + inline CSS only | No — WeasyPrint cannot resolve CDN |

All PDF-specific styles MUST be inline CSS within the PDF template. Do not reference Tailwind classes or external CDN URLs in the PDF template.

## Spacing Scale

Standard 8-point scale applied to web UI. PDF uses pt-unit equivalents.

### Web UI Spacing

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1 / gap-1) | Icon-to-label gaps, badge inner padding |
| sm | 8px (p-2 / gap-2) | Compact row padding, checkbox groups |
| md | 16px (p-4 / gap-4) | Card inner padding default, form field spacing |
| lg | 24px (p-6 / gap-6) | Card outer padding, section vertical gap |
| xl | 32px (mb-8 / gap-8) | Page section breaks |
| 2xl | 48px | History table top margin from generation form |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Sidebar width: 16rem (256px) — inherited, do not alter
- Main content left margin: 16rem — inherited from `.main-content`
- History table rows: `padding: 0.75rem 1rem` (12px / 16px) — matches existing `th, td` base rule

### PDF Spacing (pt units, A4 at 96dpi)

| Role | Value | Usage |
|------|-------|-------|
| Page margin | 2cm (all sides) | `@page { margin: 2cm }` — matches brief.html |
| Section gap | 18px top margin | Between h2 sections — matches brief.html `h2 { margin: 18px 0 8px 0 }` |
| Table cell padding | 5–6px / 8px | `td { padding: 5px 8px }` / `th { padding: 6px 8px }` — matches brief.html |
| Summary block padding | 12px | Inset box for summary section |

## Typography

### Web UI Typography

Source: base.html computed defaults + Tailwind classes observed in generate.html.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 | Table cells, form labels, description text |
| Label | 14px (text-sm) | 600 (font-semibold) | 1.4 | Column headers (th), card section headings, checkbox labels |
| Heading | 20px (text-xl) | 700 (font-bold) | 1.3 | Page h1 — "Клиентские отчёты — {site_name}" |
| Sub-heading | 16px (text-base) | 600 (font-semibold) | 1.3 | Card h2 — "Выберите блоки", "История отчётов" |

Two weights only: 400 (regular) and 600–700 (semibold/bold).

### PDF Typography

Source: brief.html inline CSS — adapted for 4-size constraint.

| Role | Size | Weight | Usage |
|------|------|--------|-------|
| Report title (h1) | 18pt | 700 | Report cover title |
| Section heading (h2) | 13pt | 600 | Problem-type group headers |
| Body text | 11pt | 400 | Instructions, page lists |
| Small — meta, caption, footer | 10pt | 400 | Table cells, dates, URLs, page footer, secondary badges |

Four sizes only: 18pt / 13pt / 11pt / 10pt. The former 9pt footer tier is merged into the 10pt small tier; footer CSS uses `font-size: 10pt`.

Font: Arial, Helvetica, sans-serif — matches brief.html. No web fonts; WeasyPrint resolves system fonts only.

Line heights: body 1.5, headings 1.2 (existing brief.html values).

## Component Inventory

### Web UI Components (Jinja2 + Tailwind)

All components use existing project patterns. No new component library introduced.

#### 1. Sidebar Navigation Entry

- Location: `app/templates/components/sidebar.html`
- New section "Клиентские отчёты" added after existing "Отчёты" section
- Icon: `document-check` (Heroicons, 1.25rem, stroke-width 1.5, stroke currentColor)
- Section type: collapsible, with single child "Генерация"
- Child URL: `/ui/client-reports/`
- Active color: `#4338ca` background on child when active

SVG path for `document-check` icon:
```
M9 12.75 11.25 15 15 9.75M21 12a9 9 0 1 1-18 0 9 9 0 0 1 18 0Z
```
Use: `<path stroke-linecap="round" stroke-linejoin="round" d="M9 12.75 11.25 15 15 9.75M21 12a9 9 0 1 1-18 0 9 9 0 0 1 18 0Z" />`

#### 2. Page Layout

- Extends `base.html`
- Page title (h1, text-2xl font-bold text-gray-900 mb-6): "Клиентские отчёты — {site_name}"
- Breadcrumbs: Dashboard / Клиентские отчёты
- Max content width: max-w-4xl (two column card layout)

#### 3. Generation Form Card

- Wrapper: `.card` (existing class — white bg, 8px radius, 1.5rem padding, shadow)
- Card heading (h2, text-lg font-semibold text-gray-800 mb-4): "Настройка отчёта"
- Site selector: full-width `<select>` using existing `.site-selector select` CSS class
- Block checkboxes: 2-column grid (`grid grid-cols-2 gap-3`), each checkbox with label
- TOP-N display note: static text below each checked block — "Топ-20 по opportunity_score"
- CTA button: `.btn .btn-primary` class + `bg-indigo-600 hover:bg-indigo-700` — full Tailwind override
- Progress indicator: `htmx-indicator` class, spinner SVG inline, visible only during `hx-request`

Checkbox block labels (Russian):
- `quick_wins` — "Quick Wins (позиции 4–20)"
- `audit_errors` — "Ошибки аудита (по impact_score)"
- `dead_content` — "Мёртвый контент"
- `positions` — "Статистика позиций"

Default checked: all four blocks.

#### 4. Delivery Options Row

- Appears below CTA, only after a report is generating or complete
- Three inline buttons: "Скачать PDF" (accent), "Отправить на email" (gray), "Отправить в Telegram" (gray)
- Each button: `inline-flex items-center gap-2 px-4 py-2 text-sm font-medium rounded-lg`
- Email and Telegram show confirmation toast on success — no modal required

#### 5. History Table

- Card heading (h2): "История отчётов"
- Table columns: Дата | Сайт | Блоки | Действия
- "Блоки" column: comma-separated Russian short-form labels (QW, Ошибки, Мёртвый, Позиции)
- "Действия" column: "Скачать" link (accent text, no button border) + "Email" icon button + "Telegram" icon button
- Email icon button: icon-only; MUST include `aria-label="Отправить на email"` — no visible text
- Telegram icon button: icon-only; MUST include `aria-label="Отправить в Telegram"` — no visible text
- Empty state: see Copywriting Contract below
- Table uses existing `th, td` base.html rules

#### 6. Toast Notifications

- Reuse existing `#toast-container` + `.toast` + `.toast-success/.toast-error/.toast-info` classes
- Trigger via HTMX `HX-Trigger` response header from backend — no JS required beyond existing pattern

Content:

#### PDF Summary Box (new, not in brief.html)

```css
.summary-box {
  background: #f3f4f6;
  border-left: 3px solid #6366f1;
  padding: 12px 16px;
  margin: 16px 0 24px 0;
  font-size: 10pt;
}
.summary-box p { margin: 4px 0; }
```

Content: проверено страниц / найдено проблем / критических / общая оценка (1 sentence).

#### PDF Problem Group Section

```css
h2 { font-size: 13pt; color: #1e1b4b; margin: 18px 0 8px 0;
     border-bottom: 1px solid #e5e7eb; padding-bottom: 4px; }
.instruction-block { margin-bottom: 8px; font-size: 11pt; }
.instruction-block strong { font-weight: 600; }
```

Structure per problem type:
1. h2 — problem type name (Russian): "Отсутствует TOC", "Нет schema.org", etc.
2. Instruction paragraph (imperative tone per D-06)
3. Table of affected pages (URL | title | priority metric)
4. If items exceed TOP-N: `<p class="more-note">И ещё {N} страниц с аналогичной проблемой.</p>`

TOP-N limit: 20 items per block (Claude's discretion — D-02).

## Interaction Contract (HTMX)

| Trigger | HTMX Verb | Target | Swap |
|---------|-----------|--------|------|
| "Сгенерировать" button click | `hx-post="/ui/client-reports/generate"` | `#generation-status` div | `innerHTML` |
| Generation completes (polling) | `hx-get="/ui/client-reports/status/{task_id}"` every 3s | `#generation-status` | `outerHTML` |
| "Скачать PDF" in history | Direct `<a href>` download — no HTMX | — | — |
| "Email" delivery button (`aria-label="Отправить на email"`) | `hx-post="/ui/client-reports/{id}/send-email"` | `#toast-container` | `beforeend` |
| "Telegram" delivery button (`aria-label="Отправить в Telegram"`) | `hx-post="/ui/client-reports/{id}/send-telegram"` | `#toast-container` | `beforeend` |
| History table load | `hx-get="/ui/client-reports/history"` on page load or after generation | `#history-section` | `innerHTML` |

HTMX 2.0.x — no `hx-ws` or `hx-sse` attributes (moved to extensions in 2.0; not needed here).

## States per UI Element

### Generation Form

| State | Visual |
|-------|--------|
| Default | All 4 checkboxes checked; "Сгенерировать PDF" button enabled, indigo |
| No site selected | Button disabled (opacity-50, cursor-not-allowed) |
| No blocks checked | Button disabled — enforce min 1 block via `required` attribute on checkbox group |
| Generating | Button replaced with spinner + "Генерируется..." text; inputs disabled |
| Complete | Status div shows success with download link; inputs re-enabled |
| Failed | Status div shows error copy; inputs re-enabled; retry allowed |

### History Table Row

| State | Visual |
|-------|--------|
| PDF ready | "Скачать PDF" link visible (accent color) |
| Email sending | "Отправить на email" button (`aria-label="Отправить на email"`) shows spinner inline |
| Telegram sending | "Отправить в Telegram" button (`aria-label="Отправить в Telegram"`) shows spinner inline |

Implements subprocess-isolated WeasyPrint rendering per D-12.

```python
import json
import subprocess
import sys
import tempfile
from pathlib import Path
from loguru import logger

def render_pdf_in_subprocess(html_string: str, timeout_seconds: int = 120) -> bytes:
    """Render HTML to PDF via WeasyPrint in a separate process.

    The child process imports weasyprint, renders, writes to a temp file,
    and exits — freeing all leaked memory. The parent reads the file and
    returns PDF bytes.

    Args:
        html_string: Full HTML document string.
        timeout_seconds: Max seconds before killing the child process.

    Returns:
        PDF file bytes.

    Raises:
        RuntimeError: If subprocess fails or times out.
    """
    # Write HTML to temp file (avoid pipe size limits)
    with tempfile.NamedTemporaryFile(suffix=".html", delete=False, mode="w", encoding="utf-8") as html_f:
        html_f.write(html_string)
        html_path = html_f.name

    pdf_path = html_path.replace(".html", ".pdf")

    # Child script: import weasyprint, render, exit
    script = f"""
import weasyprint
doc = weasyprint.HTML(filename={html_path!r})
doc.write_pdf({pdf_path!r})
"""

    try:
        result = subprocess.run(
            [sys.executable, "-c", script],
            capture_output=True,
            text=True,
            timeout=timeout_seconds,
        )
        if result.returncode != 0:
            logger.error("PDF subprocess failed", stderr=result.stderr[:500])
            raise RuntimeError(f"PDF render failed: {result.stderr[:300]}")

        pdf_bytes = Path(pdf_path).read_bytes()
        return pdf_bytes
    except subprocess.TimeoutExpired:
        raise RuntimeError(f"PDF render timed out after {timeout_seconds}s")
    finally:
        # Clean up temp files
        Path(html_path).unlink(missing_ok=True)
        Path(pdf_path).unlink(missing_ok=True)
```

This approach uses `subprocess.run` with a Python `-c` script. The child process imports weasyprint, renders, writes to disk, and exits. All WeasyPrint memory is freed when the child exits. The parent reads the PDF bytes from disk.

  <action>
    Create app/static/js/tour.js as a plain ES2020 IIFE script (no imports, no bundler). Use the reference implementation in RESEARCH Pattern 4 almost verbatim, with the following clarifications and fixes:

    1. IIFE wrapper: wrap everything in (function(){ 'use strict'; ... })(); and define const RESUME_KEY = 'gsd_tour_resume_v1'; at the top.

    2. resolveSelector(target) — mirror locators.py prefixes. Returns a CSS selector string or null:
       - target.startsWith('css=') → return target.slice(4)
       - target.startsWith('testid=') → return '[data-testid="' + target.slice(7) + '"]'
       - target.startsWith('role=') → use regex /^role=(\w+)(?:\[name=["'](.+?)["']\])?$/. If no name group: return '[role="X"]'. If name group present: return '[role="X"][aria-label="N"], button[aria-label="N"]' (fallback chain because Shepherd needs a selector, not accessible-name lookup).
       - target.startsWith('text=') → console.warn('[tour] text= locator not supported in tour mode; use css= or role=', target); return null.
       - target.startsWith('label=') → same warn + null.
       - Anything else → return target as-is (raw CSS fallback).

    3. buildTour(scenario, startAtIndex) — constructs a Shepherd.Tour with useModalOverlay:true and defaultStepOptions:{ scrollTo:{behavior:'smooth',block:'center'}, cancelIcon:{enabled:true}, classes:'shepherd-theme-arrows' }.

       Iterate scenario.steps with forEach((step, i) => ...). Skip indexes < (startAtIndex||0). Maintain a lastSayText variable initialised to scenario.tour.intro || '' so highlight/wait_for_click steps reuse the previous say text as the tooltip body.

       Op handlers (exact mapping from D-02):
       - say: tour.addStep with id 's'+i, text: step.text, buttons:[{text:'Далее', action: tour.next}]. Update lastSayText = step.text.
       - highlight: resolve sel; addStep with text: lastSayText, attachTo: sel ? {element: sel, on: 'bottom'} : undefined, buttons:[{text:'Далее', action: tour.next}].
       - wait_for_click: resolve sel; addStep with text: lastSayText, attachTo: sel ? {element: sel, on: 'bottom'} : undefined, advanceOn: sel ? {selector: sel, event: 'click'} : undefined. NO buttons key — advancing is the user click.
       - open: addStep with text: lastSayText || 'Переходим на следующую страницу...', buttons:[{text:'Продолжить', action: function(){ sessionStorage.setItem(RESUME_KEY, JSON.stringify({name: scenario.name, index: i+1})); window.location.assign(step.url); }}].
       - default (test-only ops): console.warn('[tour] skipping test-only op:', step.op) — do NOT add a Shepherd step.

       After the loop: tour.on('complete', function(){ sessionStorage.removeItem(RESUME_KEY); }); tour.on('cancel', function(){ sessionStorage.removeItem(RESUME_KEY); }); return tour.

    4. startTour(name, startAtIndex) — async. fetch('/api/tours/' + encodeURIComponent(name)). If !res.ok, console.error and return. Else parse JSON and call buildTour(scenario, startAtIndex || 0).start().

    5. bootstrap() — async:
       - First, check sessionStorage.getItem(RESUME_KEY). If present, try JSON.parse to {name, index}, call startTour(name, index), and return early. On parse error, sessionStorage.removeItem(RESUME_KEY) and continue.
       - Otherwise, fetch('/api/tours/?page=' + encodeURIComponent(location.pathname)). On !res.ok, return silently (non-admins are server-gated anyway; belt-and-braces). Parse JSON. If tours.length === 0, return. Else call renderShowTourButton(tours).

    6. renderShowTourButton(tours) — create a button element with inline styles (do NOT rely on Tailwind utility classes — Tailwind is JIT CDN and ordering is not guaranteed). Use exactly this style string:
       position:fixed;bottom:1.5rem;right:1.5rem;z-index:100;background:#4f46e5;color:#fff;border:none;border-radius:9999px;padding:.75rem 1.25rem;box-shadow:0 4px 12px rgba(0,0,0,.15);cursor:pointer;font-size:.9rem;
       Text content: "🎓 Показать тур". On click: if tours.length === 1, call startTour(tours[0].name); else use window.prompt() listing titles as "1. Title\n2. Title\n..." and parse a 1-based index (acceptable v1 UI — CONTEXT deferred the real picker). Append button to document.body.

       z-index note: site log panel is z-index:50, sidebar is 40, toast-container is 999. Using 100 keeps the button above log/sidebar and below toasts — the intended ordering.

    7. DOMContentLoaded wiring — handle the defer-after-ready case:
       if (document.readyState === 'loading') { document.addEventListener('DOMContentLoaded', bootstrap); } else { bootstrap(); }

    Target size: roughly 180-220 lines including comments. No external imports, no require, no ESM — must work as a raw script tag.

    Anti-patterns to avoid (from RESEARCH):
    - Do NOT parse YAML in the browser.
    - Do NOT use HTMX boost for the open op — window.location.assign is required so DOMContentLoaded fires fresh and the resume handler runs.
    - Do NOT try to implement text/label accessible-name lookup in JS — warn and skip.

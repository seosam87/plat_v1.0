# Phase 19.2 Plan 03: Tours API Router Summary

**One-liner:** Admin-gated FastAPI router serving tour YAML as JSON via one-shot lru_cache loader, with mock-based pytest coverage for list/detail/auth contracts.

## What Was Built

### Task 1: app/routers/tours.py

FastAPI router at prefix `/api/tours` with two endpoints:

- `GET /api/tours/?page=<pathname>` — returns JSON array of `{name, title, intro}` for tours whose `entry_url` is a prefix of the requested page. Requires admin via `Depends(require_admin)`.
- `GET /api/tours/{name}` — returns the full scenario JSON (name, description, tour metadata, steps). Returns 404 if not found. Requires admin.

Key implementation details:
- `_load_all_tours()` decorated with `@lru_cache(maxsize=1)` — loads all `scenarios/*.yaml` files once per uvicorn process lifetime. New tours require a restart.
- Scenarios without a `tour:` block are filtered out (never advertised by either endpoint).
- Malformed YAML files are silently skipped with `try/except` — the 19.1 runner owns schema validation error reporting; the tour API does not duplicate that.
- `SCENARIOS_DIR = Path(__file__).resolve().parents[2] / "scenarios"` — resolves to project root `/opt/seo-platform/scenarios`.

### Task 2: main.py registration + tests/test_tours_api.py

Router registered in `app/main.py`:
```python
from app.routers.tours import router as tours_router
app.include_router(tours_router)
```

Tests (3 cases, all passing):
1. `test_list_tours_empty_or_ok` — admin GET returns 200 with a list (empty if no tour YAMLs yet).
2. `test_list_tours_forbidden_for_non_admin` — unauthenticated request returns 401 or 403.
3. `test_get_tour_not_found` — admin GET for nonexistent tour returns 404.

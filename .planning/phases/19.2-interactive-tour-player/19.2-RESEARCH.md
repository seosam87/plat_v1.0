## Summary

Phase 19.2 is small and well-scoped: a single JS file (~250 LOC), a single FastAPI router (~80 LOC), a one-field Pydantic schema addition, a script-tag injection in `base.html`, plus at least one authored tour YAML. The only real architectural decision is **where the schema lives** — the current `tests/fixtures/scenario_runner/schema.py` is NOT importable from `app/` because `tests/` is not on the Python import path. **Recommendation: move `schema.py` to `app/services/scenario_schema.py` and have `tests/fixtures/scenario_runner/schema.py` re-export from it.** This is the cleanest option, keeps 19.1 tests untouched, and makes the server-side tour API a trivial import.

Second non-obvious finding: **there is no `StaticFiles` mount and no `app/static/` directory today.** This phase must add both. Third: **`is_admin` is already computed in `app/template_engine.py` (line 97-105)** but not exposed as a Jinja context variable — the cleanest fix is to add `ctx["is_admin"] = is_admin` two lines later, so `base.html` can do `{% if is_admin %}` server-side and skip the `/api/tours/` fetch entirely for non-admins.

**Primary recommendation:** 4 waves, 5 plans. Wave 0 (schema + static infra, parallelizable), Wave 1 (API + JS player + base.html bootstrap, parallelizable), Wave 2 (first real tour YAML + scenarios/README.md update), Wave 3 (smoke verification via existing 19.1 runner).

### Shepherd.js API cheat-sheet (verified against docs)
```js
const tour = new Shepherd.Tour({
  useModalOverlay: true,
  defaultStepOptions: {
    scrollTo: { behavior: 'smooth', block: 'center' },
    cancelIcon: { enabled: true },
    classes: 'shepherd-theme-arrows',
  },
});
tour.addStep({
  id: 'step-1',
  text: 'Some say text',
  attachTo: { element: '#seed-input', on: 'bottom' }, // null = centered modal
  advanceOn: { selector: '#submit-btn', event: 'click' }, // replaces Next button
  buttons: [{ text: 'Далее', action: tour.next }],       // omit if advanceOn used
});
tour.start();
```
- `attachTo.element` accepts a CSS selector string OR an HTMLElement.
- `advanceOn` takes a **CSS selector** + event name — so the tour.js locator resolver must produce a CSS selector, not an Element.
- Keyboard nav (Esc to cancel, arrow keys next/prev) is built-in. Do not reimplement.
- `scrollTo` is built-in — `highlight` op just maps to `attachTo`.

### Pattern 3: FastAPI Router (mirrors `app/routers/admin.py`)
```python
# app/routers/tours.py
from pathlib import Path
from functools import lru_cache
from fastapi import APIRouter, Depends, HTTPException, Query
import yaml
from app.auth.dependencies import require_admin
from app.models.user import User
from app.services.scenario_schema import Scenario

router = APIRouter(prefix="/api/tours", tags=["tours"])
SCENARIOS_DIR = Path(__file__).resolve().parents[2] / "scenarios"


@lru_cache(maxsize=1)
def _load_all_tours() -> list[Scenario]:
    out = []
    for yaml_file in sorted(SCENARIOS_DIR.glob("*.yaml")):
        raw = yaml.safe_load(yaml_file.read_text())
        sc = Scenario.model_validate(raw)
        if sc.tour is not None:
            out.append(sc)
    return out


@router.get("/")
async def list_tours(page: str = Query(...), _: User = Depends(require_admin)):
    return [
        {"name": t.name, "title": t.tour.title, "intro": t.tour.intro}
        for t in _load_all_tours()
        if page.startswith(t.tour.entry_url)
    ]


@router.get("/{name}")
async def get_tour(name: str, _: User = Depends(require_admin)):
    for t in _load_all_tours():
        if t.name == name:
            return t.model_dump()
    raise HTTPException(404, "Tour not found")
```
**Cache invalidation:** `lru_cache` means a uvicorn restart is required to pick up new tour YAML files. Acceptable for production (tours ship via git + redeploy). For dev, planner may add a `?nocache=1` bypass or simply restart uvicorn.

### Pattern 4: `tour.js` Structure (~250 LOC)
```js
// app/static/js/tour.js  — plain ES2020, no imports, no build step
(function () {
  const RESUME_KEY = 'gsd_tour_resume_v1';  // {tour_name, step_index}

  // ── Locator resolver (JS mirror of tests/fixtures/scenario_runner/locators.py) ──
  function resolveSelector(target) {
    // Returns a CSS selector string (Shepherd's advanceOn/attachTo both accept CSS).
    if (target.startsWith('css=')) return target.slice(4);
    if (target.startsWith('testid=')) return `[data-testid="${target.slice(7)}"]`;
    if (target.startsWith('role=')) {
      // Minimal support: role=button[name="X"] → [role=button][aria-label="X"] fallback.
      const m = target.match(/^role=(\w+)(?:\[name=["'](.+?)["']\])?$/);
      if (!m) return target;
      if (m[2]) return `[role="${m[1]}"][aria-label="${m[2]}"], button[aria-label="${m[2]}"]`;
      return `[role="${m[1]}"]`;
    }
    if (target.startsWith('text=')) {
      // Shepherd cannot bind to text content via selector. Author tours with CSS.
      console.warn('[tour] text= locator not supported in tours; use css= or role=', target);
      return null;
    }
    if (target.startsWith('label=')) {
      console.warn('[tour] label= locator not supported in tours; use css= or role=', target);
      return null;
    }
    return target;  // raw CSS
  }

  // ── Build Shepherd Tour from parsed scenario JSON ──
  function buildTour(scenario, startAtIndex) {
    const tour = new Shepherd.Tour({
      useModalOverlay: true,
      defaultStepOptions: {
        scrollTo: { behavior: 'smooth', block: 'center' },
        cancelIcon: { enabled: true },
        classes: 'shepherd-theme-arrows',
      },
    });
    let lastSayText = scenario.tour.intro || '';
    scenario.steps.forEach((step, i) => {
      if (i < (startAtIndex || 0)) return;
      switch (step.op) {
        case 'say':
          lastSayText = step.text;
          tour.addStep({
            id: `s${i}`, text: step.text,
            buttons: [{ text: 'Далее', action: tour.next }],
          });
          break;
        case 'highlight': {
          const sel = resolveSelector(step.target);
          tour.addStep({
            id: `s${i}`, text: lastSayText,
            attachTo: sel ? { element: sel, on: 'bottom' } : undefined,
            buttons: [{ text: 'Далее', action: tour.next }],
          });
          break;
        }
        case 'wait_for_click': {
          const sel = resolveSelector(step.target);
          tour.addStep({
            id: `s${i}`, text: lastSayText,
            attachTo: sel ? { element: sel, on: 'bottom' } : undefined,
            advanceOn: sel ? { selector: sel, event: 'click' } : undefined,
            // no buttons — advancing happens on user click
          });
          break;
        }
        case 'open': {
          // Persist resume state, then navigate.
          tour.addStep({
            id: `s${i}`, text: lastSayText || 'Переходим на следующую страницу...',
            buttons: [{
              text: 'Продолжить', action: () => {
                sessionStorage.setItem(RESUME_KEY, JSON.stringify({
                  name: scenario.name, index: i + 1,
                }));
                window.location.assign(step.url);
              },
            }],
          });
          break;
        }
        default:
          console.warn('[tour] skipping test-only op:', step.op);
      }
    });
    tour.on('complete', () => sessionStorage.removeItem(RESUME_KEY));
    tour.on('cancel', () => sessionStorage.removeItem(RESUME_KEY));
    return tour;
  }

  async function startTour(name, startAtIndex) {
    const res = await fetch(`/api/tours/${encodeURIComponent(name)}`);
    if (!res.ok) { console.error('[tour] fetch failed', res.status); return; }
    const scenario = await res.json();
    buildTour(scenario, startAtIndex || 0).start();
  }

  async function bootstrap() {
    // Resume after navigation?
    const resumeRaw = sessionStorage.getItem(RESUME_KEY);
    if (resumeRaw) {
      try {
        const { name, index } = JSON.parse(resumeRaw);
        startTour(name, index);
        return;
      } catch (_) { sessionStorage.removeItem(RESUME_KEY); }
    }
    // Check for matching tours on this page.
    const res = await fetch(`/api/tours/?page=${encodeURIComponent(location.pathname)}`);
    if (!res.ok) return;
    const tours = await res.json();
    if (tours.length === 0) return;
    renderShowTourButton(tours);
  }

  function renderShowTourButton(tours) {
    const btn = document.createElement('button');
    btn.textContent = '🎓 Показать тур';
    btn.style.cssText = 'position:fixed;bottom:1.5rem;right:1.5rem;z-index:100;background:#4f46e5;color:#fff;border:none;border-radius:9999px;padding:.75rem 1.25rem;box-shadow:0 4px 12px rgba(0,0,0,.15);cursor:pointer;font-size:.9rem;';
    btn.onclick = () => {
      if (tours.length === 1) startTour(tours[0].name);
      else {
        // Minimal picker — or defer per CONTEXT.md if only one tour ships.
        const name = prompt('Выберите тур:\n' + tours.map((t, i) => `${i + 1}. ${t.title}`).join('\n'));
        const idx = parseInt(name, 10) - 1;
        if (tours[idx]) startTour(tours[idx].name);
      }
    };
    document.body.appendChild(btn);
  }

  document.addEventListener('DOMContentLoaded', bootstrap);
})();
```

### Anti-Patterns to Avoid
- **Don't parse YAML in the browser.** Saves ~15KB of `js-yaml` CDN + keeps schema validation server-side where it belongs.
- **Don't use HTMX boost for tour navigation.** The `open` op must do a full `location.assign()` so that `sessionStorage` resume reliably fires on a fresh DOMContentLoaded. HTMX swaps do NOT fire DOMContentLoaded and the tour JS would never re-bootstrap. The CONTEXT question #8 about HTMX vs full load resolves cleanly: **always use full navigation for `open`**.
- **Don't add a `data-testid` migration to this phase.** Tours should author selectors with `css=` prefixes or plain CSS. `role=button[name="X"]` is the only non-CSS prefix supported in JS, and it's lossy (falls back to `aria-label` which may not exist).
- **Don't try to cache-bust `tour.js` via `?v=<hash>`** — `StaticFiles` serves with ETag by default. Fine.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Tooltip positioning | Custom popper | Shepherd.js `attachTo` | Handles viewport edges, scroll, resize |
| Modal overlay | Custom dimming | Shepherd `useModalOverlay: true` | Correct z-index stacking, focus trap |
| Advance-on-click | Custom event listeners | Shepherd `advanceOn` | Handles cleanup on step change, cancel |
| YAML parsing in browser | Bundled js-yaml | Server-side `/api/tours/{name}` returning JSON | Reuses Pydantic validation |
| Text/label→element lookup in JS | Custom tree walker | Author tours with `css=` selectors | `document.querySelector` doesn't do accessible name lookup; Shepherd needs a CSS selector anyway |
| Resume state after navigation | Custom URL params or cookies | `sessionStorage` (2 fields: name + index) | Auto-cleared on tab close, per-tab isolation |

**Key insight:** Shepherd.js already solves 90% of what a "tour player" is. The 10% left is: (1) translating your YAML schema into its API, (2) the resume-after-`open` bridge, (3) the admin-gated bootstrap.

## Common Pitfalls

### Pitfall 1: Schema import from `app/` fails at uvicorn boot
**What goes wrong:** `ModuleNotFoundError: No module named 'tests'` when `app/routers/tours.py` tries `from tests.fixtures.scenario_runner.schema import Scenario`.
**Why it happens:** `tests/` has no `__init__.py` at the top level (standard pytest layout) and is not in `sys.path` in production. Pytest adds it via rootdir magic; uvicorn does not.
**How to avoid:** Apply the Pattern 1 schema relocation. Never import from `tests/` at runtime.

### Pitfall 2: Shepherd `advanceOn` binds at step-start, not at `tour.start()`
**What goes wrong:** If an `open` step navigates BEFORE a `wait_for_click` step, the click target may not exist in the DOM when the tour resumes → `advanceOn` selector resolves to nothing → tour hangs.
**How to avoid:** On resume, skip straight to the target step (`startAtIndex`). Shepherd re-evaluates `attachTo`/`advanceOn` selectors when a step is shown, not when it's added. Verified by the docs (`delay advanceOn` issue #1542) — selectors are re-evaluated per-step.
**Warning signs:** Tour opens, shows no tooltip, user can't click anywhere.

### Pitfall 3: `is_admin` not exposed to Jinja context
**What goes wrong:** `base.html` uses `{% if is_admin %}` but the variable is undefined — Jinja silently evaluates to false and non-admins get no surprise, but admins ALSO get no tour button.
**How to avoid:** Add `ctx["is_admin"] = is_admin` in `app/template_engine.py` right after line 105 (where `is_admin` is computed from the JWT cookie). One-line edit.

### Pitfall 4: `StaticFiles` mount order matters
**What goes wrong:** If `app.mount("/static", ...)` is added AFTER `app.include_router(...)` for a router that has `/static/*` routes (unlikely here but worth checking), the mount is shadowed.
**How to avoid:** Mount static BEFORE any `include_router` calls in `app/main.py`. Verified safe — none of the ~40 existing routers use `/static`.

### Pitfall 5: `lru_cache` + mutable `Scenario` objects
**What goes wrong:** Returning the same `Scenario` instance from multiple requests and mutating it causes cross-request contamination.
**How to avoid:** Never mutate the cached list. `model_dump()` in the endpoint always produces a fresh dict. Safe.

### Pitfall 6: Collector picks up tour-only YAML files and tries to run them
**What goes wrong:** `tests/fixtures/scenario_runner/collector.py` line 25 matches ALL `scenarios/*.yaml`. A tour-only file like `03-keyword-suggest-tour.yaml` containing only `say`/`highlight`/`wait_for_click`/`open` steps gets loaded into pytest. 19.1's executor skips reserved ops with warnings — `open` DOES execute in 19.1 — so the tour would navigate but do nothing else. Result: the pytest item "passes" (all steps skipped or no-ops). Not broken, but noisy.
**How to avoid (two options, planner picks):**
   1. **Collector-level skip (recommended):** In `ScenarioFile.collect()`, after `Scenario.model_validate(raw)`, check `if scenario.tour is not None: return` (yield nothing). Tour-only files become invisible to pytest. One-line change, backward compatible (19.1 test files have no `tour:` block).
   2. **Runtime no-op:** Let the current behavior stand — tour files produce a "passing" pytest item. Simpler but adds noise to test output.
**Recommendation:** Option 1. It's the cleaner contract: `tour: present → tour player owns it; tour: absent → 19.1 runner owns it.`

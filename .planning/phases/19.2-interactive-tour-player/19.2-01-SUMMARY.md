# Phase 19.2 Plan 01: Scenario schema relocation + TourMeta — Summary

Relocated the Pydantic scenario schema from `tests/fixtures/scenario_runner/schema.py` to `app/services/scenario_schema.py` (verbatim, plus a new `TourMeta` model and an optional `tour: Optional[TourMeta]` field on `Scenario`), left a one-line re-export shim behind so Phase 19.1 tests keep working, and taught the pytest collector to skip any YAML file with a `tour:` block.

## What changed

- **`app/services/scenario_schema.py`** (new, canonical). Copy of the 19.1 schema plus:
  - `TourMeta(BaseModel)` with `extra="forbid"`, fields `entry_url: str`, `title: str`, `intro: Optional[str] = None`.
  - `Scenario` gains `tour: Optional[TourMeta] = None` (inserted before `steps`).
- **`tests/fixtures/scenario_runner/schema.py`** reduced to a ~20 line re-export shim (`from app.services.scenario_schema import *` + explicit named imports for IDE/star-hygiene). Import path `from .schema import Scenario` still works; class identity verified (`Scenario is Scenario` across both paths).
- **`tests/fixtures/scenario_runner/collector.py`**: `ScenarioFile.collect` now returns early when `scenario.tour is not None`, making tour-only YAML invisible to the 19.1 pytest runner.
- **`tests/fixtures/scenario_runner/test_scenario_schema_tour.py`** (new): 4 tests covering existing-scenario backward-compat, `TourMeta` parse, and `extra="forbid"` rejection.

## Verification

- `pytest tests/fixtures/scenario_runner/test_scenario_schema_tour.py -x -q` → 4 passed.
- `pytest scenarios/ --collect-only -q` → both existing scenarios (`suggest-to-results`, `site-form-submit`) still collected (neither has a `tour:` block).
- `grep -rn "from tests.fixtures.scenario_runner.schema" app/` → empty (app/ never imports from tests/).
- Python identity check: `Scenario` from `app.services.scenario_schema` is the same object as `Scenario` from `tests.fixtures.scenario_runner.schema`.

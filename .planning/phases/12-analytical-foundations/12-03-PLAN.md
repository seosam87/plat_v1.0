  <action>
    Create `app/services/dead_content_service.py` with three main functions:

    **1. `async def get_dead_content(db: AsyncSession, site_id: uuid.UUID) -> dict`**

    Returns `{"pages": [...], "stats": {"zero_traffic": N, "position_drop": M, "total": K}}`

    Detection logic (per D-05, DEAD-01):
    1. Get all pages for the site from the latest crawl job
    2. For each page, check Metrika traffic: query metrika_traffic_pages WHERE site_id = :site_id AND period_end >= CURRENT_DATE - 30 days, SUM(visits) grouped by page_url
    3. For each page, check position delta: query keyword_latest_positions WHERE site_id = :site_id, group by url, compute AVG(delta) — negative delta means drops
    4. A page is "dead content" if:
       - Zero traffic (no metrika rows or SUM(visits) = 0 for last 30 days), OR
       - Average position delta < -10 (positions dropped more than 10 on average)
    5. Use normalize_url() for URL matching between pages, metrika, and positions tables

    Each result dict: `{page_id, url, traffic_30d, keyword_count, avg_position_delta, recommendation, recommendation_reason}`

    **2. Recommendation engine** (per D-06, D-07 — Claude's discretion):

    `def compute_recommendation(traffic_30d: int, keyword_count: int, avg_delta: float | None, avg_position: float | None) -> tuple[str, str]:`

    Returns (recommendation, reason) tuple.

    Rules (applied in order):
    - If keyword_count == 0 and traffic_30d == 0 → ("delete", "Нет ключей и нет трафика")
    - If keyword_count > 0 and traffic_30d == 0 and avg_position is not None and avg_position > 50 → ("redirect", "Есть ключи, но позиции > 50 — перенаправить на релевантную страницу")
    - If keyword_count > 0 and traffic_30d == 0 and avg_position is not None and avg_position <= 50 → ("rewrite", "Есть ключи с позициями — переписать контент")
    - If avg_delta is not None and avg_delta < -10 and traffic_30d > 0 → ("rewrite", "Трафик падает, позиции ухудшились — переписать контент")
    - Default → ("merge", "Рассмотреть объединение с релевантной страницей")

    Note: D-09 defers automatic merge candidate search. Recommendation is just a label.

    **3. `async def update_recommendation(db: AsyncSession, site_id: uuid.UUID, page_url: str, recommendation: str) -> None`**

    Store user override in a new lightweight table or in a JSON column. Simplest approach: use a `dead_content_recommendations` dict stored in Redis with key `dead_content:{site_id}:{normalized_url}` and TTL 30 days. This avoids a migration for a simple preference.

    Alternative (preferred for persistence): Add a small table via the same migration in Plan 01 or create inline. Since Plan 01 is already done, store in Redis: `await redis.set(f"dead_content_rec:{site_id}:{normalize_url(page_url)}", recommendation, ex=2592000)`

    **4. `async def create_dead_content_tasks(db: AsyncSession, site_id: uuid.UUID, page_ids: list[uuid.UUID]) -> int`**

    Per D-08: create SEO tasks for selected dead content pages.
    - For each page_id, look up the page and its recommendation
    - Create SeoTask with:
      - task_type = TaskType.manual
      - title = f"Мёртвый контент: {recommendation} — {url}"
      - description = recommendation_reason
      - status = TaskStatus.open
      - priority = TaskPriority.p3
    - Return count of created tasks

    Create `tests/test_dead_content_service.py`:
    - Use db_session fixture
    - Test detection of zero-traffic and position-drop pages
    - Test recommendation computation with various inputs
    - Test task creation

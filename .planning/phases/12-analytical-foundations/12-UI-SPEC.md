## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, checkbox gap |
| sm | 8px | Button inner padding (compact), gap between filter chips |
| md | 16px | Card internal padding, table cell horizontal padding |
| lg | 24px | Section gap (mb-6), card vertical rhythm |
| xl | 32px | Page header margin-bottom |
| 2xl | 48px | Major section breaks between Quick Wins and Dead Content panels |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Table row cell padding: 12px vertical / 12px horizontal (0.75rem / 1rem) — matches existing `th, td` rule in base.html
- Batch-fix modal inner padding: 24px all sides
- Opportunity score badge: 4px vertical / 8px horizontal (py-1 px-2)

**Source:** base.html global CSS rules + existing analytics/audit template patterns.

## Component Inventory

### Quick Wins Page (`/analytics/{site_id}/quick-wins`)

**Page header**
- H2: "Quick Wins: {site.name}" — text-lg font-semibold text-gray-800
- Right-aligned controls: filter dropdowns + "Запустить фикс" button (disabled until rows selected)

**Filter bar** — bg-white rounded-lg shadow-sm border border-gray-200 p-4 mb-4
- Dropdown "Тип проблемы": Все / Без TOC / Без Schema / Мало ссылок / Тонкий контент
- Dropdown "Тип страницы": Все / Информационные / Коммерческие / Не определён
- Both dropdowns: px-2 py-1.5 border border-gray-300 rounded text-sm focus:ring-indigo-500

**Results table** — bg-white rounded-lg shadow-sm border border-gray-200 overflow-x-auto
Columns (in order):
1. Checkbox (select) — 32px wide
2. URL — truncated, max-w-[280px], text-sm, links to page detail
3. Opportunity Score — text-right, colored badge (amber/yellow/gray by tier, see Color section)
4. TOC — centered, ✓ green-500 or ✗ red-500, 16px icon or text
5. Schema — centered, ✓/✗
6. Ссылки — centered, ✓/✗
7. Контент — centered, ✓/✗
8. Ср. позиция — text-right, text-sm
9. Трафик (нед.) — text-right, text-sm

Table header: bg-gray-50, text-xs font-normal text-gray-500 uppercase tracking-wider, px-3 py-2
Table row: divide-y divide-gray-100, hover:bg-gray-50
Selected row: bg-indigo-50

**"Select all" checkbox** — in thead, left of URL column header

**Batch-fix button** — "Запустить фикс ({N} стр.)" — bg-indigo-600 text-white rounded px-3 py-1.5 text-sm font-medium. Disabled state: opacity-50 cursor-not-allowed. Enabled when at least 1 row selected.

**Batch-fix confirmation modal** (D-10)
- Title: "Запустить фикс"
- Body: scrollable list of selected page URLs (max-h-48 overflow-y-auto, text-sm)
- Checkboxes: "TOC", "Schema", "Ссылки" — each with a label, default all checked
- Footer buttons: "Не запускать" (gray) + "Запустить фикс" (bg-indigo-600 text-white)
- Modal backdrop: rgba(0,0,0,.5) — matches existing memory pattern
- Progress after dispatch: toast-info "Фикс запущен для {N} страниц" + HTMX polling on a status indicator div (hx-get, hx-trigger="every 5s", hx-swap="outerHTML") that shows spinner while tasks run, then toast-success on completion

### Dead Content Page (`/analytics/{site_id}/dead-content`)

**Page header**
- H2: "Мёртвый контент: {site.name}" — text-lg font-semibold text-gray-800
- Right: "Создать задачи ({N})" button — bg-emerald-600 text-white, disabled until rows selected

**Summary stats strip** — 3 stat cards in a row (grid grid-cols-3 gap-4 mb-4)
- "Нет трафика (30 дн.)" — count, text-3xl font-bold text-red-600
- "Падение позиций > 10" — count, text-3xl font-bold text-amber-600
- "Всего кандидатов" — count, text-3xl font-bold text-gray-700

**Results table** — same card container as Quick Wins
Columns:
1. Checkbox (select)
2. URL — truncated, max-w-[280px]
3. Трафик 30 дн. — text-right (0 shown as "—" in text-gray-400)
4. Ключей привязано — text-right, text-sm
5. Падение позиций — text-right; negative delta shown as red, "—" if no data
6. Рекомендация — colored badge (merge/redirect/rewrite/delete per Color section) + editable dropdown on hover/click (select element, same options)

Dropdown for recommendation override: `<select>` inline in the cell, same Tailwind select classes as filter bar. Changes POST to `/analytics/{site_id}/dead-content/{page_id}/recommendation` via HTMX hx-post, hx-swap="none", hx-trigger="change", shows toast-success on HX-Trigger response header.

**"Создать задачи" flow**
- Clicking button opens a small confirmation: "Создать SEO-задачи для {N} страниц?" with "Не создавать" + "Создать задачи" (bg-emerald-600)
- On confirm: POST via fetch, toast-success "Задачи созданы"

### Navigation additions
- Under the "Аналитика" sidebar section: add child links "Quick Wins" and "Мёртвый контент"
- These follow the existing child link pattern: px-4 py-1 text-sm font-medium sidebar-child, active state bg-indigo-700

### No new infrastructure UI
- `normalize_url()` and `keyword_latest_positions` are backend-only; no dedicated UI surface required

## Interaction States

### Table rows
- Default: white bg, divide-gray-100 border
- Hover: bg-gray-50
- Selected (checkbox): bg-indigo-50, checkbox checked

### Batch fix button
- Default disabled: "Запустить фикс" — opacity-50 cursor-not-allowed bg-indigo-400 text-white
- Enabled (1+ rows selected): bg-indigo-600 hover:bg-indigo-700 text-white cursor-pointer
- Loading (after dispatch): button disabled, text "Запуск..." with inline spinner SVG

### Recommendation dropdown (Dead Content)
- Display mode: colored badge showing current recommendation
- Edit mode: `<select>` replaces badge on click, auto-saves on change via HTMX

### HTMX polling for batch fix progress
- A `<div id="fix-status-{task_id}">` injected after dispatch
- `hx-get="/analytics/{site_id}/fix-status/{task_id}" hx-trigger="every 5s" hx-swap="outerHTML"`
- While pending: shows spinner + "Обрабатывается..."
- On completion: replaces itself with a static "Готово" span + fires `showToast` via HX-Trigger response header

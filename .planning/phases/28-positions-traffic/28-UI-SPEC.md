## Spacing Scale

Declared values — 8-point grid, multiples of 4 only:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps in bottom nav (gap:4px), dot indicators |
| sm | 8px | Inline gaps between flex children (`gap-2`), button row gaps |
| md | 16px | Card internal padding (`p-4`), page padding (`p-4`), input horizontal padding (`px-3`) |
| lg | 24px | Section bottom margin (`mb-6`), action button top margin (`mt-6`) |
| xl | 32px | — |
| 2xl | 48px | Empty state vertical padding (`py-12`) |
| 3xl | 64px | Bottom nav height; `padding-bottom` on `<body>` to clear nav |

Exceptions:
- Touch targets: minimum 44px height on all tappable elements (`min-h-[44px]`) — established pattern from Phase 27
- Toast container: `bottom: 80px` (above nav), `right: 16px` — fixed value, not grid-derived
- Progress bar height: 8px (visual indicator only, not touch target)

## Interaction Contracts

### Positions Page (`/m/positions`)

#### Filter Bar
- Site select: `<select>` with `min-h-[44px]`, `hx-get="/m/positions"`, `hx-target="#positions-list"`, `hx-trigger="change"`, `hx-include="[name='period']"`
- Period select: `7d` / `30d` / `all` options, same HTMX pattern
- Filters render above the tab bar, below the `<h1>`

#### Keyword Tabs
- Tab bar: two inline tabs — "Все" and "Просевшие"
- Active tab: `font-weight: 600`, `border-bottom: 2px solid #4f46e5`, color `#4f46e5`
- Inactive tab: `color: #6b7280`, no border
- Tab switch: `hx-get` partial swap of `#positions-list` with tab param
- "Просевшие" tab: shows keywords where delta < 0; each row includes "Создать задачу" inline button

#### Keyword Card Structure
```
[ phrase text (flex-1, text-sm)     ] [ position badge ] [ delta chip ]
[ engine icon/text (text-xs gray)   ] [ check date (text-xs gray)    ]
```
- Position badge: `bg-gray-100 text-gray-800 rounded px-2 py-0.5 text-xs font-semibold` — value like "14"
- Delta chip: green `bg-green-100 text-green-700` for positive, red `bg-red-100 text-red-700` for negative — value like "+3" or "−5"
- No delta (new keyword): gray chip "new" `bg-gray-100 text-gray-500`
- Touch target: entire card row `min-h-[44px]` — not individually actionable (except "Просевшие" tab)
- "Просевшие" tab: each card has "Создать задачу" button (`text-xs text-indigo-600 underline`) that triggers `hx-get` loading `task_form.html` into `#task-form-slot-{keyword_id}`

#### Sort Order
Default: by absolute delta magnitude descending (biggest drops first, per D-02).

### Position Check Launch

#### Trigger Button
- Label: "Запустить проверку"
- Style: `min-h-[44px] rounded-lg text-white font-semibold text-sm` + `style="background:#4f46e5;"`
- Attributes: `hx-post="/m/positions/check"`, `hx-swap="none"`, `hx-disabled-elt="this"`, `hx-on::after-request="if(event.detail.successful) { showToast('Проверка запущена', 'success'); startPositionPolling(); }"`

#### Progress Block (HTMX Polling)
- Shown after successful POST, replaces/appears in `#check-progress-slot`
- Template: `position_progress.html` partial
- Attributes: `hx-get="/m/positions/check/status"`, `hx-trigger="every 3s"`, `hx-target="this"`, `hx-swap="outerHTML"`
- Running state display:
  ```
  [ spinner (inline SVG animate-spin) ] Проверено {checked} из {total} ключей
  [ progress bar: div full-width h-2 bg-gray-200 > div bg-indigo-600 style="width:{pct}%" ]
  ```
- Done state (server returns `status=done`): HTMX polling stops (remove `hx-trigger`), block shows:
  ```
  [ green checkmark SVG ] Проверка завершена — {checked} ключей
  [ button "Показать результаты" style="background:#4f46e5" hx-get="/m/positions" hx-target="#positions-list" hx-swap="innerHTML" ]
  ```
- Server returns `hx-trigger: none` header OR a non-polling template when `status=done` to halt polling

### Traffic Page (`/m/traffic`)

#### Period Presets (D-07)
- Three buttons rendered as a pill toggle group:
  - "Эта неделя vs прошлая"
  - "Этот месяц vs прошлый"
  - "30 дней vs 30 дней"
- Active preset: `bg-indigo-600 text-white rounded-full px-3 py-1 text-sm font-semibold`
- Inactive preset: `bg-white text-gray-700 rounded-full px-3 py-1 text-sm border border-gray-200`
- Tap: `hx-get="/m/traffic"`, `hx-target="#traffic-content"`, `hx-vals='{"period": "..."}'`

#### Site Select
- Same `<select>` pattern as positions page, `hx-include="[name='period']"`

#### No Metrika State (D-06)
- Shown when `site.metrika_token` is null/empty
- Full-width card: `bg-yellow-50 border border-yellow-200 rounded-lg p-4`
- Icon: warning SVG (yellow, 20×20)
- Text: "Метрика не подключена" (text-sm font-semibold text-yellow-800), subtext: "Подключите Яндекс.Метрику в настройках сайта для сравнения трафика."

#### Summary Card (D-08)
```
[ Period 1 total ]  →  [ Period 2 total ]  [ delta chip large ]
  e.g. "12 450 визитов"  "10 820 визитов"  "−13%"
```
- Card: `bg-white rounded-lg p-4 mb-4`
- Delta chip: `text-lg font-semibold` — red for negative, green for positive
- Period labels: `text-xs text-gray-500` above each number

#### Per-Page List (D-08)
- Card list: `bg-white rounded-lg divide-y divide-gray-100`
- Each row: `flex items-center gap-3 p-3 min-h-[44px]`
  - URL truncated to 40 chars: `text-sm text-gray-800 flex-1 truncate`
  - Delta chip (same style as position delta)
  - Chevron right SVG `w-5 h-5 text-gray-400`
- Tap on row: `hx-get="/m/traffic/task-form"`, loads `task_form.html` into `#task-form-slot` with `prefilled_title` = page URL
- Sort: biggest drops first (most negative delta at top), per D-08

## State Inventory

Each screen must handle all states explicitly:

### `/m/positions`

| State | Trigger | Display |
|-------|---------|---------|
| Loading | Initial page load or filter change | HTMX `hx-indicator` spinner on list container |
| Empty — no keywords | Service returns empty list | Empty state card (see copywriting) |
| Populated | Keywords exist with positions | Card list with delta chips |
| Check in progress | After "Запустить проверку" POST | Progress block polls every 3s |
| Check complete | Polling detects status=done | "Показать результаты" button |
| Check error | POST returns non-2xx | Toast error message |
| Empty — no drops | "Просевшие" tab, no negative deltas | Empty state card |

### `/m/traffic`

| State | Trigger | Display |
|-------|---------|---------|
| No Metrika token | `site.metrika_token` is null | Yellow warning card (full page content area) |
| Loading | Period/site change | HTMX indicator |
| Empty — no data | Metrika returns no rows | Empty state card |
| Populated | Data returned | Summary card + per-page list |
| Metrika API error | Service throws exception | Toast error + empty state fallback |

## HTMX Patterns

These patterns are established in the codebase and MUST be followed exactly:

| Pattern | Code |
|---------|------|
| Post-request toast | `hx-on::after-request="if(event.detail.successful) showToast('msg', 'success')"` |
| Prevent double-tap | `hx-disabled-elt="this"` on every button that triggers POST |
| Filter with include | `hx-get="..." hx-include="[name='site_id'],[name='period']"` on each filter element |
| Partial list swap | `hx-target="#positions-list" hx-swap="innerHTML"` |
| Polling | `hx-trigger="every 3s" hx-target="this" hx-swap="outerHTML"` |
| Stop polling | Server returns template without `hx-trigger` attribute to halt polling |
| Task form load | `hx-get="/m/positions/task-form?keyword_id={id}&prefilled_title={phrase}" hx-target="#task-form-slot-{id}" hx-swap="innerHTML"` |

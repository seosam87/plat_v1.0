## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap (`gap-1`), badge inner padding |
| sm | 8px | Between cards in a list (`gap-2`), button inline padding |
| md | 16px | Page content padding (`p-4`), form field margin-bottom (`mb-4`) |
| lg | 24px | Section heading margin-bottom (`mb-6`) |
| xl | 32px | Major vertical rhythm between page sections |
| 2xl | 48px | Not used in mobile context |
| 3xl | 64px | Bottom nav height reservation (`padding-bottom: 64px`) |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (`min-h-[44px]`) — source: existing mobile pattern
- Bottom nav: 64px fixed height — source: base_mobile.html
- Toast bottom offset: 80px from bottom — source: base_mobile.html `#toast-container`
- Progress bar height: 8px (`h-2`) — source: position_progress.html pattern

## Typography

Source: existing mobile templates (positions.html, traffic.html, base_mobile.html).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / list item text | 14px (`text-sm`) | 400 regular | 1.5 (default) |
| Label / secondary meta | 12px (`text-xs`) | 400 regular | 1.5 |
| Page heading | 20px (`text-xl`) | 600 semibold (`font-semibold`) | 1.2 |
| Card title / section label | 14px (`text-sm`) | 600 semibold (`font-semibold`) | 1.4 |

Rules:
- Maximum 2 font weights: 400 (regular) and 600 (semibold).
- No font-size outside this declared set.
- Truncation: `truncate` class on single-line text in constrained-width containers (tool name, project name).

**Accent `#4f46e5` reserved for exactly these elements:**
1. Primary action button background: "Создать отчёт", "Запустить" CTA buttons
2. Active filter pill background (selected project, selected report type radio-card border)
3. Progress bar fill (`background:#4f46e5`)
4. Active tab underline indicator on tab bars (`border-indigo-600`)
5. Inline text links used in card actions ("Показать все")

**Accent is NOT used for:**
- Secondary buttons (use `border-gray-300 text-gray-700 bg-white`)
- Navigation items in non-active state (use `color:white`)
- Badge backgrounds (use `bg-gray-100`, `bg-green-100`, `bg-red-100`)

## Screen Inventory

All screens extend `base_mobile.html`. All include bottom nav with `active_tab` variable.

### 1. `/m/reports/new` — Report Creation Page

**Layout:** Single-page form. No multi-step wizard, no HTMX swap between steps.

**Sections (top to bottom):**
1. Page heading: "Отчёт клиенту" — `text-xl font-semibold mb-4`
2. **Project select** — `<select>` full-width, `min-h-[44px]`, placeholder option "— Выберите проект —"
3. **Report type** — 2 radio-cards horizontal: "Краткий (1-2 стр.)" and "Подробный (5-10 стр.)". Each card: `bg-white rounded-lg border-2 p-4`. Selected state: `border-indigo-600`. Unselected: `border-gray-200`. Each card shows name (bold, `text-sm font-semibold`) + description (`text-xs text-gray-500`).
4. **Recipient select** — `<select>` full-width, `min-h-[44px]`, placeholder "— Выберите клиента —". Shows clients with `email IS NOT NULL OR telegram_username IS NOT NULL`. Contact icons shown in option label: "📧" for email, "📨" for Telegram (inline emoji acceptable, not icon SVG in select).
5. **Submit button** — "Создать отчёт" full-width, `min-h-[44px]`, accent background. Disabled (grey, `opacity-50 cursor-not-allowed`) until both project and report type are selected. Recipient select can be empty at this point (delivery is optional pre-step).
6. **Result block** (hidden until report generated, revealed via HTMX swap or server-side render with `?report_id=`):
   - Card `bg-white rounded-lg p-4 mt-4`
   - Header: "Отчёт готов" (`text-sm font-semibold text-green-700`) + report type label
   - Three CTAs in a column: `[Скачать PDF]` (secondary outline), `[Отправить в Telegram]` (accent fill), `[Отправить email]` (secondary outline)
   - All CTA buttons: `w-full min-h-[44px] rounded-lg text-sm font-semibold`
   - Telegram/email buttons: disabled if recipient has no corresponding contact (show `opacity-40`)

**States:**
- Form loading: skeleton not required; standard browser rendering
- Generating report: spinner replaces button text, button disabled
- Success: result block revealed via HTMX, scroll to result block
- Send success: `showToast("Отчёт отправлен", 'success')`
- Send error: `showToast("Ошибка: клиент не привязан к Telegram", 'error')` (or email equivalent)

### 2. `/m/tools` — Tools List Page

**Layout:** Single-column card list (not 2-col grid). Source: D-08.

**Sections:**
1. Page heading: "SEO Инструменты" — `text-xl font-semibold mb-4`
2. Card list: 6 cards, `bg-white rounded-lg divide-y divide-gray-100`

**Each tool card:**
- Full card tappable (`<a>` wrapping entire card, href `/m/tools/{slug}/run`)
- Layout: `flex items-center gap-3 p-4 min-h-[44px]`
- Left: tool name (`text-sm font-semibold`) + description below (`text-xs text-gray-500`)
- Right: limit label (`text-xs text-gray-400`) e.g. "до 100 фраз"
- Badge "Требует Wordstat" shown only for `wordstat-batch` tool — `bg-amber-100 text-amber-800 text-xs rounded px-2 py-0.5 font-semibold`
- Trailing chevron icon (right arrow, 16x16, `text-gray-400`)

**OAuth redirect behavior (D-10):** Card tap triggers backend check; if no OAuth → server returns 302 to `/ui/integrations/wordstat/auth?return_to=/m/tools/wordstat-batch/run`. No special UI handling in frontend needed; browser follows redirect.

### 3. `/m/tools/{slug}/run` — Tool Entry Screen

**Layout:** Single page.

**Sections:**
1. Back link: "← Инструменты" (`text-sm text-indigo-600 underline`, links to `/m/tools`)
2. Tool name as heading: `text-xl font-semibold mb-1`
3. Tool description: `text-sm text-gray-500 mb-4`
4. Limit note: `text-xs text-gray-400 mb-3` — "Лимит: до {N} фраз"
5. **Input area** — textarea + optional file upload, both shown simultaneously:
   - `<textarea>` full-width, `min-h-[120px]`, placeholder "Введите фразы (по одной на строку)"
   - `<input type="file" accept=".txt,.xlsx">` below textarea, styled as secondary button or plain file input
   - If tool has `has_domain_field=True`: add `<input type="text">` for domain, `min-h-[44px]`
   - If tool has region field: add `<select>` for region
6. **Launch CTA** — "Запустить" full-width accent button, `min-h-[44px]`
7. **Progress slot** — initially empty, filled by HTMX after POST

**Validation error:** If both textarea and file provided → server returns error, displayed via: `showToast("Используйте одно из двух: текст или файл", 'error')`

**After launch:**
- `showToast("Запущено", 'success')`
- Progress slot filled with polling partial (same pattern as `position_progress.html`): `hx-trigger="every 3s"`

### 4. `/m/tools/{slug}/jobs/{job_id}` — Tool Result View

**Layout:** Single page with summary card + data table.

**Sections:**
1. Back link: "← {Tool Name}" links to `/m/tools/{slug}/run`
2. **Summary top card** — `bg-white rounded-lg p-4 mb-4`
   - Tool name (`text-sm font-semibold`)
   - Status badge: "Завершено" (`bg-green-100 text-green-700`) or "Ошибка" (`bg-red-100 text-red-700`)
   - Total rows count: `text-xs text-gray-500`
   - "Скачать XLSX" CTA button: secondary outline, `min-h-[44px] w-full`
3. **Results table** — top 20 rows, `bg-white rounded-lg divide-y divide-gray-100`
   - Each row: compact, `p-3 min-h-[44px]`, relevant columns per tool type (phrase + value columns)
   - Text: `text-sm` for primary data, `text-xs text-gray-500` for secondary
4. **"Показать все"** button — secondary outline, `w-full min-h-[44px] mt-3` — triggers HTMX modal with full paginated results

**Progress polling partial** (reused from Phase 28 pattern):
- Running: spinner + "Проверено X из Y" or "Выполняется..." + progress bar
- Done: green check + count + "Показать результаты" accent button linking to `/m/tools/{slug}/jobs/{job_id}`
- Error: red warning + "Ошибка. Попробуйте ещё раз." + retry button

## Interaction Contracts

### Form Validation
- Submit button disabled until required fields filled — use JS `change` listener on selects and radio-cards, toggling `disabled` attribute and `opacity-50 cursor-not-allowed` classes.
- No inline field validation required (mobile MVP, server-side errors via toast).

### HTMX Patterns
- Tool job polling: `hx-get="/m/tools/{slug}/jobs/{job_id}/status" hx-trigger="every 3s" hx-target="this" hx-swap="outerHTML"` — identical to `position_progress.html`.
- Report result reveal: full form POST → server renders result block inline, or `?report_id=` query param triggers server-side pre-render.
- "Показать все" modal: `hx-get="/m/tools/{slug}/jobs/{job_id}/all" hx-target="#modal-slot" hx-swap="innerHTML"` — modal slot appended to bottom of content.

### Navigation
- `active_tab` value for all new pages:
  - `/m/reports/new` → `active_tab = 'more'`
  - `/m/tools` → `active_tab = 'more'`
  - `/m/tools/{slug}/run` → `active_tab = 'more'`
  - `/m/tools/{slug}/jobs/{job_id}` → `active_tab = 'more'`

### Accessibility
- All interactive elements: `min-h-[44px]` touch target.
- Radio-cards for report type: use `<label>` wrapping `<input type="radio">` (hidden) + styled div, so tap on card activates radio.
- Tool cards: `<a>` tag wrapping entire card content for full-area tap.
- Progress states: include screen-reader text via `aria-live="polite"` on progress slot.

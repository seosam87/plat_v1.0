# Phase 06.1 Plan 02: Admin Proxy UI and Credential Management Summary

Admin UI for proxy pool CRUD (add/edit/delete via HTMX), XMLProxy/rucaptcha/anticaptcha credential forms with balance auto-load, and site yandex_region field.

## What Was Built

### Task 1: Admin proxy router

`app/routers/proxy_admin.py` — synchronous FastAPI router with `APIRouter(prefix="/admin/proxies")`:

- **Proxy CRUD:** `POST ""` (create), `PUT "/{id}"` (update url+type), `DELETE "/{id}"`, `GET ""` (list JSON)
- **Health checks:** `POST "/check-all"` (all proxies + balance fetch), `POST "/{id}/check"` (single proxy)
- **Credentials:** `POST "/credentials/xmlproxy"`, `POST "/credentials/rucaptcha"`, `POST "/credentials/anticaptcha"`
- **Balances:** `GET "/xmlproxy-balance"` (XMLProxy API), `GET "/rucaptcha-balance"` (rucaptcha.com API)

Router registered in `app/main.py` alongside all other routers.

8 unit tests in `tests/test_proxy_admin.py`, all passing.

### Task 2: Admin settings page and site edit form

- `app/templates/admin/settings.html` — proxy section with HTMX table, add form, check-all button, XMLProxy/rucaptcha/anticaptcha credential cards with balance auto-load widgets
- `app/templates/admin/partials/proxy_row.html` — single proxy row with inline hx-put edit, hx-delete, single-check buttons
- `app/templates/admin/partials/proxy_section.html` — full proxy section partial for check-all innerHTML swap
- `app/templates/sites/edit.html` — yandex_region number input with common region code hints
- `app/main.py` `/ui/admin/settings` handler — loads proxies + credentials via sync DB, passes to template
- `app/main.py` `/ui/sites/{id}/edit` POST handler — reads and saves yandex_region (int or None)

# Phase 06.1: Proxy Management & XMLProxy Integration - Context

**Gathered:** 2026-04-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Build proxy pool management system, integrate XMLProxy.ru API for Yandex SERP parsing, create admin UI for proxy/service management, and connect XMLProxy to the position checking pipeline. Google parsing is out of scope for this phase.

</domain>

<decisions>
## Implementation Decisions

### XMLProxy API Strategy
- **D-01:** XMLProxy is the ONLY source for Yandex SERP — no Playwright fallback for Yandex
- **D-02:** Handle -55 async errors via Celery retry with countdown=300 (5 min), max_retries=3. Repeat within 12h is free on XMLProxy side
- **D-03:** Telegram alert when XMLProxy balance < configurable threshold, pause all Yandex parsing when balance=0
- **D-04:** XMLProxy credentials (user + key) stored in DB, editable through admin UI (not .env)
- **D-05:** Yandex region (lr parameter) configured per site in site settings. Each site can target a different city

### Proxy Pool Management
- **D-06:** Support both HTTP/HTTPS and SOCKS5 proxy types in the pool
- **D-07:** Health check strategy: manual button in admin + automatic check before each position check cycle. Dead proxies marked as inactive automatically
- **D-08:** Proxy pool is for future Google parsing — in this phase, build the model and management UI but don't wire it into position checks yet

### Admin UI
- **D-09:** New "Прокси" tab on /ui/admin/settings page (alongside existing integrations)
- **D-10:** Full CRUD for proxies: add/edit/delete with URL, type (http/socks5), optional auth
- **D-11:** Proxy list table: URL, type, status (active/dead), response time, last checked
- **D-12:** XMLProxy widget: current balance, cost per query, max cost — fetched via balance API
- **D-13:** "Проверить все" button: tests all proxies + XMLProxy balance in one click
- **D-14:** Anticaptcha key stored in DB (not .env), editable through the same proxy settings page
- **D-15:** Anticaptcha service: rucaptcha.com (replace current anti-captcha.com endpoints)
- **D-16:** Show anticaptcha balance on the settings page

### Position Check Pipeline
- **D-17:** Google SERP parsing is OUT OF SCOPE for this phase — only Yandex via XMLProxy
- **D-18:** For sites with mixed engine keywords: group by engine, process Yandex keywords via XMLProxy, skip Google keywords (log as "no source configured")

### Claude's Discretion
- Mixed engine keyword grouping implementation details (D-18) — Claude decides whether to split into separate Celery tasks or handle within one task
- DB model design for proxy pool and service credentials (table structure, column types)
- XMLProxy API client implementation patterns (httpx vs raw requests, XML parsing approach)

### Folded Todos
- **Proxy management, XMLProxy integration and health checker** (from pending todos) — fully covered by this phase scope

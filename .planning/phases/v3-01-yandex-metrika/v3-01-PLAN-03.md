<action>
Create `app/routers/metrika.py`:

```python
router = APIRouter(prefix="/metrika", tags=["metrika"])
```

**Endpoints:**

1. `GET /metrika/{site_id}` → HTMLResponse
   - Renders `metrika/index.html` template
   - Context: site object, daily_data (from DB), page_data (from DB for default period = last 30 days), events list
   - If site has no metrika_counter_id → show "not configured" empty state
   - If no data in DB → show "no data fetched yet" empty state

2. `POST /metrika/{site_id}/fetch` → JSON
   - Reads date1/date2 from request body (or defaults: date1=90 days ago, date2=yesterday)
   - Dispatches `fetch_metrika_data.delay(str(site_id), date1, date2)` Celery task
   - Returns `{"status": "queued", "task_id": task.id}`

3. `GET /metrika/{site_id}/daily` → JSON
   - Query params: `date_from` (optional), `date_to` (optional)
   - Calls `metrika_service.get_daily_traffic(db, site_id, date_from, date_to)`
   - Returns `{"labels": [...], "visits": [...], "bounce_rate": [...], "page_depth": [...], "avg_duration": [...]}`

4. `GET /metrika/{site_id}/pages` → JSON
   - Query params: `period_start`, `period_end`
   - Calls `metrika_service.get_page_traffic(db, site_id, period_start, period_end)`
   - Returns list of page dicts

5. `GET /metrika/{site_id}/compare` → JSON
   - Query params: `a_start`, `a_end`, `b_start`, `b_end`
   - Calls `get_page_traffic` for both periods, then `compute_period_delta`
   - Returns list of delta dicts with is_new, is_lost flags

6. `POST /metrika/{site_id}/events` → HTMLResponse (HTMX partial)
   - Form fields: `event_date`, `label`, `color` (default `#7c3aed`)
   - Calls `metrika_service.create_event(db, site_id, ...)`
   - Returns rendered `metrika/_events_list.html` partial

7. `DELETE /metrika/{site_id}/events/{event_id}` → empty 200
   - Calls `metrika_service.delete_event(db, event_id)`
   - Returns empty string (HTMX removes the element via hx-swap="outerHTML")

8. `PUT /metrika/{site_id}/settings` → JSON
   - Body: `{"counter_id": str, "token": str}`
   - Encrypts token via `crypto_service.encrypt(token)`, saves both fields to Site
   - Returns `{"status": "saved"}`

Pydantic schemas defined in the same file (following competitors.py pattern):
```python
class MetrikaSettingsUpdate(BaseModel):
    counter_id: str
    token: str

class MetrikaFetchRequest(BaseModel):
    date1: str | None = None
    date2: str | None = None
```

All endpoints require `current_user: User = Depends(require_admin)`.

<action>
Create directory `app/templates/metrika/` and file `app/templates/metrika/index.html`:

**Template structure** (extends `base.html`):

1. **Page header row:**
   - `<h2>Трафик: {{ site.name }}</h2>`
   - Fetch button: `<button class="btn btn-verify" hx-post="/metrika/{{ site.id }}/fetch" hx-swap="none" onclick="this.textContent='Загрузка...'; this.disabled=true">Загрузить данные Метрики</button>`

2. **Empty states** (conditional):
   - If `not site.metrika_counter_id`: show card with text "Счётчик Метрики не настроен. Добавьте ID счётчика и токен в настройках сайта."
   - If `site.metrika_counter_id and not daily_data`: show "Данные ещё не загружены. Нажмите «Загрузить данные Метрики»."

3. **KPI stats row** (4 cards, grid `repeat(auto-fit, minmax(160px, 1fr))`):
   - Визиты: `{{ totals.visits }}` (color `#2563eb`)
   - Отказы: `{{ totals.bounce_rate }}%` (color `#dc2626` if >50, `#059669` if <30, `#6b7280` otherwise)
   - Глубина просмотра: `{{ totals.page_depth }}` (color `#7c3aed`)
   - Время на сайте: formatted as `Xм Yс` (color `#0ea5e9`)

4. **Traffic chart card:**
   - `<canvas id="traffic-chart" style="height:300px"></canvas>`
   - Load Chart.js 4.4.0 (already in positions templates) and chartjs-plugin-annotation@3.0.1 via CDN
   - Chart config: type `line`, line color `#4f46e5`, fill `rgba(79,70,229,0.1)`, tension 0.3, pointRadius 2
   - Event annotations: built from `events` list using `buildAnnotations()` function
   - X axis: date strings, Y axis: visits, beginAtZero true

5. **Date range pickers + comparison:**
   - Period A: date_a_start, date_a_end inputs
   - Period B: date_b_start, date_b_end inputs
   - "Сравнить периоды" button: onclick calls `fetchComparison()` JS function
   - `fetchComparison()` calls `GET /metrika/{{ site.id }}/compare?a_start=...&a_end=...&b_start=...&b_end=...`
   - Results rendered into `#compare-result` div as a table

6. **Event markers card:**
   - List existing events: `{% for ev in events %}` → `<li>` with date, label, color swatch, delete button
   - Delete: `hx-delete="/metrika/{{ site.id }}/events/{{ ev.id }}" hx-confirm="Удалить это событие?" hx-target="closest li" hx-swap="outerHTML swap:200ms"`
   - Add event form: date input, label input, color input (default `#7c3aed`), submit button "Добавить событие"
   - Form: `hx-post="/metrika/{{ site.id }}/events" hx-target="#events-list" hx-swap="innerHTML"`

7. **Period comparison table** (in `#compare-result` div):
   - Columns: Страница | Визиты (А) | Визиты (Б) | Дельта | Отказы | Глубина | Длительность
   - Delta display: `▲ +N` green (#059669) if delta > 0, `▼ N` red (#dc2626) if delta < 0, `=` gray (#6b7280) if 0
   - New page badge: `.badge-connected` with "Новая"
   - Lost page badge: `.badge-failed` with "Потеря"
   - Filter buttons: "Все страницы" | "Новые" | "Потери" — JS toggle to show/hide rows

8. **Metrika Settings section** (at bottom of page):
   - Card with counter_id input, token (password) input, "Сохранить настройки" button
   - Submit via `fetch('/metrika/{{ site.id }}/settings', {method: 'PUT', body: JSON.stringify({...})})`

9. **JS at bottom:**
   - `buildAnnotations(events)` — from RESEARCH.md section 4
   - `fetchComparison()` — fetch comparison data and render table
   - Chart initialization with event annotations
   - Auto-reload after fetch task: `setTimeout(() => location.reload(), 5000)` on fetch button click

All copy in Russian per UI-SPEC Copywriting Contract.
</action>

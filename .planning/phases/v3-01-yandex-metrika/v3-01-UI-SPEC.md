# Phase v3-01 — Yandex Metrika UI Design Contract

> Visual and interaction contract for the Metrika traffic module.
> Generated by gsd-ui-researcher; verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | Tailwind CSS (inline utility classes) | Codebase scan — base.html, all templates |
| Preset | Not applicable (no tailwind.config.js; inline styles + utility classes via CDN) | Codebase scan |
| Component library | None — custom CSS classes in base.html (`.card`, `.btn`, `.badge`) | Codebase scan |
| Icon library | Unicode symbols inline (▲ ▼ = ●) | Codebase scan — positions/index.html |
| Font | system-ui, sans-serif (OS default) | base.html: `font-family: system-ui, sans-serif` |
| Chart library | Chart.js 4.4.0 (CDN) + chartjs-plugin-annotation 3.0.1 (new, CDN) | RESEARCH.md §4 |

> Note: This project does NOT use a Tailwind JIT build pipeline. Styles are either inline `style=""` attributes or the global CSS classes defined in `base.html`. Follow this pattern strictly — do not introduce PostCSS or Tailwind class compilation.

## Spacing Scale

All spacing uses the existing 8-point base. Values are written as inline CSS in `style=""` attributes.

| Token | Value | CSS | Usage |
|-------|-------|-----|-------|
| xs | 4px | `gap:.25rem` | Icon gaps, badge padding |
| sm | 8px | `gap:.5rem` / `padding:.4rem .6rem` | Input padding, compact row gaps |
| md | 16px | `margin-bottom:1rem` / `padding:1rem` | Default card internal spacing |
| lg | 24px | `margin-bottom:1.5rem` / `padding:1.5rem` | Section breaks, card padding (default) |
| xl | 32px | `margin-bottom:2rem` | Major section breaks on the traffic page |
| 2xl | 48px | `margin: 2rem auto` | Page-level vertical rhythm (inherited from `main`) |

Exceptions:
- Touch targets for date inputs: min-height 36px (`.4rem .75rem` padding on inputs = ~36px total) — inherited from existing form pattern
- Chart canvas: height is fixed at `300px` for chart modal; `80px` for site overview widget sparkline

## Typography

All font sizes follow the existing codebase convention using `rem` units at a 16px base.

| Role | Size | CSS | Weight | Line Height | Usage |
|------|------|-----|--------|-------------|-------|
| Body / Label | 14px | `font-size:.875rem` | 400 | 1.5 | Table cell text, form labels, descriptions; use `color:#6b7280` for meta/secondary labels |
| Subheading | 16px | `font-size:1rem` | 600 | 1.4 | Card section headings (h3) |
| Heading | 20px | `font-size:1.25rem` | 600 | 1.2 | Page title (h2) |
| Display metric | 28px | `font-size:1.75rem` | 600 | 1.1 | KPI numbers in stat cards (visits total, bounce rate) |

Column header style (inherited from base.html `th`): `font-weight: 600; color: #6b7280; font-size: .875rem; text-transform: uppercase`

## Page Layout: Traffic Page (`/ui/metrika/{site_id}`)

Inherits `base.html` layout: `max-width:1100px; margin:2rem auto`.

```
[Page header row: "Трафик: {site.name}" + "Загрузить данные Метрики" button]

[KPI stats row — 4 cards: Визиты, Отказы, Глубина просмотра, Время на сайте]
  grid-template-columns: repeat(auto-fit, minmax(160px,1fr)); gap:1rem; margin-bottom:1.5rem

[Traffic chart card]
  Line chart (Chart.js) with event overlay annotations
  Date range picker: Period A start/end — Period B start/end
  "Обновить график" button

[Event markers card]
  List of existing events (date + label + color swatch + delete button)
  Add event form: date input + label input + color picker + "Добавить" button

[Period comparison card]
  Period A label | Period B label column headers
  Pages table: URL | Визиты A | Визиты B | Дельта | Отказы A | Отказы B | Глубина | Длит.
  Sortable by delta (default: sorted by Визиты B descending)
  Filters: "Все страницы" | "Новые" | "Потери"
```

## Widget Layout: Site Overview (`/ui/sites/{site_id}/detail`)

Injected via `hx-get="/metrika/{site_id}/widget" hx-trigger="load"` into the existing detail page card grid.

```
[Widget card — same .card class as existing stat cards]
  Title: "Трафик (органика)"
  Sub: "За последние 30 дней"

  [Row of 4 mini-metrics]
    Визиты: {N} | Отказы: {N}% | Глубина: {N} | Длительность: {Nм Nс}

  [Sparkline: 30-day visits line — 80px height canvas]

  [Footer link: "Подробный трафик →" → /ui/metrika/{site_id}]
```

If Metrika is not configured on the site: widget is hidden entirely (no placeholder shown).
If data has never been fetched: widget shows "Данные не загружены. Настройте счётчик →" with link to settings.

## Interaction Patterns

Consistent with existing HTMX patterns in the codebase:

### Fetch Metrika data (button)
```html
<button class="btn btn-verify"
  hx-post="/metrika/{{ site_id }}/fetch"
  hx-swap="none"
  onclick="this.textContent='Загрузка...'; this.disabled=true">
  Загрузить данные Метрики
</button>
```
After Celery task completes: show success flash, then `location.reload()` after 3s delay.

### Add event marker (HTMX form)
```
POST /metrika/{site_id}/events → JSON
Response triggers JS: chart.options.plugins.annotation.annotations updated, chart.update()
```
No page reload — partial JS update only.

### Delete event marker (HTMX)
```
DELETE /metrika/{site_id}/events/{id}
hx-confirm="Удалить это событие?"
hx-target="closest li"
hx-swap="outerHTML swap:200ms"
```

### Period comparison (modal — same pattern as positions compare modal)
- Two date range pickers (date_a_start, date_a_end, date_b_start, date_b_end)
- "Сравнить периоды" → `fetch()` → renders comparison table in `#compare-result` div
- Modal background click closes modal

### Site overview widget (HTMX lazy load)
```html
<div hx-get="/metrika/{{ site.id }}/widget" hx-trigger="load" hx-swap="innerHTML"></div>
```
Returns `_widget.html` partial or empty string if not configured.

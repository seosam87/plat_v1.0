## Deviations from Plan

All deviations are **plan-19.1-03 follow-ups** — the runner infra was landed without a real end-to-end smoke scenario, so several fatal bugs were only discoverable when the first real YAML tried to run. Fixed auto-inline.

### Auto-fixed Issues

**1. [Rule 1 - Bug] Collector did not wire `scenario_page` fixture**
- **Found during:** Task 1 (first run against real scenarios)
- **Issue:** `ScenarioItem.runtest()` did `self._request.getfixturevalue("scenario_page")` but `self._request` was never initialized on the custom `pytest.Item`. Every real scenario silently fell through to `page=None` and failed with "op='open' requires a Playwright page".
- **Fix:** In `ScenarioItem.__init__`, build a `FuncFixtureInfo` via `session._fixturemanager.getfixtureclosure(("scenario_page",), ...)`, attach as `self._fixtureinfo`, assign `self.fixturenames`, instantiate `_pytest.fixtures.TopRequest(self, _ispytest=True)` as `self._request`, and override `setup()` to call `self._request._fillfixtures()`. Also added a dummy `self.obj = lambda scenario_page=None: None` so `FixtureLookupError.formatrepr` doesn't crash when a lookup does fail.
- **Files modified:** `tests/fixtures/scenario_runner/collector.py`
- **Commit:** `17938e5`

**2. [Rule 1 - Bug] Cross-event-loop Playwright failures**
- **Found during:** Task 1
- **Issue:** Plan 19.1-03's `conftest.py` used session-scope `@pytest_asyncio.fixture` for browser/page, but `ScenarioItem.runtest()` ran its own `asyncio.run(...)`. pytest-asyncio created the Playwright objects on loop L1; `asyncio.run` created L2. Every Playwright method raised `ValueError: The future belongs to a different loop`.
- **Fix:** Rewrote `tests/fixtures/scenario_runner/conftest.py` so `scenario_page` is a **sync** `@pytest.fixture` that creates its own `asyncio.new_event_loop()`, performs playwright setup + login + context creation on that loop, stashes the loop on `page._scenario_loop`, and cleans up on teardown. `collector.runtest` now uses that stashed loop via `loop.run_until_complete(...)` instead of `asyncio.run`. Single loop end-to-end.
- **Files modified:** `tests/fixtures/scenario_runner/conftest.py`, `tests/fixtures/scenario_runner/collector.py`
- **Commit:** `17938e5`

**3. [Rule 3 - Unblocker] scenario_page fixture invisible to items under `scenarios/`**
- **Found during:** Task 1
- **Issue:** Items collected from `scenarios/*.yaml` live outside `tests/`, so `tests/fixtures/scenario_runner/conftest.py` was never loaded for them. Fixture lookup for `scenario_page` errored with "fixture not found".
- **Fix:** Root `conftest.py` now re-imports `base_url` and `scenario_page` from `tests.fixtures.scenario_runner.conftest` so they become top-level fixtures visible to any item. Attempted plugin-registration approach first, but pytest auto-imports the conftest as a module too, producing a duplicate-plugin `ValueError` — re-export is the clean path.
- **Files modified:** `conftest.py`
- **Commit:** `17938e5`

**4. [Rule 1 - Bug] Login fixture used wrong field name**
- **Found during:** Task 1
- **Issue:** Plan 19.1-03 `storage_state` fixture did `get_by_label("Username")`, but `templates/login.html` has `<input name="email">` and the seeded user logs in with email `smoke@example.com`.
- **Fix:** Login now uses `Email` label, `smoke@example.com`, and `Sign In` button. Retired the separate `storage_state` fixture — login now happens inline inside `scenario_page` to avoid cross-fixture loop complexity.
- **Files modified:** `tests/fixtures/scenario_runner/conftest.py`
- **Commit:** `17938e5`

**5. [Rule 1 - Bug] executor.open passed relative URLs to page.goto**
- **Found during:** Task 1
- **Issue:** `await page.goto("/ui/sites/new")` raised "Cannot navigate to invalid URL" because Playwright requires an absolute URL.
- **Fix:** `executor.run_scenario` now prepends `page._scenario_base_url` (stashed by `scenario_page`) to any `open.url` starting with `/`. Scenarios stay host-agnostic.
- **Files modified:** `tests/fixtures/scenario_runner/executor.py`
- **Commit:** `17938e5`

**6. [Rule 3 - Unblocker] Redis suggest cache not seeded**
- **Found during:** Task 1
- **Issue:** The seeded `SuggestJob` row has `status=complete` but no `cache_key` and no Redis payload. A fresh search for "smoke seed" from the scenario dispatched a Celery task that had no live proxies to hit, making the scenario's outcome probabilistic on CI.
- **Fix:** `tests/fixtures/scenario_runner/seed.py` now pre-writes a `suggest:y:smoke seed` Redis key with 3 deterministic rows (`smoke seed alpha/beta/gamma`), both on first seed and on re-runs when the DB seed short-circuits (Redis may have been flushed between runs).
- **Files modified:** `tests/fixtures/scenario_runner/seed.py`
- **Commit:** `17938e5`

**7. [Rule 3 - Unblocker] docker-compose.ci.yml api healthcheck used curl, not in image**
- **Found during:** Task 1
- **Issue:** `curl -fsS http://localhost:8000/health` failed with "curl: not found" because the api's `python:3.12-slim` base image doesn't ship curl. Stack never reached `Healthy` and the tester never started.
- **Fix:** Replaced with `python -c "import urllib.request,sys; sys.exit(0 if urllib.request.urlopen('http://localhost:8000/health',timeout=15).status==200 else 1)"`. Also bumped `interval` to 15s, `timeout` to 20s, `retries` to 10, `start_period` to 30s — `/health` touches db/redis/celery and can exceed 3–5s under cold reload.
- **Files modified:** `docker-compose.ci.yml`
- **Commit:** `17938e5`

**8. [Rule 3 - Unblocker] tester container missing pytest + playwright**
- **Found during:** Task 1
- **Issue:** `pip install -e .` tripped setuptools flat-layout discovery (picks up `scenarios/`, `alembic/`, `logs/` as top-level packages). After switching to `pip install -r requirements.txt`, pytest was still missing, then playwright itself (base image ships browsers under `/ms-playwright` but not the Python package in the default interpreter).
- **Fix:** `tester.command` now runs `pip install --no-cache-dir -r requirements.txt pytest pytest-asyncio 'playwright==1.47.0'` before pytest. playwright pinned to the exact version whose browsers the base image ships.
- **Files modified:** `docker-compose.ci.yml`
- **Commit:** `17938e5`

## Accomplishments

- 9 Step classes as a Pydantic v2 discriminated union: 6 P0 (open/click/fill/wait_for/expect_text/expect_status) + 3 reserved for 19.2 (say/highlight/wait_for_click). `extra="forbid"` on every class catches YAML typos. Scenario.steps is `list[Step]` with `min_length=1`.
- `pytest_collect_file` hook gated on `file_path.parent.name == "scenarios"` so only YAMLs directly under a scenarios/ directory are collected. ScenarioFile/ScenarioItem wrap parsed Scenario into a pytest Item whose `runtest` delegates to `asyncio.run(run_scenario(page, scenario))`.
- Executor skeleton iterates `scenario.steps`: reserved ops are logged via loguru at WARNING and skipped; P0 ops raise `NotImplementedError("... see plan 19.1-03")` as the explicit hook for the next plan.
- Plugin registered twice — in `tests/conftest.py` (unit-test path) and in new root-level `conftest.py` (so `pytest --collect-only scenarios/` works when the target is outside the tests/ tree). See Deviations.
- Committed `scenarios/_test_fixture.yaml` (reserved-only; will be deleted by plan 19.1-05 Task 0, per the forward note in 19.1-02-PLAN.md).
- 13/13 unit tests green; `pytest --collect-only scenarios/` lists `collector_smoke` from the fixture YAML.

## Decisions Made

- **Root-level conftest.py:** Required so the collector plugin loads when pytest is invoked with a path outside `tests/` (e.g. `pytest --collect-only scenarios/`). `tests/conftest.py` alone is not loaded for targets outside its subtree. Both registrations kept for clarity; pytest de-duplicates.
- **Task order flip (2 ↔ 3):** Executor implemented before collector so the collector's `from .executor import run_scenario` import resolves during the RED→GREEN cycle without a stub. No behavioral impact; file list matches the plan exactly.
- **loguru → caplog bridge:** Executor tests add a propagating stdlib handler to loguru temporarily so pytest's built-in `caplog` captures WARNING records without introducing a loguru-specific fixture.

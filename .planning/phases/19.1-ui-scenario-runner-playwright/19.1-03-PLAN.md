    4. Update collector.py's `ScenarioItem.runtest` to drive run_scenario + save_failure_artifacts under a SINGLE event loop. DO NOT call `asyncio.run` twice — the Page object is bound to the loop in which its BrowserContext was created, so a second `asyncio.run` would operate on a closed loop / closed Page and raise "Event loop is closed" / "Page is already closed". Instead, define an inner async helper and wrap both operations in one `asyncio.run`:

       ```python
       async def _run_with_artifacts(page, scenario):
           try:
               await run_scenario(page, scenario)
           except Exception:
               await save_failure_artifacts(page, scenario.name)
               raise
           else:
               # success path: stop tracing without writing a trace file
               await page.context.tracing.stop()

       page = self._request.getfixturevalue("scenario_page")
       asyncio.run(_run_with_artifacts(page, self.scenario))
       ```

       This guarantees the scenario run and the failure-artifact capture share the same event loop and the same Page/Context instance.

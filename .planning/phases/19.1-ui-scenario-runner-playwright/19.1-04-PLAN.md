  <action>
    Create /opt/seo-platform/docker-compose.ci.yml as a compose overlay (no `version:` field — modern compose). Two service entries:

    1. `worker` (override) — add a healthcheck only; do NOT re-declare image/command/env (overlay merges with base):
       ```yaml
       services:
         worker:
           healthcheck:
             test: ["CMD-SHELL", "celery -A app.celery_app inspect ping -d celery@$$HOSTNAME || exit 1"]
             interval: 10s
             timeout: 10s
             retries: 5
             start_period: 20s
       ```
       Note the doubled `$$HOSTNAME` — compose needs the escape.

    2. `tester` (new service) using the Microsoft Playwright Python image:
       ```yaml
         tester:
           image: mcr.microsoft.com/playwright/python:v1.47.0-jammy
           working_dir: /app
           volumes:
             - .:/app
           environment:
             BASE_URL: http://api:8000
             DATABASE_URL: ${DATABASE_URL}
             PYTHONPATH: /app
           depends_on:
             api:
               condition: service_healthy
             worker:
               condition: service_healthy
           command: >
             sh -c "pip install --no-cache-dir -e . &&
                    pytest scenarios/ tests/fixtures/scenario_runner/ -v --tb=short"
       ```
       The `pip install -e .` picks up the test deps declared in pyproject.toml by plan 01 (playwright + pyyaml). Chromium is already in the base image.

    Note: the base `docker-compose.yml` `api` service has no healthcheck today. If plan 04 needs `api:service_healthy` and it's missing, ALSO add an `api` healthcheck in the overlay: `curl -fsS http://localhost:8000/health || exit 1` with similar interval/timeout. Check the base file first; only add if absent.

  <action>
    Create scripts/run-scenarios-ci.sh (chmod +x) implementing the sequence from RESEARCH.md "CI compose lifecycle":
    ```bash
    #!/usr/bin/env bash
    set -euo pipefail
    cd "$(dirname "$0")/.."

    COMPOSE="docker compose -f docker-compose.yml -f docker-compose.ci.yml"

    echo "==> Bringing up stack (postgres, redis, api, worker, beat)"
    $COMPOSE up -d --wait postgres redis api worker beat

    echo "==> Seeding live DB (idempotent)"
    $COMPOSE exec -T api python -m tests.fixtures.scenario_runner.seed

    echo "==> Running scenarios in tester container"
    # --abort-on-container-exit so the whole compose run exits when tester finishes
    # --exit-code-from tester so this script's exit code reflects pytest's
    $COMPOSE up --abort-on-container-exit --exit-code-from tester tester
    TESTER_EXIT=$?

    if [ $TESTER_EXIT -ne 0 ]; then
      echo "==> Scenario failures — artifacts at artifacts/scenarios/"
      ls -la artifacts/scenarios/ || true
    fi

    exit $TESTER_EXIT
    ```
    Make executable: `chmod +x scripts/run-scenarios-ci.sh`.

**app/services/batch_wordstat_service.py** — NEW file per D-11 (separate from wordstat_service.py):

`fetch_wordstat_batch_sync(phrases: list[str], oauth_token: str, region_id: int = 0, timeout: int = 30) -> list[dict]`:
- Process phrases one at a time (Wordstat API limitation)
- For each phrase, make TWO API calls (per Pitfall 4):
  1. Exact match: POST to `{WORDSTAT_API_BASE}/v1/topRequests` with `"phrase": f'"{phrase}"'` (quoted for exact)
  2. Broad match: POST with `"phrase": phrase` (unquoted for broad)
- Parse response to get frequency values
- Also fetch monthly dynamics: POST to `{WORDSTAT_API_BASE}/v1/topRequests` with time dimension parameter to get per-month breakdown (if API supports it), OR use a separate endpoint for history
- Return list of dicts: `{"phrase": str, "freq_exact": int, "freq_broad": int, "monthly": [{"year_month": str, "frequency": int}]}`
- On per-phrase API error: log warning, set freq_exact=None, freq_broad=None, continue (don't fail entire batch)
- Use `httpx.Client` with connection pooling (same pattern as wordstat_service.py)
- `time.sleep(0.3)` between API calls to respect rate limits

`check_wordstat_oauth_token(db_session) -> str | None`:
- Load Yandex Direct OAuth token from service_credentials table
- Return token string or None if not configured
- Used by router to show/hide OAuth warning banner

**app/tasks/wordstat_batch_tasks.py** — Single Celery task with progress:

`run_wordstat_batch(self, job_id: str)`:
- `@celery_app.task(name="app.tasks.wordstat_batch_tasks.run_wordstat_batch", bind=True, max_retries=3, soft_time_limit=3600, time_limit=3660)` — long timeout for 1000 phrases
- Load WordstatBatchJob, set status="running", commit
- Get OAuth token via `get_credential_sync("wordstat_oauth")` or similar pattern from existing code
- If no token: set status="failed", error_message="OAuth-токен не настроен", commit, return
- Process phrases in batches of 10:
  - For each phrase: call batch_wordstat_service to get freq_exact, freq_broad, monthly data
  - Create WordstatBatchResult row
  - For each monthly entry: create WordstatMonthlyData row linked to result.id
  - Every 50 phrases: update job.result_count = processed_count, job.progress_pct = int(processed / total * 100), commit (per D-12)
- On API rate limit (429): wait 60s and retry batch
- On completion: set status="complete" (or "partial" if some phrases failed), result_count, completed_at, commit
- On total failure: status="failed", error_message, commit

**tests/test_batch_wordstat_service.py**:

- `test_fetch_wordstat_batch_sync_handles_api_error`: mock 500 response for one phrase, verify None values and continued processing
- `test_exact_vs_broad_different_quotes`: verify exact call sends `'"phrase"'` (quoted) vs broad sends `'phrase'` (unquoted) — per Pitfall 4
- `test_monthly_data_parsed`: verify monthly dynamics list is returned correctly
- Use `respx` or `unittest.mock.patch` for httpx mocking

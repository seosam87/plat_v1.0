**app/services/brief_top10_service.py** — NEW file, per D-05 separate lightweight crawler:

`crawl_top10_page(url: str) -> dict | None`:
- Uses `get_browser()` from `app.celery_app`
- Creates new BrowserContext + Page per URL
- `pw_page.goto(url, wait_until="domcontentloaded", timeout=20_000)`
- If response is None or status >= 400, return None (D-06: skip silently)
- Extract: `h2s` = list of h2 inner_text, `visible_text` = pw_page.inner_text("body")[:5000]
- Always close page + context in finally block (Pitfall 2)
- On any exception: return None (D-06)

`aggregate_brief_data(crawled_pages: list[dict], phrases: list[str]) -> dict`:
- Input: list of crawled page dicts (each has "h2s", "text" keys), list of input phrases
- H2 cloud: Counter of all h2 texts across pages, sorted by count desc -> list of {"text": str, "count": int}
- Thematic words: tokenize all visible_text with `re.findall(r'\b[а-яА-ЯёЁa-zA-Z]{3,}\b', text)`, Counter, filter stopwords (common Russian: "что", "как", "для", "это", "при", "или", "его", "все", "они", "она", etc — hardcode ~30 stopwords), top 100 by freq -> list of {"word": str, "freq": int}
- Highlights: from XMLProxy SERP snippets (passed separately), deduplicated list of bold/em text
- Title suggestions: extract `<title>` from crawled pages, deduplicated list
- Volume stats: avg_text_length = mean(len(text) for each page), avg_h2_count = mean(len(h2s)), pages_crawled = count of non-None results, pages_attempted = total URLs attempted
- Commercialization %: count pages with price/buy indicators in text ("цена", "купить", "заказать", "стоимость", "корзина", "добавить в корзину") / total pages * 100
- Return dict with keys matching BriefResult columns

**app/tasks/brief_tasks.py** — 4 Celery tasks for chain per D-02:

1. `run_brief_step1_serp(self, job_id: str) -> str`: 
   - `@celery_app.task(name="app.tasks.brief_tasks.run_brief_step1_serp", bind=True, max_retries=3, soft_time_limit=300, time_limit=360)`
   - Load BriefJob by UUID, set status="running"
   - For each phrase in input_phrases: call `search_yandex_sync(phrase, region_id=job.input_region)` from xmlproxy_service
   - Collect TOP-10 URLs per phrase (max 10 URLs per phrase)
   - Store URLs in job record as JSONB field (add `_serp_urls` transient or use a separate mechanism — store in BriefResult.highlights temporarily, or add `intermediate_data` JSONB column to BriefJob). Simplest approach: add `_intermediate_data` JSONB nullable column to BriefJob model.
   - Actually, cleaner: store intermediate data in BriefJob.error_message field is wrong. Better: add `intermediate_data: Mapped[dict | None] = mapped_column(JSONB, nullable=True)` to BriefJob model in Task 1.
   - On XMLProxyError: retry with `self.retry(exc=exc)`
   - On complete success: commit, return job_id
   - On unrecoverable failure: set status="failed", error_message, commit

2. `run_brief_step2_crawl(self, job_id: str) -> str`:
   - `@celery_app.task(... soft_time_limit=900, time_limit=960)` — longer limit for Playwright crawling (Pitfall 7)
   - Load BriefJob, read intermediate_data for URL list
   - For each URL: call `crawl_top10_page(url)` from brief_top10_service
   - Collect results (skip None per D-06)
   - Update intermediate_data with crawled page data
   - `time.sleep(0.5)` between pages to avoid overwhelming targets
   - On failure: set status="failed", commit

3. `run_brief_step3_aggregate(self, job_id: str) -> str`:
   - `@celery_app.task(... soft_time_limit=120, time_limit=180)`
   - Load BriefJob, read intermediate_data (crawled pages + SERP snippets)
   - Call `aggregate_brief_data(crawled_pages, job.input_phrases)`
   - Create BriefResult row from aggregation output
   - Update intermediate_data or clear it
   - Commit, return job_id

4. `run_brief_step4_finalize(self, job_id: str) -> str`:
   - `@celery_app.task(... soft_time_limit=30, time_limit=60)`
   - Load BriefJob, set status="complete", completed_at=now, result_count=1
   - Clear intermediate_data (set to None to save space)
   - Commit

Each step: wrap in try/except, on failure set job.status="failed" + job.error_message before re-raising (Pitfall 5).

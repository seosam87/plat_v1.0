## Visual Focal Points

**Landing page (tool input form):** The primary visual anchor is the phrase textarea + CTA button group. The textarea occupies the dominant vertical space of the content area; the CTA button (`btn-primary`, indigo-600 background, full-width on mobile) sits immediately below it. The user's eye lands here first on every tool's landing page.

**Results page:** The primary visual anchor depends on job status. When status is `pending` or `running`, the status banner (blue-tinted, top of content area, full-width card) is the focal point — it is the only element that changes dynamically via HTMX polling. When status is `complete`, the first section card (results table or, for Copywriting Brief, the Title/H1 recommendations card) becomes the focal point; the download export button is placed prominently at the top-right of the results header to draw the eye toward the primary action.

### Additional UI elements per tool

**Copywriting Brief (`brief`):**
- Region selector: `<select>` dropdown (Yandex region code) — same style as domain input field
- Phrase textarea: 8 rows, `input_type = 'phrases'`, limit 30
- Results: tabbed or accordion sections — Title/H1 suggestions, H2 headings cloud (frequency-sorted word list), Search highlights list, Thematic words table (word + frequency), Volume stats row (avg text length, avg headings count, commercialization %)
- XLSX export only (no CSV for Brief) — single download button labeled "Скачать ТЗ (XLSX)"
- Progress polling: 10s interval inherited from `job_status.html`

**PAA Parser (`paa`):**
- Phrase textarea: 8 rows, `input_type = 'phrases'`, limit 50
- Results: flat table with columns — Фраза / Вопрос / Блок (частые вопросы / похожие запросы)
- CSV + XLSX export (same as existing tools)
- No region field required (XMLProxy defaults to Yandex Russia)

**Batch Wordstat (`wordstat-batch`):**
- Phrase textarea: 10 rows, `input_type = 'phrases'`, limit 1000
- OAuth warning banner: shown when no Yandex Direct token configured — amber banner above form: "Для работы инструмента необходим Яндекс Direct OAuth-токен. [Перейти в настройки →]" linking to settings page
- Results: flat table with columns — Фраза / Частота (точная) / Частота (широкая) / Динамика
- Dynamics column: inline sparkline OR text range ("дек: 1200 → мар: 1450") — use text representation (no chart library needed)
- Progress polling: shows percentage "Обработка... 450/1000 фраз · 45%" — extend `job_status.html` to use `progress_pct` field when present
- XLSX export only (monthly dynamics data is multi-row per phrase — CSV flattening is complex, defer)

## Interaction Contract

All interactions follow the established Phase 24 HTMX polling pattern. No deviation allowed.

### Form submit flow (all three tools)

1. User fills textarea + optional fields → clicks CTA button
2. Form POSTs synchronously to `/ui/tools/{slug}/`
3. Server creates Job record → redirects to `/ui/tools/{slug}/{job_id}`
4. Results page loads with `job_status.html` partial in pending/running state
5. HTMX polls `/ui/tools/{slug}/{job_id}/status` every 10 seconds
6. When status = `complete` or `partial` → page reloads (full page swap) to show results table
7. When status = `failed` → error banner shown, no results table rendered

### Progress display for Batch Wordstat

The `job_status.html` partial receives `progress_pct` (integer 0–100) when the tool is `wordstat-batch`. Running state shows: "Проверяем частотность... {result_count}/{phrase_count} фраз · {progress_pct}%". Other tools: existing copy ("Обработка... {result_count}/{total} фраз").

### Copywriting Brief results layout

Results page for `brief` slug uses a sectioned layout instead of a flat table:

```
[Status banner]
[Section card: Title/H1 recommendations — ordered list]
[Section card: H2 headings cloud — word list sorted by frequency, badge-style chips]
[Section card: Yandex highlights — bulleted list]
[Section card: Thematic words — 2-column table: word | frequency]
[Section card: Volume stats — 4-column stat row: avg_text_length | avg_h2_count | commercialization_pct | pages_crawled]
[Download button: Скачать ТЗ (XLSX)]
```

Each section card uses the `.card` class with `mb-4`. Section headings use `text-lg font-semibold text-gray-800`.

### Delete job (all tools)

- Delete button: text-only (`text-red-600 hover:text-red-800`), no background, `min-height: 44px`
- Uses `hx-confirm` native HTMX dialog — no custom modal
- On confirm: `hx-delete` removes the `<tr>` via `hx-swap="outerHTML"`

### Re-run button (Plan 25-05)

- POST to `/ui/tools/{slug}/rerun/{job_id}` — creates new Job with same input
- Button style: secondary — border only, `border border-indigo-200 rounded px-3 py-1 text-sm text-indigo-600`

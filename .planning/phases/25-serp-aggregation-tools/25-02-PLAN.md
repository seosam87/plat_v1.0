**Router changes in app/routers/tools.py:**

1. **tool_submit** — handle `has_region_field`:
   After the existing `has_domain_field` block, add:
   ```python
   if registry.get("has_region_field"):
       region = form_data.get("region", "213") or "213"
       try:
           job_kwargs["input_region"] = int(region)
       except (ValueError, TypeError):
           job_kwargs["input_region"] = 213
   ```

2. **tool_submit** — handle chain dispatch for "brief":
   Replace the generic `task_fn = _get_tool_task(slug); task_fn.delay(str(job.id))` block with:
   ```python
   if slug == "brief":
       from celery import chain as celery_chain
       from app.tasks.brief_tasks import (
           run_brief_step1_serp, run_brief_step2_crawl,
           run_brief_step3_aggregate, run_brief_step4_finalize,
       )
       pipeline = celery_chain(
           run_brief_step1_serp.si(str(job.id)),
           run_brief_step2_crawl.si(str(job.id)),
           run_brief_step3_aggregate.si(str(job.id)),
           run_brief_step4_finalize.si(str(job.id)),
       )
       pipeline.delay()
   else:
       task_fn = _get_tool_task(slug)
       task_fn.delay(str(job.id))
   ```

3. **tool_export** — handle Brief XLSX-only export:
   In the export handler, add a branch for slug == "brief":
   - Load BriefResult for the job (single row)
   - Create openpyxl Workbook with sheets:
     - Sheet "Title-H1": one title per row from title_suggestions
     - Sheet "H2": columns "Заголовок" / "Частота" from h2_cloud
     - Sheet "Подсветки": one highlight per row
     - Sheet "Тематические слова": columns "Слово" / "Частота" from thematic_words
     - Sheet "Объём": single row with avg_text_length, avg_h2_count, commercialization_pct, pages_crawled, pages_attempted
   - Return StreamingResponse with content_type `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
   - Filename: `brief_{job_id}.xlsx`

4. **tool_results** — pass BriefResult to template:
   In tool_results handler, when slug == "brief", load BriefResult (single row per job) and pass as `result` to template context alongside the job.

5. **_EXPORT_HEADERS** — add "brief" key (even though export is custom, needed for the guard):

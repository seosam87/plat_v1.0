## Objective

Set up Playwright browser lifecycle management in Celery workers via process-level signals, add a dedicated crawler Docker service, and update crawl_site task with browser context management.

## Tasks

1. Add `CRAWLER_DELAY_MS` and `CRAWLER_MAX_PAGES` to `app/config.py`
2. Add `worker_process_init` / `worker_process_shutdown` Celery signals to `app/celery_app.py` for module-level Browser init/teardown
3. Update `app/tasks/crawl_tasks.py` with site_active_guard, BrowserContext per task, soft time limit
4. Create `Dockerfile.crawler` using Playwright base image
5. Add `crawler` service to `docker-compose.yml` (crawl queue only, concurrency=2)
6. Update `worker` service to exclude crawl queue (wp,default only)

# Plan v3-05-03: Gap router, UI, and integration

<objective>
Create the gap analysis router with endpoints for gap detection, file import, keyword management, group management, and proposal workflow. Build the gap UI page with keyword table, grouping, proposals, and file upload. Add "Gap-анализ" button to site detail.
</objective>

<tasks>

<task id="01">
<title>Create gap router</title>
<read_first>
- app/routers/analytics.py (router pattern)
- app/services/gap_service.py
</read_first>
<action>
Create `app/routers/gap.py`:

Router: `prefix="/gap"`, tags `["gap"]`

**Page:**
1. `GET /{site_id}` — Main gap analysis page (HTML). Shows gap keywords, groups, proposals.

**Gap detection:**
2. `POST /{site_id}/detect` — Run gap detection from a session. Body: `{session_id, competitor_domain}`.
3. `POST /{site_id}/import` — Upload CSV/XLSX file with competitor keywords. Multipart form: file + competitor_domain field.

**Gap keywords:**
4. `GET /{site_id}/keywords` — List gap keywords with filters (competitor, group, min_score). JSON.
5. `DELETE /{site_id}/keywords` — Bulk delete. Body: `{keyword_ids: []}`.

**Groups:**
6. `POST /{site_id}/groups` — Create group. Body: `{name}`.
7. `GET /{site_id}/groups` — List groups.
8. `PUT /{site_id}/groups/assign` — Assign keywords to group. Body: `{keyword_ids, group_id}`.
9. `DELETE /{site_id}/groups/{group_id}` — Delete group.

**Proposals:**
10. `POST /{site_id}/proposals` — Create proposals from selected gap keywords. Body: `{keyword_ids: []}`.
11. `GET /{site_id}/proposals` — List proposals with status filter.
12. `POST /proposals/{proposal_id}/approve` — Approve proposal. Body: `{project_id?}`.
13. `POST /proposals/{proposal_id}/reject` — Reject proposal.

**Score info:**
14. `GET /score-formula` — Returns SCORE_FORMULA_DESCRIPTION text.

All endpoints `require_admin`.
</action>

</task>

<title>Create gap analysis page template</title>
<read_first>
- app/templates/analytics/index.html (UI pattern)

</read_first>
<action>
Create `app/templates/gap/index.html`:

Extends `base.html`.

**1. Header:** "Gap-анализ: {site.name}" + back link

**2. Import section** (card):
- Source selector: "Из сессии аналитики" / "Загрузить файл (CSV/XLSX)"
- Session mode: session dropdown + competitor domain input + "Найти gap-ключи" button
- File mode: file input + competitor domain input + "Загрузить" button
- Status message area

**3. Summary stats** (3 cards):
- Gap-ключей: total count — red #dc2626
- Средний потенциал: avg score — blue #2563eb
- Предложений: proposals count — green #059669

**4. Gap keywords table** (card):
- Columns: ☐ | Фраза | Частотность | Позиция конкурента | Наша позиция | Потенциал (?) | Группа | Источник

- Filter by: group (select), min score (input), competitor domain (select)
- Sort by: potential_score desc (default), frequency, competitor_position
- "Выбрать все" toggle
- Action buttons: "Создать группу" (modal), "Добавить в группу" (select), "Создать предложения"

**5. Groups card** (collapsible):
- List of groups with keyword count per group
- Delete button per group

**6. Proposals card:**
- Table: Название | Ключ | Частотность | Потенциал | Статус | Действия
- Status badges: pending=yellow, approved=green, rejected=gray
- Action buttons per row: "Одобрить" / "Отклонить"
- Approved proposals show link to content plan item

**Tooltip for potential score:**
```html
<span title="{{ score_formula }}" style="cursor:help; color:#6b7280; font-size:.85rem">?</span>
```

All copy in Russian. JS for selection, grouping, file upload, proposal actions.

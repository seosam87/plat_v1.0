**Fix 3 — Dashboard status badges (UAT Test 1):**

In `app/services/dashboard_service.py`, the `projects_table()` function builds a list of dicts from SQL rows (around line 75). The `row["status"]` comes from a raw SQL query so it may be a SQLAlchemy enum object rather than a plain string. The Jinja2 template compares `p.status == 'active'` which fails against an enum.

Convert the status to a string in the dict comprehension at line 75:
```python
"status": str(row["status"].value) if hasattr(row["status"], "value") else str(row["status"]),
```

This ensures the template comparison `p.status == 'active'` matches correctly, rendering the colored badge spans (emerald for active, amber for paused, gray for completed).

No template changes needed — the existing `dashboard/index.html` lines 150-158 already have correct badge markup with `{% if p.status == 'active' %}` conditionals.

**Fix 4 — Report schedule page styling (UAT Test 5):**

**Fix 5 — Browser page icon sizing (UAT Test 3 cosmetic):**

**Investigation result:** The WeasyPrint PDF templates are `reports/brief.html` and `reports/detailed.html` (confirmed via `report_service.py` line 191: `template_name = "reports/brief.html" if report_type == "brief" else "reports/detailed.html"`). These templates contain **zero SVG icons or images** — they are pure HTML/CSS table reports rendered by WeasyPrint. The "icons stretched across the whole page" issue the user reported is about the **browser page** `reports/generate.html`, which has SVG icons in the PDF and Excel download buttons (lines 53, 63). This is a browser rendering issue, not a PDF output issue.

Add explicit `width` and `height` attributes to the SVG elements in `reports/generate.html` as a safety net for rendering engines that ignore Tailwind CSS classes (e.g., when Tailwind CSS CDN is slow to load, or in certain zoom levels):
```html

```

Apply this to both SVG icons (the PDF download button SVG at line 53 and the Excel download button SVG at line 63). Also add `flex-shrink-0` to prevent flex layout from squishing/stretching the icons:
```html
<svg class="w-4 h-4 flex-shrink-0" width="16" height="16" fill="none" stroke="currentColor" viewBox="0 0 24 24">
```

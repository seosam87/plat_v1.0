       - After computing delta_sessions_pct, delta_conversions_pct, delta_cost_pct, add:
         ```python
         cr_a = (va["conversions"] / va["sessions"] * 100) if va["sessions"] > 0 else 0.0
         cr_b = (vb["conversions"] / vb["sessions"] * 100) if vb["sessions"] > 0 else 0.0
         cpc_a = (va["cost"] / va["conversions"]) if va["conversions"] > 0 else None
         cpc_b = (vb["cost"] / vb["conversions"]) if vb["conversions"] > 0 else None
         delta_cr_pct = _delta(cr_a, cr_b)
         delta_cpc_pct = _delta(cpc_a, cpc_b) if cpc_a and cpc_b else None
         ```
       - Add these to the comparison dict per source: `cr_a`, `cr_b`, `delta_cr_pct`, `cpc_a`, `cpc_b`, `delta_cpc_pct`.

    2. Add new function `ad_traffic_trend()` in app/services/report_service.py:
       - `async def ad_traffic_trend(db: AsyncSession, site_id: UUID, granularity: str = "weekly") -> dict`:
       - Query AdTraffic grouped by source and date_trunc(granularity, traffic_date):
         ```sql
         SELECT source, date_trunc(:gran, traffic_date) AS period,
                SUM(sessions) AS sessions, SUM(conversions) AS conversions, SUM(cost) AS cost
         FROM ad_traffic WHERE site_id = :sid
         GROUP BY source, period ORDER BY period
         ```
       - Transform into Chart.js-compatible format:
         ```python
         return {
             "labels": sorted_unique_periods,  # list of date strings
             "datasets": [
                 {"label": source, "data": [sessions_per_period...]}
                 for source in sorted_sources
             ]
         }
         ```

    3. Add trend API endpoint in app/routers/reports.py:
       - `GET /reports/sites/{site_id}/ad-traffic/trend?granularity=weekly|monthly`:
         - Calls `report_service.ad_traffic_trend(db, site_id, granularity)`.
         - Returns the dict directly (JSON).

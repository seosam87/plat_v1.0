## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, cell padding-top/bottom |
| sm | 8px | Button inner padding (vertical), inline element spacing |

## Layout Contracts

### Desktop — Matrix View (`qa/index.html`)

Extends `base.html`. Single `.card` full-width. Table layout:

```
[ Page heading ]
[ Subheading + "Add User Flow" button (right-aligned) ]
[ Table ]
  Columns: User Flow | Retest (days) | Desktop | Mobile | Telegram | Actions
  Row: feature.name | feature.retest_days | [status cell] × 3 | Edit / Delete
[ Status cell ]
  - Background color from STATUS_COLORS map
  - Text: status label (12.8px, weight 600)
  - Click → HTMX hx-get="/ui/qa/checks/{id}/edit" hx-target="this" hx-swap="innerHTML"
  - Min-height 44px on all surfaces for touch target compliance
```

Pagination: none — maximum 20 entries (D-01); full table rendered on one page.

### Desktop — Feature Form (`qa/feature_form.html`)

Extends `base.html`. Single `.card` with max-width 640px. Fields:

```
Name (text, required)
Slug (text, required, auto-populated from name, editable)
Description (textarea, optional)
Retest every [N] days (number input, min=1, default=30)
[ Save ] [ Cancel ]
```

### Desktop — Check Edit Cell Fragment (`qa/_check_cell.html`)

HTMX partial — not a full page. Rendered inline inside a `<td>`. Contains:

```
[ Status dropdown: not_tested / passed / failed / needs_retest ]
[ Notes textarea (optional, 2 rows) ]
[ Mark as Tested button → POST /ui/qa/checks/{id}/mark-tested ]
[ Cancel (hx-get to restore original cell content) ]
```

### Desktop — Candidates Page (`qa/candidates.html`)

Extends `base.html`. Two-panel layout within a `.card`:

```
Left/Top panel: Discovered routes list (route path, tier classification, checkbox)
Right/Bottom panel: Form — Flow Name (text), Description (textarea), [ Add User Flow ]
Routes grouped by prefix: /ui/, /m/, /api/ (if any HTML)
```

### Mobile / Telegram Surface

The QA matrix is an admin tool — it is NOT exposed on the mobile (`/m/`) surface or as a Telegram Mini App page. The mobile/Telegram surfaces are **targets of QA testing**, not operators of the QA tool itself. No mobile templates are needed for this phase.

**Source:** CONTEXT.md D-02 (SurfaceCheck tracks state per surface independently); RESEARCH.md — router prefix is `/ui/qa/` (admin namespace, not `/m/`)

## Interaction Contracts

### HTMX Patterns

| Trigger | Endpoint | Target | Swap | Fallback |
|---------|----------|--------|------|---------|
| Click status cell | GET `/ui/qa/checks/{id}/edit` | `<td id="check-{id}">` | innerHTML | Full page redirect if JS disabled |
| Submit check form | POST `/ui/qa/checks/{id}/mark-tested` | `<td id="check-{id}">` | outerHTML | Form POST redirect |
| Submit feature form | POST `/ui/qa/features` | `#qa-table-body` | beforeend | Full page redirect |
| Delete feature | DELETE `/ui/qa/features/{id}` | `tr#feature-{id}` | outerHTML (delete) | Form POST redirect |

### Loading States

- All HTMX-triggered cells show `.htmx-indicator` spinner (existing platform class) during request
- "Mark as Tested" button disabled after click until response returns (prevent double-submit via `hx-disabled-elt="this"`)

### Status Transition Rules (UI enforcement mirrors backend)

- "Mark as Tested" button always available regardless of current status
- After POST returns, cell background updates immediately to `passed` (#d1fae5) with no page reload
- `needs_retest` cells display amber background with a clock icon (SVG inline, 14×14px) before the status label
- `failed` cells display red background with an X icon (SVG inline, 14×14px) before the status label

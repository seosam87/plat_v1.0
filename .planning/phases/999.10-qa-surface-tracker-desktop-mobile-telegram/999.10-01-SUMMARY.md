# Phase 999.10 Plan 01: QA Surface Data Layer Summary

**One-liner:** SQLAlchemy parent-child models (FeatureSurface + SurfaceCheck) with 9-function service layer including overdue scan that only transitions `passed` -> `needs_retest`.

## What Was Built

**Task 1: Models + Migration + Service Layer**

Created `app/models/qa_surface.py` with:
- `Surface` str enum: `desktop`, `mobile`, `telegram`
- `CheckStatus` str enum: `not_tested`, `passed`, `failed`, `needs_retest`
- `FeatureSurface` model: parent table with slug, name, description, retest_days, is_active
- `SurfaceCheck` model: child table with `UniqueConstraint("feature_id", "surface", name="uq_feature_surface")`
- All `updated_at` fields use `onupdate=lambda: datetime.now(timezone.utc)` (per D-03 pitfall #4)

Created `alembic/versions/0058_add_feature_surfaces_and_surface_checks_tables.py`:
- Manual migration (autogenerate blocked by pre-existing unrelated model issue)
- Creates `surface` and `checkstatus` PostgreSQL enum types via raw DDL
- Uses `postgresql.ENUM(create_type=False)` in `create_table` to reference pre-created types
- Migration verified: database is at version 0058, both tables exist

Created `app/services/qa_surface_service.py` with 9 functions:
1. `create_feature_surface` — auto-creates 3 SurfaceCheck child rows per D-02
2. `list_features_with_checks` — selectinload for N+1-free matrix query
3. `get_feature_by_id` — raises 404 if not found
4. `get_feature_by_slug` — returns None if not found
5. `update_feature_surface` — updates parent fields only
6. `delete_feature_surface` — cascade deletes children
7. `get_check_by_id` — raises 404 if not found
8. `mark_check_tested` — sets status, last_tested_at, notes, tested_by
9. `scan_overdue` — ONLY transitions `passed` -> `needs_retest` per D-03

**Task 2: Smoke Seed Integration**

- Added `feature_surface_id: a1a1a1a1-...` and `surface_check_id: a2a2a2a2-...` to `SMOKE_IDS`
- Added `FeatureSurface` + 3 `SurfaceCheck` rows (one per Surface enum) to `seed_extended()`
- Added `"/m/"` to `UI_PREFIXES` in `tests/_smoke_helpers.py` for mobile route smoke coverage

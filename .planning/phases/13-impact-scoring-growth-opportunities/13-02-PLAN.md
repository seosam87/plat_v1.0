    **`async def get_gap_summary(db: AsyncSession, site_id: uuid.UUID, limit: int = 100) -> dict`**
    - Query GapKeyword WHERE site_id, ORDER BY potential_score DESC NULLS LAST, LIMIT
    - Return `{"count": total_count, "total_potential_traffic": SUM(frequency) where not null, "items": [{"id": str(id), "phrase": phrase, "competitor_domain": competitor_domain, "competitor_position": competitor_position, "our_position": our_position, "potential_score": potential_score}]}`
    - If frequency is null for >50% of rows, set `"frequency_available": False` in response

    **`async def get_lost_positions(db: AsyncSession, site_id: uuid.UUID, limit: int = 200) -> dict`**
    - Raw SQL using `text()`:
      ```sql
      SELECT klp.keyword_id, k.phrase, klp.url, klp.position, klp.previous_position, klp.delta
      FROM keyword_latest_positions klp
      JOIN keywords k ON k.id = klp.keyword_id
      WHERE klp.site_id = :site_id AND klp.delta <= -5 AND klp.position IS NOT NULL
      ORDER BY klp.delta ASC LIMIT :limit
      ```
    - Return `{"count": len(items), "items": [{"keyword_id", "phrase", "url", "position", "previous_position", "delta"}]}`

    **`async def get_cannibalization(db: AsyncSession, site_id: uuid.UUID) -> dict`**
    - Raw SQL using `text()` — query keyword_latest_positions for keywords with 2+ distinct URLs where position <= 50 (per research: top-50 reduces noise):
      ```sql
      WITH multi_page AS (
          SELECT keyword_id, COUNT(DISTINCT url) AS page_count
          FROM keyword_latest_positions
          WHERE site_id = :site_id AND position IS NOT NULL AND position <= 50 AND url IS NOT NULL
          GROUP BY keyword_id HAVING COUNT(DISTINCT url) >= 2
      )
      SELECT klp.keyword_id, k.phrase, klp.url, klp.position
      FROM keyword_latest_positions klp
      JOIN keywords k ON k.id = klp.keyword_id
      JOIN multi_page mp ON mp.keyword_id = klp.keyword_id
      WHERE klp.site_id = :site_id AND klp.position <= 50
      ORDER BY klp.keyword_id, klp.position
      ```
    - Group results in Python by keyword_id into `{"keyword_id", "phrase", "pages": [{"url", "position"}], "page_count"}` 
    - Sort groups by page_count DESC
    - Return `{"count": len(groups), "items": groups}`

    **`def compute_visibility_trend(daily_rows: list[dict]) -> dict`** (pure function, no DB)
    - Takes list of `{"traffic_date": date, "visits": int}` rows
    - Compute current week (last 7 days) vs previous week (8-14 days ago) SUM(visits)
    - Compute current month (last 30 days) vs previous month (31-60 days ago) SUM(visits)
    - `week_change_pct = ((current - prev) / prev * 100)` if prev > 0 else 0.0
    - `month_change_pct = ((current - prev) / prev * 100)` if prev > 0 else 0.0
    - Return dict with all 6 values, rounded to 1 decimal

    **`async def get_visibility_trend(db: AsyncSession, site_id: uuid.UUID) -> dict`**
    - Query MetrikaTrafficDaily WHERE site_id AND traffic_date >= (today - 60 days) ORDER BY traffic_date
    - Pass to compute_visibility_trend() pure function

    **Tests** (`tests/test_opportunities_service.py`): Test pure functions (compute_visibility_trend, build_cannibalization_groups helper if extracted). Mock DB calls for async functions.

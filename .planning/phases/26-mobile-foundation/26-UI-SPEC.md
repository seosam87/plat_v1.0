## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Tailwind CSS utility classes via CDN) |
| Icon library | Inline SVG (existing pattern from base.html) |

**Rationale:** The project uses Tailwind CSS 3.x via CDN + inline Jinja2 templates throughout all 45 shipped phases. No shadcn/React — the stack is Python/FastAPI/Jinja2/HTMX. This is consistent with project constraints (see CLAUDE.md: "Jinja2 + HTMX — fixed, no substitutions."). Source: RESEARCH.md § Standard Stack.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, bottom-nav icon-to-label gap |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, page `<main>` padding (p-4 = 16px) |
| lg | 24px | Section padding, card padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Bottom nav bar height (fixed) |

**Exceptions:**

- Bottom nav bar height: **64px** (3xl token) — matches `padding-bottom: 64px` on `<body>` to prevent content overlap. Source: RESEARCH.md Pattern 1.
- Touch targets (all tappable elements): minimum **44px** height — iOS HIG minimum for accessible tap areas. Each bottom-nav tab, CTA button, and link-list item must meet this minimum.
- iPhone safe-area bottom padding: `env(safe-area-inset-bottom)` added to `.bottom-nav` in addition to 64px height — accounts for iPhone X+ home indicator gesture area. Source: RESEARCH.md Pitfall 7.

| Dominant (60%) | `#f5f5f5` | Page background (matches existing base.html body) |
| Secondary (30%) | `#1e1b4b` | Bottom navigation bar background, version-switch header strip |
| Accent (10%) | `#4f46e5` | Active bottom-nav tab indicator, primary CTA button background, hyperlinks |

**Accent reserved for:** active bottom-nav tab underline/icon tint, "Привязать Telegram" primary CTA button, and "Полная версия" / "Мобильная версия" link text. Not for decorative borders, disabled states, or informational badges.

**PWA branding (locked by D-08):**
- `theme_color`: `#1e1b4b` (indigo) — matches secondary nav bar color, applied in manifest.json and `<meta name="theme-color">`. Source: CONTEXT.md D-08.
- `background_color`: `#ffffff` — PWA splash screen background. Source: CONTEXT.md D-08.

**Offline stub page:** White background (`#ffffff`), body text in `#6b7280` (gray-500), centered layout. No accent color on the offline page — neutral signal only.

**Version-switch strip:** Very light strip at top of mobile page. Background `#f3f4f6` (gray-100), text `#6b7280` (gray-500), link color `#4f46e5` (accent). Source: RESEARCH.md Pattern 1.

**Unlinked Telegram screen:** White card on `#f5f5f5` background. Instruction text in `#374151` (gray-700). The "go to desktop profile" link uses `#4f46e5` accent. No destructive color used — this is an informational, not error, state.

## Component Inventory

These are the visual components that must be implemented in Phase 26. No third-party component library — all are Tailwind CSS utility constructions.

### 1. `base_mobile.html` — Mobile Base Template

**Structure:**
```
<html>
  <head> ... viewport-fit=cover, theme-color, manifest link, SW registration ... </head>
  <body style="padding-bottom: 64px">
    [version-switch strip]
    <main class="p-4"> {% block content %} </main>
    [bottom-nav bar]
    [toast container]
    [SW registration script]
  </body>
</html>
```

**Version-switch strip:** full-width, bg `#f3f4f6`, text-xs (12px), text-center, py-1 (4px top/bottom). Contains one `<a>` link "Полная версия" in accent color.

**Bottom nav bar:** `position: fixed; bottom: 0; left: 0; right: 0; height: 64px`. Background `#1e1b4b`. `padding-bottom: env(safe-area-inset-bottom)`. `display: flex; justify-content: space-around; align-items: center`. Contains 4 flex children (tabs).

**Each bottom-nav tab:** `display: flex; flex-direction: column; align-items: center; gap: 4px`. Icon: 24x24 inline SVG, white fill/stroke. Label: text-xs (12px), white, weight 400. Active tab: icon and label use accent `#4f46e5`, weight 600 on label. Min tap target: 44px height achieved by making each tab `min-h-[44px] justify-center`.

**Toast container:** `position: fixed; bottom: 80px; right: 16px; z-index: 999; display: flex; flex-direction: column; gap: 8px`. (Offset from bottom to clear nav bar.) Reuses `.toast`, `.toast-success`, `.toast-error`, `.toast-info` from base.html pattern.

### 2. `/m/` Index Page (`mobile/index.html`)

Extends `base_mobile.html`. Single `{% block content %}` with:
- Page heading: "SEO Platform" at heading size (20px, weight 600).
- Site selector: Full-width `<select>` styled consistently with desktop `.site-selector select` pattern, background `#eef2ff`, border `#c7d2fe`.
- Quick-link grid: placeholder cards pointing to future phases (disabled state for Phases 27+, active only on foundation routes).
- Empty state: shown when no sites exist. Heading + body copy from copywriting contract above.

### 3. Telegram Link Required Screen (`mobile/tg_link_required.html`)

Extends `base_mobile.html`. Centered card layout:
- White card, `border-radius: 8px`, `padding: 24px`, `box-shadow: 0 1px 3px rgba(0,0,0,.1)` (matches `.card` from base.html).
- Telegram icon (inline SVG, 48x48, `#4f46e5` fill).
- Heading: "Привяжите Telegram-аккаунт" (20px, weight 600).
- Body paragraph: instruction copy (16px, weight 400, line-height 1.5).
- CTA link-button: "Открыть полную версию" — styled as `.btn .btn-primary` equivalent using Tailwind: `bg-[#4f46e5] text-white rounded px-4 py-2 text-sm font-semibold`.

### 4. Offline Stub Page (`mobile/offline.html`)

Static page cached by service worker. Extends `base_mobile.html` or is a standalone minimal HTML (preferred: standalone to avoid dependency on template engine for offline cache).
- White background, centered content, icon (SVG wifi-off), heading + body copy from contract. No interactive elements.

### 5. Profile: "Привязать Telegram" Section

Added to existing `app/templates/profile.html` (desktop). Not a new template — a new `<section>` within the profile page:
- Section heading: "Telegram" (existing profile section style).
- If `telegram_id` already linked: show linked status badge (green, "Привязан: @username") + "Отвязать" link (destructive — not in Phase 26 scope; deferred).
- If not linked: embed Telegram Login Widget `<script>` tag + instructional text.

## Interaction Contracts

### Bottom Navigation Active State

- The currently active tab receives: icon in `#4f46e5`, label weight 600.
- Active tab is determined server-side: the route handler passes `active_tab: str` to the template context. Values: `"digest"`, `"sites"`, `"positions"`, `"more"`.
- No JavaScript tab-switching — full page navigation via `<a href>` links.

### HTMX Usage in Phase 26

Phase 26 is a foundation phase with minimal dynamic content. HTMX usage is limited to:
- Toast notifications: `hx-on::after-request` or server-sent `HX-Trigger` response headers trigger `showToast()`.
- No `hx-boost` on bottom nav links (full page load is acceptable for the shell; booting HTMX history on nav is deferred to later mobile phases).

### Telegram WebApp Auth Flow (UI)

1. User opens Mini App in Telegram → `base_mobile.html` loads.
2. JavaScript reads `window.Telegram.WebApp.initData` synchronously on page load.
3. If `initData` is non-empty, immediately POST to `/auth/telegram-webapp` (fetch API, before rendering content).
4. On success (200 + JWT cookie): reload `window.location.href = "/m/"` — now authenticated.
5. On failure (401/422): show inline error message (not a toast — a permanent banner) with copy from the error copywriting contract. The banner is `bg-[#fee2e2] text-[#991b1b] p-3 rounded text-sm`.
6. If `initData` is empty (not running in Telegram): render normal page (standard JWT cookie auth via `get_current_user`).

**No loading spinner required for Phase 26** — the auth POST is fast (stdlib HMAC, no external calls). If a spinner is added in future, use a full-screen white overlay with centered `animate-spin` SVG at 32x32px, `#4f46e5` color.

### PWA Install Prompt

No custom install UI in Phase 26. Browser's native install banner is triggered by the manifest + service worker presence. No interception of `beforeinstallprompt` event — deferred to Phase 27+ if needed.

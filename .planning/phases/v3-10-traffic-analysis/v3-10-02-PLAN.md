# Plan v3-10-02: Router and dashboard UI

<objective>
Create traffic analysis router and dashboard UI with Chart.js timeline of visits by type, anomaly alerts, bot/injection pattern tables, log upload, Metrika analysis trigger, and period comparison.
</objective>

<tasks>

<task id="01">
<title>Create traffic_analysis router</title>
<action>
Create `app/routers/traffic_analysis.py`:

Router: `prefix="/traffic-analysis"`, tags `["traffic-analysis"]`

1. `GET /{site_id}` — Dashboard page (HTML)
2. `POST /{site_id}/analyze-metrika` — Trigger Metrika analysis. Body: {date_from, date_to}.
3. `POST /{site_id}/upload-log` — Upload access log file.
4. `GET /{site_id}/sessions` — List analysis sessions.
5. `GET /sessions/{session_id}` — Session detail with analysis results.
6. `GET /sessions/{session_id}/visits` — Visit data (JSON, for Chart.js).
7. `GET /sessions/{session_id}/anomalies` — Anomaly detection results.
8. `GET /sessions/{session_id}/bots` — Bot-flagged visits.
9. `GET /sessions/{session_id}/injections` — Injection patterns detected.

All `require_admin`.
</action>
<acceptance_criteria>
- [ ] `python -c "from app.routers.traffic_analysis import router"` succeeds
- [ ] `grep -c "@router" app/routers/traffic_analysis.py` returns at least 8
</acceptance_criteria>
</task>

<task id="02">
<title>Create dashboard template</title>
<action>
Create `app/templates/traffic_analysis/index.html`:

Extends `base.html`.

**1. Header:** "Анализ трафика: {site.name}"

**2. Analysis source** (card):
- "Анализ Метрики" — date range pickers + button
- "Загрузить access.log" — file upload
- Session history list

**3. Traffic timeline** (Chart.js):
- Line chart with daily visits colored by source type
- Anomaly days marked with red vertical annotations
- Legend: organic/direct/referral/bot/injection

**4. Anomaly alerts** (card):
- If anomalies detected: red alert with dates and deviation
- "Нет аномалий" if clean

**5. Traffic sources** (card):
- Pie chart or bar chart: organic vs direct vs referral vs bot vs injection
- Top referers table
- Top landing pages table

**6. Bot detection** (card):
- Table of bot-flagged visits: timestamp, IP, UA, referer, reason
- Filter by bot_reason

**7. Injection patterns** (card):
- Detected patterns with evidence and confidence
- Comparison: "до подлива" vs "во время" vs "после" — 3-column summary if anomaly dates known

**8. Period comparison:**
- Select two periods
- Side-by-side: visits, bounce, duration, source distribution

All copy in Russian. Register in main.py. Add "Анализ трафика" button to site detail.
</action>

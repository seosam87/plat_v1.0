<task id="02">
<title>Create traffic_analysis_service.py</title>
<read_first>
- app/services/metrika_service.py
- app/models/traffic_analysis.py
</read_first>
<action>
Create `app/services/traffic_analysis_service.py`:

### Bot detection (pure)

**`def classify_visit(ua: str, ip: str, referer: str, bot_patterns: list[dict]) -> dict`**
Check against bot patterns. Return `{is_bot: bool, source: str, bot_reason: str|None}`.

**`def detect_anomalies(visits_by_day: list[dict]) -> dict`**
Analyze daily visit counts for anomalies:
- Calculate mean + std_dev
- Day with visits > mean + 2*std_dev → anomaly
- Return `{anomaly_detected: bool, anomaly_days: [{date, visits, expected}], baseline_avg: float}`

**`def analyze_traffic_sources(visits: list[dict]) -> dict`**
Group visits by source. Return `{organic: N, direct: N, referral: N, bot: N, injection: N, by_referer: {ref: count}, by_landing: {url: count}}`.

**`def detect_injection_patterns(visits: list[dict]) -> list[dict]`**
Detect traffic injection patterns:
- Same referer with many different IPs → suspected injection
- Burst of visits from unusual geo → suspected
- Very short session duration + high bounce → bot-like
Return list of `{pattern: str, evidence: str, confidence: float}`.

### Metrika integration (async)

**`async def analyze_from_metrika(db, site_id, date_from, date_to) -> dict`**
Load MetrikaTrafficDaily + MetrikaTrafficPage for period.
Run anomaly detection on daily data.
Return analysis summary.

### Access log parser

Create `app/parsers/access_log_parser.py`:

**`def parse_access_log(content: str, limit: int = 50000) -> list[dict]`**
Parse Apache/Nginx combined log format. Extract: ip, timestamp, method, url, status, ua, referer.
Return list of dicts.

### Session management (async)

- `create_analysis_session(db, site_id, name, period_start, period_end, source_type)`
- `save_visits(db, session_id, visits: list[dict])`
- `get_session_summary(db, session_id) -> dict`
- `list_sessions(db, site_id) -> list`
</action>

- `app/services/agent_service.py` ‚Äî pure service layer:
  - `parse_template_variables(template)` ‚Äî extracts unique `{{var}}` tokens in order
  - `render_template(template, inputs)` ‚Äî substitutes placeholders
  - `list_categories`, `list_agents` (with category/search/favourite filters), `get_agent`, `create_agent`, `update_agent`, `delete_agent`
  - `fork_agent` ‚Äî copies all fields, appends " (–∫–æ–ø–∏—è)" to name, sets fork_of_id
  - `toggle_favourite` / `is_favourited` ‚Äî returns bool of new state
  - `create_job`, `get_job`, `increment_usage_count`

- `app/services/llm/llm_service.py` ‚Äî added `call_agent()`:
  - Accepts api_key, system_prompt, user_message, model, temperature, max_tokens
  - Returns (output_text, input_tokens, output_tokens) tuple
  - Appended at end of file without touching existing functions

- `app/tasks/agent_tasks.py` ‚Äî `run_agent_job` Celery task:
  - `name="agents.run_agent_job"`, `max_retries=3`, `queue="default"`
  - Sync wrapper with `asyncio.new_event_loop()` matching llm_tasks.py pattern
  - `_TransientAgentError` sentinel for retry signaling
  - Circuit breaker check via Redis (per-user)
  - Transient errors (429/529/5xx/connection/timeout) ‚Üí exponential backoff retry
  - Permanent errors (401/403/400) ‚Üí immediate error, no retry
  - Calls `increment_usage_count` on success

- `app/celery_app.py` ‚Äî added `"app.tasks.agent_tasks"` to include list

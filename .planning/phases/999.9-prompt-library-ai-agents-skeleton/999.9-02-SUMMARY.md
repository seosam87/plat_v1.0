## What Was Built

### Task 1: Router + Integration

- `app/routers/agents.py` — `APIRouter(prefix="/ui/agents")` with 12 routes:
  - `GET /` — catalogue with HTMX partial support (HX-Request → grid partial)
  - `GET /parse-vars` — inline variable badge HTML for live preview
  - `GET /new`, `POST /new` — create agent form
  - `GET /jobs/{job_id}/status` — HTMX polling endpoint (registered before `/{agent_id}/*`)
  - `GET /{agent_id}/edit`, `POST /{agent_id}/edit` — edit form
  - `POST /{agent_id}/delete` — delete with redirect
  - `GET /{agent_id}/run`, `POST /{agent_id}/run` — run page + Celery dispatch
  - `POST /{agent_id}/favourite` — toggle star, returns updated button HTML
  - `POST /{agent_id}/fork` — fork → redirect to edit

- `app/main.py` — agents_router registered after playbooks_router

- `app/navigation.py` — "AI-агенты" section (cpu-chip icon) with Каталог + Избранное children inserted before Settings

### Task 2: Templates

- `index.html` — extends base.html, category select (hx-trigger=change), search input (delay:300ms), #agents-grid container with include
- `_agent_card_grid.html` — 3-col responsive grid, empty state for both normal and favourite mode
- `_agent_card.html` — category badge (content→emerald, audit→rose, analytics→sky, technical-seo→teal, misc→slate), star toggle via hx-post, agent icon/name/desc/model, Запустить/Редактировать/Форк actions
- `form.html` — create/edit form: font-mono textareas for prompts, hx-get=/ui/agents/parse-vars live preview, #variables-preview div, LLM settings grid, delete button with hx-confirm
- `run.html` — agent header with category badge, system prompt collapsible details, variable input form (textarea for text/content/description vars), hx-post dispatch to #agent-run-result
- `_agent_job_status.html` — 3 states: pending/running (hx-trigger="every 2s"), done (no hx-trigger, markdown filter for output), error (no hx-trigger, retry link)

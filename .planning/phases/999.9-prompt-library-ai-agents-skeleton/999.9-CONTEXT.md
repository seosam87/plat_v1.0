# Phase 999.9: Prompt Library — каркас AI-агентов — Context

**Gathered:** 2026-04-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Каталог AI-агентов (промптов с переменными), сгруппированных по категориям. Пользователь создаёт агента (system prompt + шаблон запроса с `{{переменными}}`), запускает через форму, получает результат через Celery-задачу. Библиотека с поиском и избранным.

</domain>

<decisions>
## Implementation Decisions

### D-01: Структура промптов
- Свободный текст с `{{переменными}}` — платформа парсит переменные из шаблона и автоматически строит форму ввода
- Без автоматической истории версий — пользователь вручную создаёт копию (форк) удачной версии промпта
- Форк хранит ссылку на оригинал

### D-02: AI-агент — отдельная сущность
- Новая модель, НЕ связанная с ExpertSource из плейбуков
- Свой каталог, своя навигация
- Максимально расширенная структура полей:

**Основные:** название, описание, иконка/аватарка, system prompt, шаблон запроса (user message с переменными), категория

**LLM-настройки:** модель (claude-sonnet / claude-haiku / etc), температура, max_tokens, output format (текст / JSON / markdown)

**Мета:** автор (кто создал), дата создания, теги/метки, рейтинг/количество использований (статистика), публичный/приватный (видимость для команды), источник-форк (ссылка на оригинал если скопирован)

### D-03: Запуск через Celery
- Пользователь заполняет переменные в форме → задача отправляется в Celery
- HTMX-поллинг статуса, результат появляется когда готово
- Консистентно с существующим паттерном LLM briefs (`app/routers/llm_briefs.py`)
- Retry=3 для API-вызовов (стандарт проекта)

### D-04: Каталог и навигация
- Группировка по категориям (Контент / Аудит / Аналитика / Техническое SEO и т.д.)
- Текстовый поиск по названию и описанию
- Избранное (звёздочка) для быстрого доступа к часто используемым агентам

### Claude's Discretion
- Конкретные имена таблиц и структура миграций
- Детали HTMX-поллинга и UI-компонентов
- Механизм парсинга `{{переменных}}` из шаблона

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **LLM service** (`app/services/llm/`): готовая обёртка Anthropic SDK с circuit breaker и логированием — переиспользовать для вызовов агентов
- **LLM briefs паттерн**: Celery task → Redis polling → HTMX partial — тот же flow для запуска агентов
- **PlaybookBlock.body + `| markdown` фильтр**: паттерн рендеринга markdown-контента в шаблонах
- **BlockCategory enum**: паттерн категоризации через модель с seed-данными

### Established Patterns
- Celery задачи с `retry=3` и `task_acks_late=True` для внешних API
- HTMX-поллинг через `hx-trigger="every 2s"` для long-running операций
- UUID первичные ключи во всех моделях
- SQLAlchemy 2.0 async + AsyncSession

### Integration Points
- `app/main.py` — регистрация нового роутера
- `app/navigation.py` — новая секция в сайдбаре
- `app/celery_app.py` — регистрация новых задач
- Alembic миграция для новых таблиц

</code_context>

<specifics>
## Specific Ideas

- Пользователь хочет понимать AI-агентов как "карточки" с ролью и инструкциями, которые можно запускать через форму
- Примеры агентов для SEO: копирайтер, мета-тег генератор, аудитор контента, генератор ТЗ
- Форк = осознанное действие пользователя ("эта версия хорошая, сохраню копию"), не автоматическое

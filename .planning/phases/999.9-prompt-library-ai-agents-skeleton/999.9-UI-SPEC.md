## Component Inventory

These are the UI surfaces required by this phase. All implemented as Jinja2 templates extending `base.html`.

### 1. Agent Catalogue Page (`/ui/agents/`)
- Page heading + "+ Создать агента" button (indigo-600, admin only)
- Category filter dropdown (HTMX `hx-get`, `hx-trigger="change"`) — same pattern as playbooks/index.html
- Text search input (HTMX `hx-get`, `hx-trigger="input changed delay:300ms"`) — searches name + description
- Agent card grid: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`
- Empty state (see Copywriting Contract)

### 2. Agent Card (`_agent_card.html` partial)
- Category badge (colour per table above)
- Agent icon/avatar: 32px circle with emoji or initials fallback, `bg-indigo-100 text-indigo-700`
- Agent name: `text-base font-semibold text-gray-800`
- Description: `text-sm text-gray-500` truncated to 120 chars
- Model badge: `text-xs bg-gray-100 text-gray-600` (e.g. "claude-haiku")
- Usage count: `text-xs text-gray-400` (e.g. "12 запусков")
- Star/favourite toggle: inline SVG star icon, filled `text-amber-400` when favourited, outline `text-gray-300` otherwise — HTMX POST, hx-swap="outerHTML" on the icon wrapper
- "Запустить" link → agent run page
- "Редактировать" link (admin only) → agent edit page
- "Форк" button → creates copy, redirects to edit page of fork

### 3. Agent Create/Edit Form (`/ui/agents/new`, `/ui/agents/{id}/edit`)
- Single-column form inside `.card` wrapper
- Field groups separated by 16px gap
- **Основные поля:** название (text), описание (textarea rows=2), иконка (text, emoji or URL), категория (select), публичный (checkbox)
- **Промпт-поля:** system_prompt (textarea rows=6, font-mono), user_template (textarea rows=6, font-mono) with inline hint: "Используйте `{{переменная}}` для полей ввода"
- **LLM-настройки:** модель (select: claude-haiku / claude-sonnet / claude-opus), температура (number input 0.0–1.0, step=0.1), max_tokens (number input), output_format (select: text / markdown / json)
- **Мета:** теги (text, comma-separated), публичный (checkbox)
- Parsed variables preview: `<div id="variables-preview">` updated live via HTMX as user types in user_template (`hx-trigger="keyup changed delay:500ms"`, `hx-get="/ui/agents/parse-vars"`) — shows detected `{{var}}` as slate badges
- Fork origin notice: if agent is a fork, show indigo-bordered callout: "Форк от: [original name]" with link
- Save button: `btn btn-primary` (indigo-600)
- Cancel button: `border border-gray-300 text-gray-700 hover:bg-gray-50`

### 4. Agent Run Page (`/ui/agents/{id}/run`)
- Agent header: icon + name + category badge + description
- System prompt preview: `<details>` collapsed by default, `bg-gray-50 rounded p-3 font-mono text-xs text-gray-600`
- Variable input form: one `<input>` or `<textarea>` per detected `{{variable}}`, labelled with the variable name, styled `text-sm border border-gray-300 rounded focus:ring-indigo-500`
- "Запустить агента" button: full-width on mobile, normal on desktop, indigo-600
- HTMX polling result container `<div id="agent-run-result">` — `hx-get="/ui/agents/jobs/{job_id}/status"`, `hx-trigger="every 2s"`, `hx-swap="outerHTML"` — stops polling when job status is `done` or `error`
- Pending state: spinner + "Агент работает…" (`text-sm text-gray-500`)
- Result state: rendered output (markdown via `| markdown` filter if output_format=markdown, else `<pre class="whitespace-pre-wrap text-sm">`)
- "Форкнуть агента" secondary button appears after successful run

### 5. Sidebar Navigation Entry
- New section "AI-агенты" in `app/navigation.py`
- Icon: inline SVG (cpu/brain icon, 20px)
- Children: "Каталог" → `/ui/agents/`, "Избранное" → `/ui/agents/?favourite=1`
- Active state: `background: #3730a3` for section header, `background: #4338ca` for child (matches existing sidebar CSS)

### 6. HTMX Polling Status Partial (`_agent_job_status.html`)
Three states, consistent with `_llm_job_preview.html` pattern:
- **pending:** spinner SVG + `text-sm text-gray-500` "Агент работает…" + `hx-trigger="every 2s"`
- **done:** result block, no polling trigger
- **error:** `bg-red-50 border border-red-200 rounded p-4` + error message + retry button

## Interaction Contract

| Interaction | Mechanism | Target | Notes |
|-------------|-----------|--------|-------|
| Category filter | HTMX `hx-get` `hx-trigger="change"` | `#agents-grid` | Push URL, same as playbooks/index.html |
| Text search | HTMX `hx-get` `hx-trigger="input changed delay:300ms"` | `#agents-grid` | Debounced to avoid server thrash |
| Variable preview | HTMX `hx-get` `hx-trigger="keyup changed delay:500ms"` | `#variables-preview` | GET `/ui/agents/parse-vars?template=...` |
| Star toggle | HTMX `hx-post` `hx-swap="outerHTML"` | star icon wrapper | Instant optimistic-style swap |
| Run agent | Form POST → Celery task | Redirects to run page with job_id | Consistent with LLM briefs pattern |
| Job polling | HTMX `hx-trigger="every 2s"` | `#agent-run-result` | Remove trigger attribute when done/error |
| Fork | HTMX `hx-post` | Redirect to edit page of new fork | Server-side redirect after creation |
| Delete agent | Form POST with `hx-confirm` | Page reload | Confirmation dialog: see Copywriting |

## Accessibility Notes

- All form fields have `<label for="...">` associations
- Star toggle includes `aria-label="Добавить в избранное"` / `"Убрать из избранного"` toggled via HTMX swap
- Delete buttons include `aria-label="Удалить агента {name}"`
- Textarea fields for prompts have `aria-describedby` pointing to the `{{переменная}}` hint paragraph
- Polling container includes `aria-live="polite"` for screen reader announcements of job completion

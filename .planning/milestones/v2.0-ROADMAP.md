## Key Decisions

| Decision | Rationale |
|----------|-----------|
| `normalize_url()` + `keyword_latest_positions` in Phase 12 | All analytical JOINs fail silently without URL normalization; DISTINCT ON on partitioned table causes 8–15s scans at 100K keywords |
| WeasyPrint subprocess isolation in Phase 14 | WeasyPrint memory leak (GitHub #2130, #1977) cannot be deferred |
| Keyword Suggest in isolated Phase 15 | IP ban risk from autocomplete scraping is entirely separate from analytical features |
| LLM Briefs + GEO Readiness co-located in Phase 16 | Both depend on existing crawl + content audit data; anthropic SDK introduced once |
| In-app Notifications last (Phase 17) | Touches global sidebar chrome; cleanest to add after all feature surfaces are finalized |
| HTMX polling (30s) over SSE for notifications | Under 20 users; polling is simpler and sufficient |
| Template brief always generated first; LLM is enhancement only | Prevents LLM API failures from blocking brief delivery |
| Yandex Suggest primary, Google secondary (opt-in), Wordstat opt-in | User note: focus on Yandex, no complex Google services |
| Static beat_schedule replaced by explicit RedBeatSchedulerEntry registration | RedBeat scheduler ignores static `celery_app.conf.beat_schedule` — entries must be persisted via `.save()` in `beat_init` hook (discovered during Phase 17 UAT) |
| dismiss endpoint returns 200 empty body (not 204) | HTMX ignores swap entirely on HTTP 204 regardless of `hx-swap` value (discovered during Phase 17 UAT) |

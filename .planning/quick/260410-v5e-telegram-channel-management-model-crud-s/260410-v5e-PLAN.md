5. **app/services/channel_service.py** — Create service with these functions:
   - `TELEGRAM_API_BASE = "https://api.telegram.org/bot{token}"` — base URL template
   - `async def _tg_request(method: str, payload: dict) -> dict` — helper: builds URL from `settings.TELEGRAM_BOT_TOKEN`, uses `httpx.AsyncClient(timeout=15)`, POSTs to `{base}/{method}`, raises on non-ok response, returns result JSON. Log via loguru.
   - `async def list_posts(db: AsyncSession, status: str | None = None, sort: str = "created_at_desc", page: int = 1, per_page: int = 50) -> tuple[list[TelegramChannelPost], int]` — query with optional status filter, sort by created_at desc or scheduled_at, return (items, total_count). Use `select()` + `func.count()`.
   - `async def get_post(db: AsyncSession, post_id: int) -> TelegramChannelPost | None` — simple get by ID.
   - `async def create_post(db: AsyncSession, title: str, content: str, user_id: uuid.UUID) -> TelegramChannelPost` — create with status=draft.
   - `async def update_post(db: AsyncSession, post_id: int, title: str, content: str) -> TelegramChannelPost` — update draft/scheduled post. Raise ValueError if published (cannot edit content directly, use edit_published).
   - `async def delete_post(db: AsyncSession, post_id: int) -> None` — if published and has telegram_message_id, call `deleteMessage` via Bot API first (ignore errors — message may already be deleted), then delete from DB.
   - `async def publish_post(db: AsyncSession, post_id: int) -> TelegramChannelPost` — call `_tg_request("sendMessage", {"chat_id": settings.TELEGRAM_CHANNEL_ID, "text": post.content, "parse_mode": "HTML"})`, store `message_id` from response, set status=published, published_at=utcnow.
   - `async def edit_published(db: AsyncSession, post_id: int, content: str) -> TelegramChannelPost` — call `_tg_request("editMessageText", {"chat_id": settings.TELEGRAM_CHANNEL_ID, "message_id": post.telegram_message_id, "text": content, "parse_mode": "HTML"})`, update content in DB.
   - `async def toggle_pin(db: AsyncSession, post_id: int) -> TelegramChannelPost` — if pinned: call `unpinChatMessage`, set pinned=False. If not pinned: call `pinChatMessage` with `{"chat_id": ..., "message_id": ...}`, set pinned=True.
   - `async def schedule_post(db: AsyncSession, post_id: int, scheduled_at: datetime) -> TelegramChannelPost` — set status=scheduled, scheduled_at=value.
   - All DB-mutating functions: `await db.commit()`, `await db.refresh(post)`.

   - All endpoints require `current_user: User = Depends(get_current_user)` and `db: AsyncSession = Depends(get_db)`
   - Use `from app.template_engine import templates` for rendering
   - **GET /** — list page: parse query params `status` (optional filter), `sort` (default "created_at_desc"), `page` (default 1). Call `channel_service.list_posts()`. Render `channel/index.html` with `posts`, `total`, `current_status`, `current_sort`, `page`, `pages` (total // per_page + 1).
   - **GET /new** — render `channel/edit.html` with `post=None` (create mode).
   - **POST /** — create: parse form fields `title`, `content`. Call `channel_service.create_post()`. Redirect to `/ui/channel/{post.id}`.
   - **GET /{post_id}** — render `channel/edit.html` with the post (edit mode).
   - **POST /{post_id}** — update: parse form `title`, `content`. If post is published, call `channel_service.edit_published()`. Else call `channel_service.update_post()`. Redirect back to `/ui/channel/{post_id}`.

   - **GET /{post_id}/preview** — get post, render content as HTML for preview. Use `markdown` library if available, otherwise simple `content.replace('\n', '<br>')` for the UI preview. Return partial `_preview.html`.
   - Error handling: wrap Bot API calls in try/except, return 400 with error message on failure. Use `request.headers.get("HX-Request")` to detect HTMX and return appropriate response.

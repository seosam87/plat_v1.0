## POST 1: Вступительный пост (закреплённый)

**SEO Management Platform — единая панель управления SEO для агентств**

Управляете SEO для клиентов и тонете в вкладках? GSC в одном окне, Яндекс.Вебмастер в другом, таблички с ключами в третьем, WP-админки — на каждом сайте отдельно.

Мы это собрали в одном месте.

SEO Platform — self-hosted веб-приложение для команды, которая ведёт от 20 до 100 WordPress-сайтов. Всё в одном интерфейсе: позиции, краулинг, контент, задачи, отчёты, инструменты. Плюс мобильные мини-приложения прямо в Telegram.

**Что внутри:**
- Мониторинг позиций и трафика по всем сайтам
- Автоматический краулер на Playwright
- Контент-оптимизация: TOC, Schema.org, внутренняя перелинковка
- SEO-проекты с Kanban-досками
- CRM клиентов и генерация КП/отчётов в PDF
- 6 SEO-инструментов (коммерциализация, брифы, PAA, Wordstat...)
- Мобильные приложения в Telegram Mini Apps
- Уведомления в Telegram, email-отчёты

**Стек:** Python 3.12, FastAPI, PostgreSQL 16, Redis 7, Celery, Playwright, HTMX 2.0
**Развёртывание:** Docker Compose, один VPS — без облачных зависимостей

## POST 4: Какие задачи решает

#задачи

**10 проблем SEO-агентства, которые закрывает платформа**

**1. "У меня 50 сайтов, и я не знаю что происходит прямо сейчас"**
→ Дайджест показывает изменения позиций, ошибки, алерты и просроченные задачи по всем сайтам в одном экране. Открываешь утром — видишь, что горит.

**2. "Проверка позиций — это три сервиса, пять вкладок и полчаса работы"**
→ Одна кнопка «Запустить проверку». Celery проверяет все ключи сайта в фоне, прогресс обновляется в реальном времени. Результат — список с дельтами, сортировка по просадкам.

**3. "Клиент спрашивает отчёт, а я полдня собираю данные"**
→ Генератор документов: выбираешь шаблон, платформа подставляет данные (позиции, трафик, ошибки), жмёшь кнопку — PDF/DOCX готов. Отправка в Telegram или на email.

**4. "Копирайтер не понимает, что писать — ТЗ из головы"**
→ Копирайтерский бриф: платформа парсит TOP-10 по ключу, извлекает заголовки, тематические слова, структуру. Выгружаешь в XLSX — готовое ТЗ с данными из выдачи.

**5. "Сайт упал, а узнал через два дня от клиента"**
→ Мониторинг изменений: краулер детектит критические изменения (noindex, 404, редиректы), платформа сразу шлёт Telegram-алерт.

**6. "Задачи в голове, в чате, в заметках — нигде нет единого списка"**
→ Проекты + Kanban. Задачи создаются из любого экрана: увидел просадку позиций — тут же создал задачу. Увидел ошибку аудита — задача. Всё привязано к сайту.

**7. "WP-правки — это зайти в админку каждого сайта"**
→ WP Pipeline: редактируешь TOC/Schema/Meta в платформе → approve → push в WordPress через REST API. Один интерфейс для всех сайтов.

**8. "На встрече с клиентом нужна картинка 'что мы делаем'"**
→ CRM + шаблоны КП: карточка клиента, история, привязка сайтов. Генерация КП с подстановкой данных. PDF за минуту.

**9. "Нужно быстро глянуть позиции с телефона, а десктоп-версия не удобна"**
→ Мобильные мини-приложения в Telegram: дайджест, позиции, трафик, здоровье — со смартфона, одним тапом.

**10. "По каждому инструменту — отдельный сервис и подписка"**
→ 6 встроенных инструментов: Wordstat, PAA, брифы, коммерциализация, мета-парсер, релевантные URL. Всё внутри, без внешних подписок (кроме XMLProxy для Яндекс-выдачи).

## POST 5: Технический стек

#стек

**Под капотом: технологии SEO Platform**

```
Backend:
  Python 3.12 + FastAPI 0.115
  SQLAlchemy 2.0 (полностью async)
  PostgreSQL 16 (партиционирование, JSON-колонки)
  asyncpg (async драйвер)
  Alembic (миграции БД)

Очереди и кэш:
  Celery 5.4 + Redis 7
  redbeat (расписания в Redis, а не в файле)
  3 очереди: default, crawl, tools
  Flower (мониторинг задач)

Браузерная автоматизация:
  Playwright 1.47 (async API, Chromium)
  Стелс-контексты для парсинга SERP

Фронтенд:
  Jinja2 (серверный рендеринг)
  HTMX 2.0.3 (динамика без SPA)
  Tailwind CSS
  CodeMirror 6 (редактор шаблонов)
  D3.js (визуализация архитектуры)

Мобильный:
  PWA (manifest + service worker)
  Telegram WebApp SDK
  HMAC-SHA256 авторизация
  HTMX polling для real-time

Интеграции:
  XMLProxy.ru (Яндекс SERP)
  DataForSEO (позиции)
  Google Search Console (OAuth 2.0)
  Яндекс.Вебмастер
  Яндекс.Метрика (трафик, постраничная аналитика)
  WordPress REST API
  Yoast/RankMath API
  Telegram Bot API
  SMTP (отчёты)
  Anthropic Claude (opt-in, LLM-брифы)

Безопасность:
  JWT (24h exp, 3 роли)
  bcrypt (пароли)
  Fernet AES-128 (внешние credentials)
  slowapi (rate limiting)
  HTTPS + HSTS

Развёртывание:
  Docker Compose v2
  python:3.12-slim (основа)
  mcr.microsoft.com/playwright (краулер)
  Один VPS — никаких облачных зависимостей
```

**Почему именно такой стек:**
- FastAPI + HTMX вместо React: нет билд-пайплайна, нет npm, серверный рендеринг — быстрее в разработке, проще в поддержке
- PostgreSQL + партиционирование: 18 миллионов строк позиций в год без деградации запросов
- Celery + Redis: проверка позиций для 100 сайтов параллельно, каждый сайт — отдельная задача с ретраями
- Playwright вместо Selenium: async, быстрее, стелс-режим для SERP-парсинга
- Self-hosted: данные клиентов на вашем сервере, не в чужом облаке

## POST 6: Мини-приложения — обзор

#miniapps

**Telegram Mini Apps: SEO с телефона**

Платформа работает не только в браузере. Четыре мини-приложения доступны прямо в Telegram — открываются мгновенно, авторизация автоматическая.

**Как это работает:**
1. Пользователь нажимает кнопку в Telegram-боте
2. Открывается Mini App (внутри Telegram, не браузер)
3. Авторизация автоматическая: Telegram передаёт initData → сервер проверяет подпись HMAC-SHA256 → выдаёт JWT
4. Пользователь сразу видит данные — без логина и пароля

**4 приложения уже работают:**

📊 **Дайджест** (`/m/digest`)
Утренняя сводка по всем сайтам — одна страница, четыре блока

🏥 **Здоровье сайта** (`/m/health/{id}`)
6 метрик здоровья любого сайта + действия

📈 **Позиции** (`/m/positions`)
Проверка позиций и анализ просадок

📉 **Трафик** (`/m/traffic`)
Сравнение трафика из Яндекс.Метрики за два периода

**Общие возможности всех мини-приложений:**
- Создание задач в один тап из любого экрана
- Touch-friendly интерфейс (44px минимум для тап-зон)
- Toast-уведомления о действиях
- Работает как PWA (можно добавить на главный экран)

## POST 7: Мини-приложение — Дайджест

#miniapps #дайджест

**📊 Дайджест — вся SEO-картина за 30 секунд**

Открыл с утра — понял, что горит. Без логина в три сервиса.

**Четыре блока:**

**1. Изменения позиций**
ТОП-5 резких изменений за неделю. Ключевое слово, сайт, дельта позиции (зелёная/красная). Если "Двери межкомнатные" упало с 3 на 15 — ты это увидишь первым.

**2. Ошибки краулера**
Страницы с HTTP 404 или noindex. Если клиентский сайт отдаёт 404 на главной — это в первом же блоке.

**3. Алерты мониторинга**
Критические изменения, пойманные автоматическим мониторингом: noindex появился, редирект поменялся, robots.txt обновился.

**4. Просроченные задачи**
Задачи с прошедшим дедлайном. Количество дней просрочки. Красным горят те, что больше недели.

**Как пользоваться:**
- Открыть Telegram → нажать кнопку бота → Mini App открывается
- Пролистать 4 блока (30 секунд)
- Тапнуть на проблему → переход на полную версию для действий
- Можно добавить `/m/` как PWA на главный экран → открывается без Telegram

**Когда полезен:**
- Утренний чекап: что произошло за ночь
- Быстрая проверка: всё ли в порядке по сайтам
- Приоритизация дня: с чего начать работу

## POST 8: Мини-приложение — Здоровье сайта

#miniapps #здоровье

**🏥 Здоровье сайта — 6 KPI на одном экране**

Тапнул на сайт — увидел, живой ли он.

**6 метрик с цветовой индикацией:**

🟢🟡🔴 **Доступность**
Статус последнего краулинга. Зелёный = done, красный = failed, серый = нет данных.

🟢🟡🔴 **Ошибки краулера**
Количество 404/noindex в последнем обходе. 0 = зелёный, 1-5 = жёлтый, >5 = красный.

🟢🟡🔴 **Свежесть краула**
Когда последний раз краулер обходил сайт. Меньше 7 дней = зелёный, больше = жёлтый.

🟢🟡🔴 **Резкие изменения позиций**
Ключевые слова с дельтой > 10 за неделю. 0 = зелёный, есть = красный.

🟢🟡🔴 **Просроченные задачи**
Задачи с прошедшим дедлайном. 0 = зелёный, 1-3 = жёлтый, >3 = красный.

🟢⚪ **Метрика**
Подключена ли Яндекс.Метрика. Зелёный = есть токен, серый = не настроена.

**Два действия:**

**"Запустить краулинг"** — кнопка отправляет задачу в Celery, краулер начинает обход. Toast "Краулинг запущен". Результат — в полной версии.

**"Создать задачу"** — форма появляется прямо на странице (HTMX). Заголовок, приоритет, сохранить. Toast "Задача создана".

**Когда полезен:**
- Быстрый скрининг: сайт клиента — живой? краулер работает? позиции не упали?
- На встрече с клиентом: открыл телефон, показал статус
- Проактивность: увидел жёлтый/красный — действуешь до того, как клиент заметил

## POST 9: Мини-приложение — Позиции

#miniapps #позиции

**📈 Позиции — проверка и анализ просадок с телефона**

Запустил проверку, подождал 2 минуты, увидел результат. Просевшие ключи → задача в один тап.

**Интерфейс:**

**Селектор сайта** — выпадающий список всех сайтов. Выбрал → список обновляется.

**Фильтр периода** — "Все время", "7 дней", "30 дней". Тап → мгновенная фильтрация.

**Табы** — "Все" (полный список) и "Просевшие" (только delta < 0). Переключение без перезагрузки.

**Кнопка "Запустить проверку":**
1. Тап → Celery-задача стартует
2. Toast "Проверка запущена"
3. Прогресс-бар: "Проверено 15 из 47 ключей" (обновляется каждые 3 секунды)
4. Завершение → "Проверка завершена" + кнопка "Показать результаты"
5. Тап "Показать результаты" → список обновляется

**Карточка ключевого слова:**
- Фраза ("двери межкомнатные москва")
- Текущая позиция (5)
- Дельта (+3 зелёным / -7 красным / NEW серым)
- Поисковая система (Яндекс/Google)
- Дата проверки

**Создание задачи из просадки:**
На табе "Просевшие" у каждого ключа — кнопка "Создать задачу". Тап → форма с предзаполненным заголовком (фраза ключа) → сохранить → задача в проекте.

**Как настроить:**
1. Добавить сайт в платформу (полная версия → Сайты → Добавить)
2. Импортировать ключевые слова (CSV/XLSX)
3. Настроить прокси XMLProxy (для Яндекса) или DataForSEO
4. Открыть `/m/positions` → выбрать сайт → запустить проверку

**Технические детали:**
- Данные из `keyword_latest_positions` (кэш-таблица, не партиционированная — быстрые запросы)
- Celery task `check_positions` с `update_state(PROGRESS)` на каждом ключе
- Task ID хранится в Redis (TTL 600 сек)
- HTMX polling каждые 3 секунды до завершения
- Сортировка по abs(delta) DESC — самые резкие изменения наверху

## POST 10: Мини-приложение — Трафик

#miniapps #трафик

**📉 Трафик — сравнение периодов из Яндекс.Метрики**

Какие страницы просели? Где рост? Два периода, одна страница, все ответы.

**Интерфейс:**

**Селектор сайта** — выпадающий список (показываются только сайты с подключённой Метрикой).

**Пресеты периодов** (pill-кнопки):
- "Эта неделя vs прошлая"
- "Этот месяц vs прошлый"
- "30 дней vs 30 дней" (по умолчанию)

Нет кастомного date-picker — только пресеты. Быстро и без путаницы с датами.

**Сводка (верхняя карточка):**
```
Период A: 12 450 визитов
    ↓
Период B: 10 230 визитов
Дельта: -17.8% (красным)
```

**Список страниц:**
TOP-50 страниц, сортировка по просадке (самые большие потери наверху):
```
/catalog/dveri-mezhk...  -340 визитов (красный)
/blog/kak-vybrat-dv...   -120 визитов (красный)
/contacts/                +45 визитов (зелёный)
```

**Создание задачи:**
Тап на просевшую страницу → форма "Создать задачу" с предзаполненным URL → сохранить → задача привязана к сайту.

**Если Метрика не подключена:**
Жёлтый баннер: "Метрика не подключена. Настройте в полной версии платформы."

**Как настроить:**
1. В полной версии → Сайт → Настройки → Яндекс.Метрика
2. Ввести `metrika_token` (OAuth токен) и `metrika_counter_id` (номер счётчика)
3. Открыть `/m/traffic` → выбрать сайт → данные загрузятся

**Технические детали:**
- Данные: сначала из кэша (`PageTrafficSnapshot`), при промахе — API-запрос к Метрике
- Сервис: `get_traffic_comparison()` вычисляет дельту по страницам
- Обработка ошибок: API timeout → toast ошибки, пустой стейт
- Ограничение: TOP-50 страниц по дельте

## POST 11: Настройка и развёртывание

#настройка

**Как развернуть SEO Platform**

**Требования:**
- VPS: 4 GB RAM, 2 vCPU (минимум), 50 GB SSD
- Docker + Docker Compose v2
- Домен с SSL (Let's Encrypt / Cloudflare)

**Быстрый старт:**
```bash
git clone <repo>
cp .env.example .env
# Заполнить .env: DB, Redis, JWT secret, Telegram bot token
docker compose up -d
docker compose exec app alembic upgrade head
```

**Сервисы в Docker Compose:**
- `app` — FastAPI приложение
- `db` — PostgreSQL 16
- `redis` — Redis 7
- `celery-worker` — Celery воркер (3 очереди)
- `celery-beat` — Планировщик задач (redbeat)
- `crawler` — Playwright воркер (отдельный образ)
- `flower` — Мониторинг Celery (порт 5555)

**Ключевые переменные .env:**
```
DATABASE_URL=postgresql+asyncpg://...
REDIS_URL=redis://redis:6379/0
JWT_SECRET_KEY=<random 32 chars>
TELEGRAM_BOT_TOKEN=<from @BotFather>
XMLPROXY_API_KEY=<from xmlproxy.ru>
FERNET_KEY=<generated via python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())">
```

**Настройка Telegram Mini App:**
1. Создать бота через @BotFather
2. Включить Web App: `/mybots` → Bot Settings → Menu Button → задать URL `https://your-domain.com/m/`
3. В платформе: пользователь привязывает Telegram ID в профиле
4. Готово — Mini App открывается из меню бота

**Настройка Метрики:**
1. Получить OAuth-токен Яндекс.Метрики
2. В платформе: Сайт → Настройки → ввести token + counter_id
3. Трафик доступен в десктоп и мобильной версии

**Регулярные задачи (Celery Beat):**
- Ежедневный утренний дайджест (Telegram + email)
- Еженедельный отчёт по расписанию
- Автокраулинг по расписанию (настраивается per-site)
- Очистка уведомлений старше 30 дней (03:00)

## POST 12: Для разработчиков

#стек #разработка

**Архитектура для разработчиков**

**Структура проекта:**
```
app/
├── routers/          # FastAPI endpoints (20+ файлов)
├── services/         # Бизнес-логика (30+ сервисов)
├── models/           # SQLAlchemy 2.0 модели
├── tasks/            # Celery задачи
├── templates/        # Jinja2 шаблоны
│   ├── mobile/       # Мобильные шаблоны
│   │   └── partials/ # HTMX-фрагменты
│   └── ...           # Десктоп шаблоны
└── static/           # CSS, JS, иконки
```

**Паттерны:**
- Async everywhere: `AsyncSession`, `async def`, `await`
- HTMX partials: запрос с `HX-Request` header → возврат фрагмента HTML
- Celery tasks: `retry=3`, `acks_late=True`, `task_track_started=True`
- Service layer: тонкие роутеры, толстые сервисы
- Фильтрация: query params → WHERE clause в сервисе → HTMX partial refresh

**Мобильная архитектура:**
- Все мобильные роуты в одном файле: `app/routers/mobile.py`
- Все шаблоны наследуют `base_mobile.html`
- Bottom navigation с 4 табами
- Toast: `showToast(msg, type)` — глобальная JS-функция
- HTMX: `hx-get/hx-post` + `hx-target` + `hx-swap="innerHTML"` для SPA-like UX
- Touch targets: минимум 44px (Apple/Google HIG)
- `hx-disabled-elt="this"` для защиты от double-tap

**Telegram WebApp flow:**
```
User in Telegram → Opens /m/
  → Telegram SDK injects initData
  → POST /m/auth/telegram-webapp (initData)
  → Server: HMAC-SHA256 validation
  → Lookup User by telegram_id
  → Set JWT cookie (httponly, 24h)
  → Page reload → authenticated
```

**База данных:**
- 45+ таблиц
- Партиционирование `keyword_positions` по месяцам
- Кэш-таблица `keyword_latest_positions` (flat, без партиций)
- `PageTrafficSnapshot` — кэш данных Метрики
- Все миграции через Alembic (45+ миграций)

**Тестирование:**
- pytest + httpx AsyncClient
- Factory Boy для фикстур
- respx для мока HTTP-запросов
- Playwright для E2E сценариев (YAML-формат)
